<p>Are you having trouble exporting data from your instance into an Excel file? Do you find that all of your data is not exported or that portions of records are missing from your reports?</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>The reason for this misplaced data may be that the number of records you are trying to export is higher than the maximum number of rows that Excel allows. In some cases, even though all records have been exported successfully into the file, Excel will display only a certain number of rows if your data exceeds the limit.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>Excel is intended for relatively small exports, generally 50,000 rows. Older versions of Excel can only export up to 65,000 rows while newer versions (Office 2007 or later) have a maximum limit of just over 1,000,000 rows. If your data number is larger than this, you may not be able to successfully export, and receive this notification:<br/><br/></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3266-12015/ExcelExportThreshold1.png"><img  alt="ExcelExportThreshold1.png" class="image-0 jive-image jiveImage" height="137" src="963b148edbd45344e9737a9e0f961936.iix" style="margin-left: auto; margin-right: auto; display: block;" width="505"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>To fix, it is recommended to export your data into a different file type, such as CSV or XML, which can handle larger exports.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>Another option is to increase the maximum export cell threshold. You can do this by customizing the threshold in the <span style="font-family: courier new,courier;">glide.excel.max_cells</span> integer property. See <a title="k-external-small" class="jive-link-external-small" href="http://wiki.servicenow.com/index.php?title=Excel_Export_Threshold#Excel_Export_Threshold" rel="nofollow" target="_blank">Excel Export Threshold</a> for more information.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>Note that increasing the threshold to a higher number can cause resource issues in your instance. If you have a very high number of records to export and need to use Excel, you can break up these records into smaller groups. See <a title="k-external-small" class="jive-link-external-small" href="http://wiki.servicenow.com/index.php?title=Exporting_Data#Breaking_Up_Large_Exports" rel="nofollow" target="_blank">Breaking Up Large Exports</a> for more information.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>For more information on troubleshooting exports, see:</p><ul><li><a title="k-external-small" class="jive-link-external-small" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0538324" rel="nofollow" target="_blank">Determine if the configured maximum rows to export is too low</a> (KB0538324)</li><li><a title="k-external-small" class="jive-link-external-small" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0538306" rel="nofollow" target="_blank">Determine if the version of Excel cannot handle the data set size</a> (KB0538306)</li></ul><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>See <a title="k-external-small" class="jive-link-external-small" href="http://wiki.servicenow.com/index.php?title=Exporting_Data#Export_Limits" rel="nofollow" target="_blank">Export Limits</a> and <a title="" _jive_internal="true" data-containerid="2102" data-containertype="14" data-objectid="695346" data-objecttype="2" href="/community?id=community_question&sys_id=ff684beddb1cdbc01dcaf3231f961999">Exporting Large Data Sets</a> for additional information.</p>
