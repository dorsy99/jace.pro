<p>If you are seeing incorrect start and end dates on change requests — for example, users are able to submit or modify a request with a <strong>Planned start date</strong><em> later</em> than the <strong>Planned end date</strong> without a warning message - you can implement a custom script that verifies the start and end dates <em>before</em> you enter the time warp:</p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3582-15233/Screen Shot 2014-10-30 at 12.22.25 PM.JPG"><img  alt="Screen Shot 2014-10-30 at 12.22.25 PM.JPG" class="image-0 jive-image" height="360" src="4c3bbb79db54d3049c9ffb651f961997.iix" style="height: 360px; width: 328.208px; display: block; margin-left: auto; margin-right: auto;" width="328"/></a></p><p>A date validation script for the start- and end-date fields can be set to trigger on the change request record. This script will check the start date against the current date, and display an error message if the start date is <em>before</em> the current date. It will also check the end date against the start date, and also display an error message if the end date is <em>before</em> the current date.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>To verify these dates on a change request, one way is to create a business rule before insert or update:</p><ul><li>For example: <strong>Condition = current.start_date.changes() || current.end_date.changes() || current.work_start.changes() || current.work_end.changes())</strong></li><li>Use this script:</li></ul><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: 'courier new', courier;">if (!current.end_date.nil() &amp;&amp; !current.start_date.nil()) {<br/>if (current.end_date.getGlideObject().getNumericValue() &lt;=<br/>current.start_date.getGlideObject().getNumericValue() ) {<br/>gs.addErrorMessage('Planned start must be before Planned end');<br/>current.start_date.setError('Planned start must be before Planned end');<br/>current.setAbortAction(true);<br/>}<br/>}<br/>if (!current.work_start.nil() &amp;&amp; !current.work_end.nil()) {<br/>if (current.work_end.getGlideObject().getNumericValue() &lt;=<br/>current.work_start.getGlideObject().getNumericValue() ) {<br/>gs.addErrorMessage('Actual start must be before Actual end');<br/>current.work_start.setError('Actual start must be before Actual end');<br/>current.setAbortAction(true);</span><br/>}<br/>}</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>This message should now appear at the top of the change request if the dates are incorrect:</p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3582-15224/Screen Shot 2014-10-30 at 12.10.30 PM.JPG"><img  alt="Screen Shot 2014-10-30 at 12.10.30 PM.JPG" class="image-1 jive-image" height="319" src="f5665042db189fc03eb27a9e0f96194a.iix" style="height: 319px; width: 325.51px; display: block; margin-left: auto; margin-right: auto;" width="326"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>For more about using scripts to verify dates, see:</p><p><a title="k-external-small" class="jive-link-external-small" href="http://wiki.servicenow.com/index.php?title=Verify_Request_Start_and_End_Dates" rel="nofollow" target="_blank">Verify Start and End Dates</a></p><p><a title="k-external-small" class="jive-link-external-small" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0535508" rel="nofollow" target="_blank">Change requests can be saved with a start dates later than the end date without warnings</a></p>
