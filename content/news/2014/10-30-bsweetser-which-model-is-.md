<p><span style="font-family: Arial;">In the third part of this series on Model Management, I provided you with some perspectives on Model Names and Model numbers and how to more effectively set them up in your environment. There was a little problem, though: When you try to work with those Models, only the <strong>Display name</strong> field displays when you need to select a Model. </span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15164/Select HP Model without update to see number.png"><img  alt="Select HP Model without update to see number.png" class="image-0 jive-image" height="251" src="856bd982db58d304b322f4621f9619d3.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="379"/></a></p><p><span style="font-family: Arial;">One easy way to get around this is to use the magnifying glass to bring up the list of items. The default view here displays the <strong>Model number</strong> in the list to help you select which Model you want. </span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15174/Product Models list - no short description.png"><img  alt="Product Models list - no short description.png" class="image-1 jive-image" height="412" src="69777042db9cdfc03eb27a9e0f961958.iix" style="height: 356px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="717"/></a></p><p><span style="font-family: Arial;">If you want more complete information here, though, you might add the <strong>Short description</strong> to this list. An administrator would need to navigate to here to <strong>Personalize &gt; List Layout</strong>, but the <strong>Short description</strong> can make your decision much clearer.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15175/Product Models list - short description.png"><img  alt="Product Models list - short description.png" class="jive-image image-2" height="498" src="bd67d8cadbd41f048c8ef4621f961904.iix" style="height: 431px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="717"/></a></p><p><span style="font-family: Arial;">This is still not very fast, though. I like fast. I want to see the Model number when I type the Model to help make my decision. To do this, right-click the Model label and select <strong>Personalize Dictionary</strong>. </span></p><p><span style="font-family: Arial;"><br/></span></p><p><span style="font-family: Arial;"><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15176/Personalize Dictionary.png"><img  alt="Personalize Dictionary.png" class="jive-image image-3" height="294" src="b867f37ddb90d3049c9ffb651f96193c.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="331"/></a><br/></span></p><p><span style="font-family: Arial;">Append <strong>ref_auto_completer=AJAXTableCompleter,ref_ac_columns=model_number,ref_ac_order_by=name</strong> to the Attributes and <strong>Update</strong>. Note: because this is a comma separated list, you need to include a comma after the existing text.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15177/Add attributes to see the Model number.png"><img  alt="Add attributes to see the Model number.png" class="jive-image image-4" height="382" src="356bd982db58d304b322f4621f961957.iix" style="height: 253px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="936"/></a></p><p><span style="font-family: Arial;">Try it out:</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p style="text-align: center;"><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15178/Search model with Model number.png"><img  alt="Search model with Model number.png" class="jive-image image-5" height="212" src="4e77bc02dbd89344e9737a9e0f96199b.iix" style="height: auto; border: #000 solid 1px;" width="481"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">This is closer, but if I know what the Model number is, I want to be able to type that value instead of the Model name. To do this, Personalize Dictionary again and append <strong>ref_ac_columns_search=true</strong> to the Attributes. Again, you need a comma after the existing text. Now the search allows me to type the Model number directly into the field:</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3588-15179/Search by Model number.png"><img  alt="Search by Model number.png" class="jive-image image-6" height="178" src="8f669042db189fc03eb27a9e0f961935.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="481"/></a></p><p><span style="font-family: Arial;">Now this makes the creation of a new asset much quicker. If the Model number can be scanned, I could potentially select the Model with a scanner. </span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">Up next in the series on Model Management is how to use ServiceNow's Normalization capabilities to support this name setup.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">If you missed the previous posts:</span></p><ul><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3535" data-objecttype="38" href="/community?id=community_blog&sys_id=4fdd2ae9dbd0dbc01dcaf3231f9619ef">Role Model, or Why Model Management is Important</a></li><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3562" data-objecttype="38" href="/community?id=community_blog&sys_id=246d6a29dbd0dbc01dcaf3231f9619d9">Models vs. Model Categories: Model Management, Part 2</a></li><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3567" data-objecttype="38" href="/community?id=community_blog&sys_id=f23e22addbd0dbc01dcaf3231f961912">What's in a Name?: Model Management Part 3</a></li></ul>
