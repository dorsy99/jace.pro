<p>If you are trying to create an email notification using the HTML editor, you may find yourself adding excessive space every time you hit return. What's going on here? If you look in the source code, you may see something similar to this:</p><p></p><p><a _jive_internal="true" href="/servlet/JiveServlet/downloadImage/38-3719-16385/Screen Shot 2014-12-10 at 3.09.16 PM.JPG"><img   alt="Screen Shot 2014-12-10 at 3.09.16 PM.JPG" class="image-0 jive-image" height="428" src="abe05546db989f048c8ef4621f9619bd.iix" style="height: 428px; width: 570.963px;" width="571"/></a></p><p></p><p>If the email notification content type is<strong> HTML</strong>, and you use return to create a line break, the HTML editor will insert extra paragraph tags (<strong>&lt;p&gt;</strong> <strong><strong>&amp;nbsp &lt;/p&gt;</strong></strong>) to your notification, which will cause the gaping white space.</p><p></p><p>There are some basic ways to fix this issue:</p><ul><li>If you want to keep lines closer together, hold down the <strong>Shift </strong>key while you press<strong> Return</strong>. This creates a single line break, which is &lt;br&gt; in HTML.</li><li>You can also choose to create the HTML template in a JavaScript/HTML editor first, and then copy that to your email client template.</li><li>If you're really desperate, just go into the HTML editor and manually delete out the <strong><strong>&lt;p&gt; &amp;nbsp &lt;/p&gt;</strong></strong> to close the space.</li><li>If you are familiar with CSS, you can apply a new style to the &lt;p&gt; tags to keep paragraphs closer together. For example, <strong>&lt;p style="margin: 0px; padding: 0px;"&gt;</strong> or <strong>&lt;style&gt;p {margin: 0px; padding: 0px;}&lt;/style&gt;</strong> into the HTML field to set a default value.</li><li>In addition, when editing an existing notification that contains literal carriage returns, these breaks appear as <strong>&lt;div&gt;</strong> <strong>&amp;nbsp</strong>;<strong><span style="font-size: 12.0pt; font-family: Cambria; mso-ascii-theme-font: minor-latin; mso-fareast-font-family: 'Times New Roman'; mso-hansi-theme-font: minor-latin; mso-bidi-font-family: 'Times New Roman'; mso-ansi-language: EN-US; mso-fareast-language: EN-US; mso-bidi-language: AR-SA;"> </span>&lt;/div&gt;</strong> (&lt;p&gt; tag's mischievous cousin). These can also be replaced with &lt;br&gt; tags to fix the gap.</li></ul><p></p><p>For more information, see:</p><p><a>ServiceNow KB: Extra HTML "paragraph" tags being added for carriage returns sent in the email client whe (KB0535106)</a></p><p><a title="Excessive HTML Spacing when using the Email Client (&amp;lt;p&amp;gt; &amp;lt;/p&amp;gt;)" __default_attr="684668" __jive_macro_name="message" class="jive_macro_message jive_macro" data-orig-content="Excessive HTML Spacing when using the Email Client (&amp;amp;lt;p&amp;amp;gt; &amp;amp;lt;/p&amp;amp;gt;)" href="/community?id=community_question&sys_id=e5a70fa9db1cdbc01dcaf3231f9619a1">Excessive HTML Spacing when using the Email Client (&amp;lt;p&amp;gt; &amp;lt;/p&amp;gt;)</a></p><div><a __default_attr="3697" __jive_macro_name="blogpost" class="jive_macro jive_macro_blogpost" data-orig-content="&amp;lt;div&amp;gt; tags messing with your email client templates" href="/community?id=community_blog&sys_id=269d6e69dbd0dbc01dcaf3231f961956" modifiedtitle="true" title="&amp;lt;div&amp;gt; tags messing with your email client templates">&lt;div&gt; tags messing with your email client templates</a></div>
