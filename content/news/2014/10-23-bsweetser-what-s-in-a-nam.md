<p style="text-align: center;"><span style="font-family: Arial;">What's in a name? that which we call a rose </span></p><p style="text-align: center;"><span style="font-family: Arial;">By any other name would smell as sweet;</span></p><p style="text-align: center;"><span style="font-family: Arial;">-Juliet, Shakespeare's <em>Romeo and Juliet</em></span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>This is part 3 in a series of posts on Model Management.   If you have not read them, be sure check out the earlier posts on <a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3535" data-objecttype="38" href="/community?id=community_blog&sys_id=4fdd2ae9dbd0dbc01dcaf3231f9619ef">why Model Management is important</a> and how <a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3562" data-objecttype="38" href="/community?id=community_blog&sys_id=246d6a29dbd0dbc01dcaf3231f9619d9">Models and Model Categories relate</a>.</p><p><span style="font-family: Arial;"><br/></span></p><p><span style="font-family: Arial;">There are a few things about Models that I like to make sure people understand when they take the ServiceNow Asset Management class:</span></p><ul><li><span style="font-family: Arial;">The Manufacturer's name does not belong in the Model name</span></li><li><span style="font-family: Arial;">Each Model and default configuration should have its own record</span></li><li><span style="font-family: Arial;">Model numbers should be unique</span></li></ul><p><span style="font-family: Arial;">Why not put the Manufacturer in the Model name? You might call a car a Ford Fusion, but if you were to fill it out on a form, the Make (Manufacturer) would be Ford, and the Model would be Fusion. The same applies to your Models in ServiceNow. To make it easier to find and manage, though, Models have a Display name that joins these items to give you the more familiar Ford Fusion. If you put Ford in the Model Name, you would then end up with a Ford Ford Fusion, which would just leave people in a state of con-Fusion.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14880/Screen Shot 2014-10-21 at 9.35.04 PM.png"><img  alt="Screen Shot 2014-10-21 at 9.35.04 PM.png" class="image-0 jive-image" height="112" src="7b2a6f31dbd09b048c8ef4621f961940.iix" style="height: 74px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="938"/></a></p><p><span style="font-family: Arial; font-size: 10pt; line-height: 1.5em;">Each default configuration of a Model has its own Model number from the Manufacturer. This includes different default configurations of the same Model. To help distinguish, each configuration has its own Model Number and Short description.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14881/Screen Shot 2014-10-21 at 9.36.31 PM.png"><img  alt="Screen Shot 2014-10-21 at 9.36.31 PM.png" class="image-1 jive-image" height="698" src="d302918edb989f048c8ef4621f96193a.iix" style="height: 457px; width: 620px; display: block; margin-left: auto; margin-right: auto;" width="946"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">With the same Name for each of these Models, the Model number acts as the unique identifier for the various configurations. You can use Short description to identify the details about each of the Model's default configurations. Here is an example of three different default configurations of a single Model and how to configure them:</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14882/HP EliteBook 850 G1 E3W20UT.png"><img  alt="HP EliteBook 850 G1 E3W20UT.png" class="jive-image image-2" height="241" src="6dc920cedbd457049c9ffb651f96199c.iix" style="height: 159px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="938"/></a></p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14883/HP EliteBook 850 G1 E3W21UT.png"><img  alt="HP EliteBook 850 G1 E3W21UT.png" class="jive-image image-3" height="241" src="830b880edb5017041dcaf3231f9619d5.iix" style="height: 159px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="938"/></a></p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14884/HP EliteBook 850 G1 G4U.png"><img  alt="HP EliteBook 850 G1 G4U.png" class="jive-image image-4" height="241" src="f248eb3ddb14dfc0b322f4621f961986.iix" style="height: 159px; width: 620px; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="938"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">Each of these is a Hewlett-Packard EliteBook 850 G1, but they have different default configurations, as described in their Short descriptions.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">Why would you do this? I make this recommendation because it allows you to easily report on the model as a whole regardless of default configuration:</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14885/HP Computers by Model Name.png"><img  alt="HP Computers by Model Name.png" class="jive-image image-5" height="365" src="1d465d8edb50d7041dcaf3231f961984.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;" width="383"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">But, if necessary, you can also report on the default configurations separately:</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14886/HP Computers by Model Number.png"><img  alt="HP Computers by Model Number.png" class="jive-image image-6" height="365" src="eb853086db9c5f048c8ef4621f96193c.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;" width="383"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">These charts contain a subtle grouping setting that is important. The first report is grouped by Model Name, not by Model. If you group by Model, the report looks similar to the grouping by Model number, but it does not show the distinctions between the Models in the chart's key:</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14887/HP Computers by Model.png"><img  alt="HP Computers by Model.png" class="jive-image image-7" height="365" src="a960584edb9417049c9ffb651f9619da.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;" width="383"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">The ability to make this distinction in your reports allows you to more effectively:</span></p><ul><li><span style="font-family: Arial;">Match up compatible add-ons, such as docking stations</span></li><li><span style="font-family: Arial;">Identify replacement parts </span></li><li><span style="font-family: Arial;">Find systems affected by a recall</span></li></ul><p><span style="font-family: Arial;">This is not an exhaustive list, and there may be other opportunities for these reports to provide value to your organization.</span></p><p><span style="font-family: Arial;"><br/></span></p><p><span style="font-family: Arial;">There you have it: some recommendations for your Model Names, which is an important aspect of Model Management. There is one drawback to this approach, though. When you add a new asset, it becomes more difficult to select the Model for your asset because all the default configurations display with the same name: </span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3567-14888/Select HP Model without update to see number.png"><img  alt="Select HP Model without update to see number.png" class="jive-image image-8" height="251" src="b7fe2b71dbd4df04e9737a9e0f9619f0.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto; border: #000 solid 1px;" width="379"/></a></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">In part 4, I cover some steps to make it easier to identify the different Models in the interface.</span></p>
