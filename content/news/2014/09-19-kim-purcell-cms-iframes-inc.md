<p>If you have recently upgraded to Dublin, you may have encountered the problem of wonky-sized iframes in IE9. This is because the CMS is no longer rendering the height of your iframes properly. Not an optical illusion - you'll see things like your iframes briefly appearing and then reducing in height.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>Yes, very odd. It's the property added in the Dublin release — when <strong>height: auto;</strong> is not present in the CSS, the main content frame is automatically set to 10px when the page is viewed in IE9.</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>If you've made customizations to your CMS and modified the style sheet, then this is likely what's causing your problem when upgrading. The good new is, this is no longer an issue in Dublin Patch 2, so upgrade if you can. If this is not an option, just add <strong>height: auto; </strong>in the BODY tag of your <strong>Base</strong> <strong>CSS style sheet</strong>:</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>BODY {</p><p>       margin-top: 5px;</p><p>       margin-left: 5px;</p><p>       margin-bottom: 5px;</p><p>               margin-right: 5px;</p><p>       padding:2px;</p><p>       border: 2x;</p><p>       color: #666666</p><p><strong>       height: auto;</strong></p><p>}</p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p>For more information, see:</p><ul><li><a title="k-external-small" class="jive-link-external-small" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0534768" rel="nofollow" target="_blank">CMS pages size incorrectly in IE9 when BODY height is not set to auto</a></li><li><a title="k-external-small" class="jive-link-external-small" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0529664" rel="nofollow" target="_blank">"Expand to fit Content" sizing property is not working for iframes</a></li><li><a title="k-external-small" class="jive-link-external-small" href="http://wiki.servicenow.com/index.php?title=Using_Content_Blocks#iFrames" rel="nofollow" target="_blank">iFrames</a></li></ul>
