<p><span style="font-family: Arial;">This is Part 5 in a series of blog posts on Model Management. If you have not read the others, you can find them here:</span></p><ul><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3535" data-objecttype="38" href="/community?id=community_blog&sys_id=4fdd2ae9dbd0dbc01dcaf3231f9619ef">Role Model, or Why Model Management is Important</a></li><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3562" data-objecttype="38" href="/community?id=community_blog&sys_id=246d6a29dbd0dbc01dcaf3231f9619d9">Models vs. Model Categories: Model Management, Part 2</a></li><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3567" data-objecttype="38" href="/community?id=community_blog&sys_id=f23e22addbd0dbc01dcaf3231f961912">What's in a Name?: Model Management Part 3</a></li><li><a title="" _jive_internal="true" data-containerid="2927" data-containertype="37" data-objectid="3588" data-objecttype="38" href="/community?id=community_blog&sys_id=0bad66a9dbd0dbc01dcaf3231f96197a">Which Model Is Which?: Model Management Part 4</a></li></ul><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">Up to this point, I provided you with suggestions on how to name Models and how to make it easier to select the Model you want, but how do you make the Model names consistent without a lot of extra manual work?</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">This is where you can leverage the ServiceNow platform's normalization capabilities. Normalization allows you to find different values and set them to a single, consistent value. To keep consistent with the example from earlier posts in the series, consider the different possibilities you might end up with for a Hewlett-Packard EliteBook 850 G1 Model name. </span></p><p><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3641-15571/Screen Shot 2014-11-11 at 10.34.13 AM.png"><img  alt="Screen Shot 2014-11-11 at 10.34.13 AM.png" class="image-0 jive-image" height="544" src="584fd08edbd85304b322f4621f961962.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;" width="543"/></a></p><p><span style="font-family: Arial;">How do you end up with inconsistent data? You may have multiple sources for your Model data:</span></p><ul><li><span style="font-family: Arial;">Discovery source</span></li><li><span style="font-family: Arial;">Vendor data</span></li><li><span style="font-family: Arial;">Manual input</span></li></ul><p><span style="font-family: Arial;">To get consistent data, you can create a Normalization configuration to handle the different possibilities. Here are some steps you can use to create a Normalization configuration for your Model name with some recommendations to facilitate Model management. Before you begin, create a new field on the Product Model [cmdb_model] table to store the original raw value from the Name field.</span></p><p><span style="font-family: Arial;"><br/></span></p><ol><li><span style="font-family: Arial;">Navigate to <strong>Field Normalization &gt; Configurations &gt; Normalizations</strong>.</span></li><li><span style="font-family: Arial;">Click <strong>New</strong>.</span></li><li><span style="font-family: Arial;">Name the Normalization configuration and set the Table to <strong>Product Model [cmdb_model]</strong> and Field to <strong>Name</strong>.</span></li><li><span style="font-family: Arial;">Right-click the form header and <strong>Save</strong>.</span></li><li><span style="font-family: Arial;">Set the Raw field value to the field you created to store the raw value. This can be used to pull other information you might find useful, such as the Model number if your source provides that as part of the Model name.</span></li><li><span style="font-family: Arial;">Create Normal Values for each of the Models you manage. </span></li><li><span style="font-family: Arial;">When possible, create Rules for your Normal Values to facilitate the possibility of new options that match specific criteria. For example, you could create a Rule that finds the text <strong>elitebook</strong> and <strong>850</strong> and <strong>g1</strong> to match different possible variations of the name. <br/></span><a _jive_internal="true" href="/servlet/JiveServlet/showImage/38-3641-15572/Normalization Rule.png"><img  alt="Normalization Rule.png" class="image-1 jive-image" height="298" src="ec03000edb1c5704ed6af3231f96191b.iix" style="height: 197px; width: 620px; border: #000 solid 1px;" width="936"/></a></li><li><span style="font-family: Arial;">Test the Normal Values and set the Mode to <strong>Active</strong> when you are ready. </span></li></ol><p><span style="font-family: Arial;"><br/></span></p><p><span style="font-family: Arial;">For additional safety, you can create Normalization configurations in a test instance and capture them in an Update Set. </span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">If you are interested in more details on how to work with Field Normalization, be sure to check the <a title="k-external-small" class="jive-link-external-small" href="http://wiki.servicenow.com/index.php?title=Normalizing_a_Field" rel="nofollow" target="_blank">Wiki</a> for details. Normalization is also covered in more detail in our <a title="k-external-small" class="jive-link-external-small" href="http://www.servicenow.com/services/training-and-certification/asset-management.html" rel="nofollow" target="_blank">Asset Management</a> and <a title="k-external-small" class="jive-link-external-small" href="http://www.servicenow.com/services/training-and-certification/discovery.html" rel="nofollow" target="_blank">Discovery</a> courses.</span></p><p style="min-height: 8pt; height: 8pt; padding: 0px;">  </p><p><span style="font-family: Arial;">Coming soon: Part 6 will explore how to work with Models as the default configuration for items in the Service Catalog. </span></p>
