<p><span style="font-size: 12pt;">1). What is web service? </span><br /><span style="font-size: 12pt;"> A web service is any piece of software that makes itself available over the internet and uses a standardized XML messaging system. XML is used to encode all communications to a web service. For example, a client invokes a web service by sending an XML message, then waits for a corresponding XML response.</span></p>
<p><span style="font-size: 12pt;">2). Scoped application debugging / Scoped script logging </span><br /><span style="font-size: 12pt;"> gs.error(), gs.warn(), gs.info(), gs.debug()</span><br /><span style="font-size: 12pt;">Note: gs.log() and gs.print () will not support in scoped application.</span><br /> <br /><span style="font-size: 12pt;"><strong>Refer:</strong> https://developer.servicenow.com/app.do#!/document/content/app_store_doc_scoped_script_helsinki_c_ScopedScriptLogging?v&#61;madrid</span></p>
<p><span style="font-size: 12pt;">3). Difference between * and none in ACL?</span><br /><span style="font-size: 12pt;">The easiest way I found I understood them was the following:</span><br /><span style="font-size: 12pt;">incident.assigned_to &#61; a rule for this specific field (assigned_to), in this table (incident). </span><br /><span style="font-size: 12pt;"> Think of it as a key for a certain cabinet in a room</span><br /><span style="font-size: 12pt;">incident.* &#61; a rule for other fields that aren&#39;t covered by specific field ones. </span><br /><span style="font-size: 12pt;"> Think of it as an unlabelled key that is used if you aren&#39;t given a cabinet key</span><br /><span style="font-size: 12pt;">incident.NONE &#61; a rule for the table itself. </span><br /><span style="font-size: 12pt;"> Think of it as a key to the room&#39;s door - if there&#39;s no table.none rule, all other field rules are redundant.</span><br /><span style="font-size: 12pt;">On our sysadmin courses, I demonstrate this by creating a new table and permit app access to a user but the user finds they can&#39;t see the underlying table until I add some ACLs. </span><br /><span style="font-size: 12pt;"> Once I add table.* &#61; read:userrole, they can see the list (and also note that a table.NONE rule has been added) but later when I remove this .NONE rule, any other read rules are completely ignored.</span></p>
<p><span style="font-size: 12pt;">In my opinion, the .NONE rule doesn&#39;t really add further security that can&#39;t be handled by a combination of .* and .fieldname rules. </span><br /><span style="font-size: 12pt;"> Other than the constraint of it having to be there, I&#39;ve yet to be given a use case where it&#39;s needed - it feels like an unnecessary extra step that doesn&#39;t actually add anything.</span></p>
<p><br /><span style="font-size: 12pt;"><strong>Refer :</strong> https://community.servicenow.com/community?id&#61;community_question&amp;sys_id&#61;fa788feddb1cdbc01dcaf3231f961966</span></p>
<p> </p>
<p><span style="font-size: 12pt;">4). How to restrict access to module/ how to give access to module?</span><br /><span style="font-size: 12pt;">      Using role</span><br /><span style="font-size: 12pt;"><strong>     Note:</strong> ACL is not required to give access to the application/module.</span></p>
<p> </p>
<p><span style="font-size: 12pt;">5). Which one best practice in using hasRole(), server side or client side?</span><br /><span style="font-size: 12pt;">     server side</span></p>
<p><span style="font-size: 12pt;">6). Purpose of hasRole() and hasExactlyRole()?</span><br /><span style="font-size: 12pt;">     Just an example use case for your understanding.</span></p>
<p><span style="font-size: 12pt;">g_user.hasRole(&#39;itil&#39;) True if the current user has the role specified, false otherwise. ALWAYS returns true if the user has the &#39;admin&#39; role.</span><br /><span style="font-size: 12pt;"> So for admin it will return true</span></p>
<p><span style="font-size: 12pt;">hasRoleExactly(&#39;itil&#39;) True if the current user has the exact role specified, false otherwise, regardless of &#39;admin&#39; role.</span><br /><span style="font-size: 12pt;"> Will return false for admin</span></p>
<p><span style="font-size: 12pt;">7). Why we can create custom application? </span><br /><span style="font-size: 12pt;"> i). To replace the outdated, inadequate, custom business applications and process</span></p>
<p><span style="font-size: 12pt;">ii). To extend service delivery and management to all enterprise departments</span><br /> <br /><span style="font-size: 12pt;"> iii). To extend the value of Service Now</span><br /> <br /><span style="font-size: 12pt;"> iv). To bring greater of automation and consolidation to enterprise services and their management.</span></p>
<p><span style="font-size: 12pt;">8). What is Application? </span><br /><span style="font-size: 12pt;"> Applications are a collection of files and data that deliver a service and manage business processes.</span></p>
<p><span style="font-size: 12pt;">Refer: https://docs.servicenow.com/bundle/london-application-development/page/build/applications/concept/c_Applications.html</span></p>
<p><span style="font-size: 12pt;">9). Best practice in service now while creating new application ? </span><br /><span style="font-size: 12pt;"> Scoped application, because scoped applications are more secure.</span></p>
<p><span style="font-size: 12pt;">10). Which one is not supported in flow designer?</span><br /><span style="font-size: 12pt;"> Run as user</span></p>
<p><span style="font-size: 12pt;">11). Can inbound used send emails in service now?</span><br /><span style="font-size: 12pt;"> No</span></p>
<p> </p>
<p><span style="font-size: 12pt;">12). What is purpose of weight field on notification form ? </span><br /><span style="font-size: 12pt;"> When there are multiple notifications in the Outbox that are from the same notification Table and addressed to the same recipients, </span><br /><span style="font-size: 12pt;"> notifications with the Highest Weight are sent - the default value 0 causes ServiceNow to always send the notification. (assuming the Conditions are met).</span></p>
<p><span style="font-size: 12pt;">13). Purpose of When to send/ Who will receive / What it will contain in notifications?</span><br /><span style="font-size: 12pt;"> Use the following tabs in the Notification form to configure an email notification:</span></p>
<p><br /><span style="font-size: 12pt;"> i) When to send — Conditions required to send the notification.</span></p>
<p><span style="font-size: 12pt;">ii) Who will receive — Recipients of the notification.</span></p>
<p><span style="font-size: 12pt;">iii) What it will contain — Contents of the notification.</span><br /><span style="font-size: 12pt;">What Digest will contain — Contents of the email digest if the notification can be delivered in a digest.</span></p>
<p><span style="font-size: 12pt;">Refer : https://docs.servicenow.com/bundle/london-servicenow-platform/page/administer/notification/task/t_CreateANotification.html</span></p>
<p><span style="font-size: 12pt;">14). Did Client script and UI Policy will work based on Views ? </span><br /><span style="font-size: 12pt;"> Yes</span></p>
<p><span style="font-size: 12pt;">15). ACL debugging color? </span><br /><span style="font-size: 12pt;"> A green checkmark Indicates the table or field passed the criteria.</span></p>
<p><span style="font-size: 12pt;">Refer: https://docs.servicenow.com/bundle/london-platform-administration/page/administer/contextual-security/concept/c_AccessControlRulesDebug.html</span></p>
<p><span style="font-size: 12pt;">16). What are the default access for global application?</span><br /><span style="font-size: 12pt;"> Souce control and Delegated development access</span></p>
<p><span style="font-size: 12pt;">17). How to call variable in record producer ? </span><br /><span style="font-size: 12pt;"> producer.variable_name</span></p>
<p><span style="font-size: 12pt;">18). How to remove local changes? </span><br /><span style="font-size: 12pt;"> using Stash</span></p>
<p><span style="font-size: 12pt;">19) Servicenow Default access in scoped application ? </span><br /><span style="font-size: 12pt;"> Can read</span></p>
<p><span style="font-size: 12pt;">Refer: https://docs.servicenow.com/bundle/london-application-development/page/build/applications/concept/c_DefaultDesignAccessPermissions.html</span></p>
<p><span style="font-size: 12pt;">20). Things to consider while creation an application ?</span><br /><span style="font-size: 12pt;"> Process Design</span></p>
<p><span style="font-size: 12pt;">21). What are the functionalities that can be achieved using the flow designer ? </span><br /><span style="font-size: 12pt;"> Refer: https://docs.servicenow.com/bundle/madrid-servicenow-platform/page/administer/flow-designer/concept/flow-designer.html</span></p>
<p><span style="font-size: 12pt;">22). Can we access old value in UI Policy </span><br /><span style="font-size: 12pt;"> No</span></p>
<p><span style="font-size: 12pt;">23). Which one will runs first UI Policy action or UI Policy script?</span><br /><span style="font-size: 12pt;"> UI Policy script</span></p>
<p><span style="font-size: 12pt;">24). Can we delete scoped application record in other scope?</span><br /><span style="font-size: 12pt;"> TRUE</span></p>
<p><span style="font-size: 12pt;">25). Which operation is not supported in REST API Explorer?</span><br /><span style="font-size: 12pt;"> Copy operation</span></p>
<p><span style="font-size: 12pt;">26). Best pratice in servicenow to describe about a field in Servicenow?</span><br /><span style="font-size: 12pt;"> Adding wiki/docs URL about that particular field, example Priority Field on incident form</span></p>
<p><span style="font-size: 12pt;">27). Can we delete a service now record using web services? </span><br /><span style="font-size: 12pt;"> TRUE</span></p>
<p><span style="font-size: 12pt;">28). Purpose of Allow access to this table via web services in Application Access?</span><br /><span style="font-size: 12pt;"> The Allow access to this table via web services option determines whether users can make inbound web services requests to the table. </span><br /><span style="font-size: 12pt;"> The user requesting the access must be authorized to access the records. </span><br /><span style="font-size: 12pt;"> If the user making the request is not authorized, the web services request will not return records even if web services are enabled for the table.</span><br /> <br /><span style="font-size: 12pt;"><strong>Refer:</strong> https://developer.servicenow.com/app.do#!/training/article/app_store_learnv2_securingapps_madrid_securing_applications_against_access_from_other_applications/app_store_learnv2_securingapps_madrid_application_access_web_services?v&#61;madrid</span></p>
<p><span style="font-size: 12pt;">29). What is Application Access in servicenow?</span><br /><span style="font-size: 12pt;"> Application Access is set at the table level and allows developers to:</span></p>
<p><span style="font-size: 12pt;">Grant read, create, update, and delete privileges</span><br /><span style="font-size: 12pt;">Grant access to the table through web services</span><br /><span style="font-size: 12pt;">Allow application files from other application scopes to add new fields to the table</span></p>
<p><span style="font-size: 12pt;"><strong>Refer:  </strong>https://developer.servicenow.com/app.do#!/training/article/app_store_learnv2_securingapps_madrid_securing_applications_against_access_from_other_applications/app_store_learnv2_securingapps_madrid_application_access?v&#61;madrid</span></p>
<p><span style="font-size: 12pt;">30). While creating new table what are all the fields to be added by default? </span><br /><span style="font-size: 12pt;"> Created, created by, updated, updated by, tags, updates</span></p>
<p><span style="font-size: 12pt;">31). What is an event in servicenow? </span><br /><span style="font-size: 12pt;"> ServiceNow can only respond to events that are registered with the Event Registry. Registering an event lets the ServiceNow processes recognize and respond to the event.</span></p>
<p><span style="font-size: 12pt;">32). What is an event Registry in servicenow? </span><br /><span style="font-size: 12pt;"> Event registry is table contains list of events</span></p>
<p><span style="font-size: 12pt;">33). What are the objects accessible in script action scripts in servicenow ? </span><br /><span style="font-size: 12pt;"> Current, event</span></p>
<p><span style="font-size: 12pt;"><strong>Refer :</strong> https://docs.servicenow.com/bundle/madrid-platform-administration/page/administer/platform-events/reference/r_ScriptActions.html</span></p>
<p><span style="font-size: 12pt;">34). If servicenow requesting something then servicenow called as ? </span><br /><span style="font-size: 12pt;"> Consumer</span></p>
<p><span style="font-size: 12pt;">35). If servicenow providing request to outside of servicenow then servicenow called as ? </span><br /><span style="font-size: 12pt;"> Producer</span></p>
<p><span style="font-size: 12pt;">36). What are objects accessible in OnBefore business Rule ?</span><br /><span style="font-size: 12pt;"> The When field on the Business Rule form indicates whether the Business Rule script runs before or after the current object is saved to the database. </span><br /><span style="font-size: 12pt;"> The most commonly used Business Rules are before and after rules.</span><br /><span style="font-size: 12pt;">You can use an async Business Rule in place of an after Business Rule. </span><br /><span style="font-size: 12pt;"> Async Business Rules are similar to after rules in that they run after the database commits a change. </span><br /><span style="font-size: 12pt;"> Unlike after rules, async rules run in the background simultaneously with other processes. </span><br /><span style="font-size: 12pt;"> Async Business Rules allow ServiceNow to return control to the user sooner but may take longer to update related objects.</span></p>
<p><span style="font-size: 12pt;"><strong>Refer :</strong> https://developer.servicenow.com/app.do#!/document/content/app_store_doc_technical_best_practices_london_know_when_to_run_business_rules?v&#61;london</span></p>
<p><span style="font-size: 12pt;">37). What is Team Development in servicenow?</span><br /><span style="font-size: 12pt;"> Team Development supports parallel development on multiple, non-production ServiceNow instances.</span></p>
<p><span style="font-size: 12pt;">Team Development provides the following features:</span><br /><span style="font-size: 12pt;">Branching operations, including pushing and pulling record versions between instances.</span><br /> <br /><span style="font-size: 12pt;">The ability to compare a development instance to other development instances.</span><br /><span style="font-size: 12pt;">A central dashboard for all Team Development activities.</span><br /> <br /><span style="font-size: 12pt;"><strong>Refer :</strong> https://docs.servicenow.com/bundle/madrid-application-development/page/build/team-development/concept/c_TeamDevelopment.html</span></p>
<p><span style="font-size: 12pt;">38). Can we override the application menu role ? </span><br /><span style="font-size: 12pt;"> True</span></p>
<p><span style="font-size: 12pt;"><strong>Refer :</strong> https://hi.service-now.com/kb_view.do?sysparm_article&#61;KB0716421</span></p>
<p><span style="font-size: 12pt;">39). Can we call a workflow in flow designer?</span><br /><span style="font-size: 12pt;"> Yes (We can call workflow in flow designer - Madrid Version)</span></p>
<p><span style="font-size: 12pt;">40). What Are Application Properties?</span><br /><span style="font-size: 12pt;"> Application properties are configurable application parameters to change an application&#39;s behavior. </span><br /><span style="font-size: 12pt;"> In the typical case, only system administrators and/or application administrators can read and write application properties.</span><br /><span style="font-size: 12pt;"><strong>Refer :</strong> https://developer.servicenow.com/app.do#!/lp/servicenow_application_developer/app_store_learnv2_automatingapps_madrid_what_are_application_properties?v&#61;madrid</span></p>
<p><span style="font-size: 12pt;">41). Application Settings ?</span><br /><span style="font-size: 12pt;"> Application settings determine whether an application can access resources from another application. Application designers can configure:</span><br /><span style="font-size: 12pt;"> i). Availability of cross-scope tables in Studio</span><br /><span style="font-size: 12pt;"> ii). Ability for cross-scope scripts to run on application resources</span><br /><span style="font-size: 12pt;"><strong>Refer :</strong> https://developer.servicenow.com/app.do#!/lp/servicenow_application_developer/app_store_learnv2_securingapps_madrid_application_settings?v&#61;madrid </span></p>
