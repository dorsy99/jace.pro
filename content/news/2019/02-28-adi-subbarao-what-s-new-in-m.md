<p>The Madrid release of Virtual Agent is here, and it includes several new and exciting features that benefit both developers and non-coders. These features introduce a better user experience, enhanced productivity, and increased accessibility for each of our customers using Virtual Agent. If there is any specific topic or feature you would like to see covered in more depth, please leave a comment on this post.</p>
<p>Overview of new Madrid features to explore:</p>
<ul><li>Workplace by Facebook Integration</li><li>Additional Conversational Templates</li><li>No Code/Low Code Editors</li><li>Enhanced Data Cards</li><li>Virtual Agent Dashboard</li></ul>
<h4>ServiceNow Chatbots on Workplace by Facebook!          </h4>
<p><img style="max-width: 100%; max-height: 480px; float: left; margin-right: 10px;" src="https://community.servicenow.com/811a06c7dbb72f08a39a0b55ca96196d.iix" width="163" height="215" /></p>
<p> </p>
<p>In addition to supporting Virtual Agent on the Service Portal, Slack, and Microsoft Teams, we’re thrilled to allow customers to deploy enterprise chatbots on Workplace by Facebook.</p>
<p>Developers and administrators will now be able to create conversational workflows and improve customer and employee productivity within Workplace.</p>
<p>The out-of-the-box conversation topics and the ability to transfer to a live agent will also be available using Workplace. We look forward to seeing more chatbots servicing customers and employees on this new messaging channel.</p>
<p> </p>
<p> </p>
<h4>More conversation templates to choose from!</h4>
<p> </p>
<p> </p>
<p><img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/9d9750bbdbb727c4190dfb24399619f1.iix" width="677" height="311" /></p>
<p>With Madrid, we have expanded our pre-defined collection of conversation templates to include Tier 2 support tasks such as FAQs, tutorials, status updates, incident/case creation, diagnostics, feedback surveys, and more. These templates are included out-of-the-box for you to quickly create chatbots in the Virtual Agent Designer.</p>

<h4>Build, test, and update chatbots with less code!</h4>
<p><img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/30697f1fdb7feb8423f4a345ca9619ae.iix" width="676" height="480" /></p>
<p>To make chatbot development a breeze, we’re enabling customers to quickly build logic and make changes to their chatbots. Our new editor experience designer experience allows admins to do string substitutions, build conditional logic, and perform lookup, insert, and update queries without any code. Now, in a matter of a few clicks, you can build, test, and update bots without spending much time or development resources.</p>
<h4>New Look and Feel with Enhanced Data Cards!</h4>
<p>We’re giving Virtual Agent a new look and feel using a more responsive design. Customers can now create data cards to appear natively within chat conversations.</p>
<p><img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/65747727db37e3c4190dfb2439961900.iix" width="222" height="355" />   </p>
<p>These enhanced data cards offer a more visually appealing and condensed way to see real-time ServiceNow data records. This ultimately enhances the Virtual Agent user experience and allows users to interact in the native card formats for Service Portal, Slack, Microsoft Teams, and Workplace.</p>
<p><img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/8fb9d07bdbf727c4190dfb2439961975.iix" width="173" height="116" />  <img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/d72b5433db3b27c4190dfb24399619d8.iix" width="181" height="119" />  <img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/c31a98fbdbf727c4190dfb24399619f8.iix" width="275" height="119" /> <img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/f17b18b3db3b27c4190dfb24399619c4.iix" width="111" height="126" /></p>
<p><strong>Slack enhancements:</strong></p>
<p>Virtual Agent now supports the Slack Enterprise Grid (<a href="https://slack.com/enterprise" rel="nofollow">https://slack.com/enterprise</a>).</p>
<p><strong>Teams enhancements:</strong></p>
<p>Virtual Agent now supports rendering of date/time in calendar format and single select in dropdown format.</p>
<h4>Built-in Chat Analytics and Reporting</h4>
<p><img style="max-width: 100%; max-height: 480px;" src="https://community.servicenow.com/e5db8e03db3b2f08a39a0b55ca961932.iix" /></p>
<p>We are excited to roll out a new <a title="Virtual Agent Analytics" href="https://docs.servicenow.com/bundle/madrid-servicenow-platform/page/use/dashboards/application-content-packs/virtual-agent-dashboard.html" target="_blank" rel="noopener noreferrer nofollow">Virtual Agent dashboard</a> for <a title="Performance Analytics Documentation" href="https://staging-docs-servicenow.zoominsoftware.io/bundle/madrid-release-notes/page/release-notes/performance-analytics-reporting/par-performance-analytics-rn.html" target="_blank" rel="noopener noreferrer nofollow">Performance Analytics</a>. This allows customers to collect real-time insights on the Virtual Agent interactions taking place across all workflows.</p>
<p>This dashboard comes equipped with out-of-the box widgets to measure common KPIs and performance data such as the most used conversation topics, % of cases resolved/deflected, user feedback, and comments.</p>
<p>With the power of analytics built into Virtual Agent, we expect customers to implement best practices and consistently improve their chatbot performance.</p>
<p>We’re looking forward to getting these enhancements into your hands and hearing your feedback, so start planning those Madrid upgrades today! As always, be sure to check the <a title="Virtual Agent - Madrid Release Notes" href="https://docs.servicenow.com/bundle/madrid-servicenow-platform/page/administer/virtual-agent/concept/virtual-agent-overview.html" target="_blank" rel="noopener noreferrer nofollow">release notes</a> for a full list of changes to Virtual Agent.</p>
