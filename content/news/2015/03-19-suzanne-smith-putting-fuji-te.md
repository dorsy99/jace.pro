<p style="text-align: center;"><em>"More than the act of testing, the act of designing tests is one of the best bug preventers known."</em></p><p style="text-align: center;">-Boris Beizer</p><p></p><p>Everyone wants to test their software thoroughly, but deadlines, lack of resources, competing projects, and so many other demands always seem to get in the way of doing the very best testing possible. One way to get a handle on testing is to get better organized. Are you using email and spreadsheets for managing tests? There is an alternative - the new <a title="ki.servicenow.com/index.php?title=Test_Management" href="http://wiki.servicenow.com/index.php?title=Test_Management">Test Management</a> application in the Fuji release. <span style="font-size: 10pt; line-height: 1.5em;">Test Management enables you to create tests, group tests into cases and suites, write test plans, lead testers through execution, manage the process, and review results.</span></p><p></p><h3>Creating tests, test cases, and test suites</h3><p>Test suites contain test cases. Create test suites to logically group test cases together.</p><ol><li>Navigate to <strong>Test Management &gt; Test Repository &gt; Test Suites</strong>.</li><li>Click <strong>New</strong>.</li><li>Add a <strong>Name</strong>.</li><li>Add a <strong>Description</strong> of the suite.<br/><img   alt="test_management1.jpg" class="image-1 jive-image" src="a6a50d4adbd013043eb27a9e0f9619a8.iix" style="height: 225px; width: 620px;"/></li><li>Click <strong>Submit</strong>.</li></ol><p><span style="font-size: 10pt; line-height: 1.5em;"><br/></span></p><p><span style="font-size: 10pt; line-height: 1.5em;">Test cases contain a single or several individual tests.</span></p><ol><li>Navigate to <strong>Test Management &gt; Test Repository &gt; Test Cases</strong>.</li><li>Click <strong>New</strong>.</li><li>Specify a <strong>Test Suite</strong> for the test case.</li><li>Add a <strong>Short Description</strong>.</li><li>If required, add <strong>Prerequisites</strong>.</li><li>Click <strong>Submit</strong>.</li></ol><p></p><p>Individual tests contain steps with an expected outcome. Tests are created on a specific test case.</p><ol><li>Navigate to <strong>Test Management &gt; Test Repository &gt; Test Cases</strong>.</li><li>In the <strong>Short Description</strong> column, click a test case.</li><li>On the <strong>Tests</strong> related list, click <strong>New.</strong></li><li>In <strong>Test</strong>, type the steps necessary for determining if a feature is working correctly.</li><li>In <strong>Expected Result</strong>, add the information to determine if the test passes or fails.<br/><img   alt="test_management2.jpg" class="image-0 jive-image" src="593364cadb5c5344e9737a9e0f96197b.iix" style="height: 333px; width: 620px;"/><br/><br/></li><li>Click <strong>Submit</strong>.<strong><br/></strong></li></ol><p></p><h3>Writing test plans</h3><p>After you have test suites, test cases, and test plans, pull them together into a test plan. Users with the test manager (tm_test_manager) role can write test plans, assign test cases to the appropriate testers, and track progress.</p><ol><li>Navigate to <strong>Test Management &gt; Test Execution &gt; Test Plans</strong></li><li>Click <strong>New</strong>.</li><li>Add a <strong>Short Description</strong>.</li><li>Click <strong>Save</strong>.</li><li>Do one of the following:<ul><li>If you already have a test suite containing the appropriate test cases, go to <strong>Related Links</strong>, click <strong>Add Test Cases from Test Suite</strong>, select a suite, and click OK.</li><li>On the <strong>Test Cases</strong> tab, click <strong>New</strong> (you can follow the steps in the section above).</li></ul></li><li>Click <strong>Save</strong>.</li></ol><p></p><h3>Tips and Tricks when Using Test Management</h3><p></p><ul><li>To get a better sense of the overall testing approach, see <a href="http://wiki.servicenow.com/index.php?title=Test_Management#Process_Flow" title="http://wiki.servicenow.com/index.php?title=Test_Management#Process_Flow">Process Flow</a>. You have a great deal of flexibility in what options you choose to use and the order in which steps take place. Create a process that works for you and your organization.</li><li>Test Management is part of the <a title="ki.servicenow.com/index.php?title=Project_Portfolio_Suite#Applications_and_Features" href="http://wiki.servicenow.com/index.php?title=Project_Portfolio_Suite#Applications_and_Features">Project Portfolio Suite</a> and can be used with other applications in the suite. For example, the project manager can create a test phase in the project workbench.</li><li>There are two <a href="http://wiki.servicenow.com/index.php?title=Test_Management#Roles" title="http://wiki.servicenow.com/index.php?title=Test_Management#Roles">roles</a> associated with the Test Management application: test manager and tester. Give the test manager role to users to need control of process. Give the tester role to users that just need to perform tests and record results.</li><li>You must set the status of a test cases manually. The status of individual tests in a test case does not change or update test case status automatically.</li><li>If you use <a href="http://wiki.servicenow.com/index.php?title=Visual_Task_Boards" title="http://wiki.servicenow.com/index.php?title=Visual_Task_Boards">Visual Task Boards</a>, you be pleased to know that a visual task board is automatically created for each test plan. Any tester assigned to a test case included in the test plan is added to the visual task board.<br/><img   alt="test_management3.jpg" class="image-1 jive-image" src="d5344cc2db58db048c8ef4621f961992.iix" style="height: 250px; width: 620px;"/></li></ul><p>ServiceNow offers more information about Test Management in the product documentation:</p><ul><li><a href="http://wiki.servicenow.com/index.php?title=Test_Management" title="http://wiki.servicenow.com/index.php?title=Test_Management">Test Management</a></li><li><a href="http://wiki.servicenow.com/index.php?title=Test_Manager_Tasks" title="http://wiki.servicenow.com/index.php?title=Test_Manager_Tasks">Test Manager Tasks</a></li><li><a href="http://wiki.servicenow.com/index.php?title=Tester_Tasks" title="http://wiki.servicenow.com/index.php?title=Tester_Tasks">Tester Tasks</a></li></ul>
