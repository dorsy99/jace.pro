<p>The ability to describe sets of data in <a title="w.json.org/" href="http://www.json.org/">JSON</a> format is a natural extension to the JavaScript language. ServiceNow supports a web service interface that operates on the JSON object as the data input and output format. It is a web service provided by a platform-level processor similar to the services for<a title="ki.servicenow.com/index.php?title=Web_Services" href="http://wiki.servicenow.com/index.php?title=Web_Services"> SOAP, WSDL, CSV, Excel, and XML. </a>Like those services, the JSON web service is triggered by a standalone <a title="ki.servicenow.com/index.php?title=JSONv2_Web_Service" href="http://wiki.servicenow.com/index.php?title=JSONv2_Web_Service">JSONv2</a> URL parameter.</p><p></p><p></p><p><strong>For example:</strong></p><p><a title="" _jive_internal="true" href="/" rel="nofollow" target="_blank">https://</a><span>&lt;instance name&gt;.service-now.com/mytable.do?JSONv2</span></p><p></p><p></p><p>Having the JSON object available as a data format for web services means that you can create (insert), update, and query any data in the ServiceNow platform using the JSON object format, and get results in the JSON object format. The ability to do so is available in all instances starting with the <a href="http://wiki.servicenow.com/index.php?title=Dublin_Release_Notes" style="font-size: 10pt; line-height: 1.5em;" title="Dublin Release Notes">Dublin release</a>. <span style="font-size: 10pt; line-height: 1.5em;">When executing a JSONv2 request to retrieve data from a table, by default it will return all the fields associated with the record. </span></p><p></p><h1>Limiting fields that you want to see using a JSONv2 call</h1><p></p><p>Sometimes there are many fields that we do not care about and do not want to be a part of the data retrieved. You can avoid including unwanted data by using the Exclude Columns function and alternate it by using a view instead. This way you can limit fields that you want to see when making a JSONv2 call.</p><p><span style="font-size: 10pt; line-height: 1.5em;"><br/></span></p><h2><span style="font-size: 10pt; line-height: 1.5em;">Example of limiting fields to view using JSONv2:</span></h2><p>When we do the following we see all the fields associated to this record on the Incident table.</p><p><span><span><br/></span></span></p><p><span><span><strong>Example URL:</strong> </span></span></p><p><span><a title="k-external-small" class="jive-link-external-small" href="http://community.servicenow.com/" rel="nofollow" target="_blank">http://</a></span>myinstance.service-now.com/incident.do?JSONv2&amp;sysparm_sys_id=46edaa6aa9fe198101b9d14ced16619f<a href="https://demonightlyfuji.service-now.com/incident.do?JSONv2&amp;sysparm_sys_id=46edaa6aa9fe198101b9d14ced16619f" title="https://demonightlyfuji.service-now.com/incident.do?JSONv2&amp;sysparm_sys_id=46edaa6aa9fe198101b9d14ced16619f"> </a></p><p></p><p>See example below,</p><p><img   alt="Allrows.jpg" class="image-1 jive-image" src="77f7c0cadb9c5fc068c1fb651f96198a.iix" style="height: 290px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p>In the case that we only want to see two field values "Number" and "Short Description" of an Incident, we can <a title="ki.servicenow.com/index.php?title=Database_Views%23Task_1:_Create_a_Database_View" href="http://wiki.servicenow.com/index.php?title=Database_Views%23Task_1:_Create_a_Database_View">create a view using a Database View.</a></p><p></p><h2>To create a Database View:</h2><p></p><ol><li>Navigate to System Definition &gt; Database Views.</li><li>Click New.</li><li>The Database View form appears.</li><li>Name the view as you would name a new table.</li><li>The Label and Plural fields define how the database view is labeled in lists and forms.</li></ol><p><img   alt="CreateDBView.jpg" class="jive-image image-2" src="acd563f9db1493049c9ffb651f961953.iix" style="height: 257px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>Once you have created the database view, click on <strong>New</strong> from View tables</p><p><img   alt="ViewTableCreate.jpg" class="jive-image image-3" src="bb7c8186dbd41b04ed6af3231f9619cd.iix" style="height: 218px; This time we need to execute the query using the view   name "u_blog" instead of the base table name</p><p></p><p><span><span><strong>Example URL: </strong></span></span></p><p><span><strong><a title="k-external-small" class="jive-link-external-small" href="http://community.servicenow.com/" rel="nofollow" target="_blank">http://</a></strong></span>myinstance.service-now.com/u_blog.do?JSONv2&amp;sysparm_sys_id=bcf0c1972b3631006c59ae6219da15a0</p><p><span style="font-size: 10pt; line-height: 1.5em;"><br/></span></p><p><span style="font-size: 10pt; line-height: 1.5em;">This should only return the two values as needed.</span></p><p><img   alt="RestClientTwoReturned.jpg" class="jive-image image-5" src="f1978402dbd81304b322f4621f9619d0.iix" style="height: 204px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p><span style="font-size: 10pt; line-height: 1.5em;">From this post you have now learned how to limit fiends to a view using JSONv2, create a database view with the short description and the number available, and execute the query.
