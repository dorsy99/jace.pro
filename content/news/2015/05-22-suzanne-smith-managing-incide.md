<p>Incident priority is a critical part of resolving incidents. Without a framework for establishing priority, it is difficult to meet service level agreements and trigger incident escalations appropriately. If priority becomes a subjective process of personal interpretation, incidents cannot be resolved in an efficient and logical way.</p><p></p><p><span style="font-size: 20px; font-weight: bold;">Incident priority in ServiceNow:</span></p><p></p><p>The ServiceNow Incident module supports three factors for determining incident priority:</p><ul><li><strong>Impact:</strong> Business loss and potential damage (for example, financial, customer, regulation, security, reputation, brand) caused by the incident</li><li><strong>Urgency:</strong> Speed at which the business expects the incident to be resolved</li><li><strong>Priority:</strong> sequence in which the incident should be resolved</li></ul><p><br/>In the ServiceNow base system, incident priority is determined by impact and urgency based on the following <a title="ki.servicenow.com/index.php?title=Data_Lookup_and_Record_Matching_Support" href="http://wiki.servicenow.com/index.php?title=Data_Lookup_and_Record_Matching_Support">data lookup rules</a>:</p><p></p><table border="1" class="jiveBorder" height="294" style="border: 1px solid #000000; On the incident form, the <strong>Priority</strong> field is read-only and automatically set based on the values in the <strong>Impact</strong> and <strong>Urgency</strong> fields. That said, h</span><span style="font-size: 10pt; line-height: 1.5em;">ere's where things get interesting because administrators can: </span></p><ul><li><span style="font-size: 10pt;">edit the priority lookup rules</span></li><li><span style="font-size: 10pt;">disable the <span style="color: #333333; font-family: Omnes-pro, Arial, Verdana, sans-serif;"><strong>Priority is managed by Data Lookup - set as read-only</strong></span><span style="color: #333333; font-family: Omnes-pro, Arial, Verdana, sans-serif;"> UI policy and create their own business logic</span></span></li></ul><p></p><p></p><p><span style="font-size: 20px; font-weight: bold; line-height: 1.5em;">Editing priority lookup rules:</span></p><p></p><p>The priority lookup rules are stored in a data lookup table. The table is comprised of matcher and setter fields. The data lookup queries for values equaling the matcher fields. When the data lookup finds a match, it returns the setter field value. So, in our priority data lookup, if the query finds an <strong>Impact</strong> of <strong>Medium</strong> with an <strong>Urgency</strong> of <strong>Low</strong>, it returns a <strong>Priority</strong> of <strong>Low</strong>. You can edit the setter and matcher fields using the following process:</p><p></p><ol><li>Navigate to <strong>System Policy &gt; Rules &gt; Priority Lookup Rules</strong>.</li><li>Click an order number. For example, click <strong>600</strong>.</li><li>Edit the <strong>Priority Data Lookup</strong>. For example, if you clicked <strong>600</strong> in step 2, you could change the <strong>Priority</strong> from <strong>Low</strong> to <strong>Medium</strong> for an incident with an <strong>Impact</strong> of <strong>Medium</strong> and an <strong>Urgency</strong> of <strong>Low</strong>.</li><li>Edit or delete existing information and add new information as required.<br/><br/></li></ol><p><span style="font-size: 10pt; line-height: 1.5em;"><br/></span></p><h2>Disabling priority UI policy<span style="line-height: 1.5em;">:</span></h2><p></p><p>UI policies can be used to dynamically change information on a form or define custom process flows. In the base system, there is an active incident priority UI policy. First, you need to disable that UI policy. Then, you can create your own business logic.</p><ol><li>Navigate to <strong>System UI &gt; UI Policies</strong>.</li><li>Filter the list to find the <span style="color: #333333; font-family: Omnes-pro, Arial, Verdana, sans-serif;"><strong>Priority is managed by Data Lookup - set as read-only</strong></span> UI policy<span style="color: #333333; font-family: Omnes-pro, Arial, Verdana, sans-serif;"><strong>.</strong></span></li><li>Clear the <strong>Active</strong> option.<br/><img   alt="incident_priority.jpg" class="image-0 jive-image" src="2617fff5dbdc9b048c8ef4621f9619e1.iix" style="height: 357px; width: 620px;"/><br/><br/></li></ol><p>Now you have lots of possibilities.
