<p>Are you using Internet Explorer 11 when viewing a <a title="ki.servicenow.com/index.php?title=Content_Management_Administration" href="http://wiki.servicenow.com/index.php?title=Content_Management_Administration">CMS</a> site?   Does the site utilizes CMS pages with i<a title="ki.servicenow.com/index.php?title=Using_Content_Blocks#iFrames" href="http://wiki.servicenow.com/index.php?title=Using_Content_Blocks#iFrames">Frames</a>?   If so, you may have noticed two vertical scrollbars (one for the iFrame and one for the browser) are <a title="i.service-now.com/kb_view.do?sysparm_article=KB0552071" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0552071">overlaid</a> on top of each other.   This doesn't happen with Chrome or Firefox.</p><p></p><p>As of Fuji, Chrome (or Firefox) displays two scrollbars on a page with an iFrame like this:</p><p><span class="s1"><img   alt="overlap cms iframe.jpg" class="image-0 jive-image" src="fcee9586db10df048c8ef4621f9619b3.iix" style="height: 188px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></span></p><p></p><p></p><p>As a result and a secondary issue, IE11 displays the two scrollbars on top of each other and overlaps the page components:<br/><img   alt="scrollbar render browser.jpg" class="image-1 jive-image" src="4fca78cedb1497041dcaf3231f9619f1.iix" style="height: 217px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>In an effort to identify a workaround for the overlapped scrollbars in Internet Explorer 11, the<strong> -ms-overflow-style property</strong> can be used to define the overflow behavior, as mentioned in <a title="dn.microsoft.com/en-us/library/ie/hh771902(v=vs.85).aspx" href="http://msdn.microsoft.com/en-us/library/ie/hh771902(v=vs.85).aspx">MSDN.</a></p><p></p><h2>Set the style to scrollbar to get the scrollbars back:</h2><p><code style="padding: 1px 5px; font-size: 13px; font-family: Consolas, Menlo, Monaco, 'Lucida Console', 'Liberation Mono', 'DejaVu Sans Mono', 'Bitstream Vera Sans Mono', 'Courier New', monospace, sans-serif; background-color: #eeeeee;">-ms-overflow-style: scrollbar;</code></p><blockquote class="jive-quote" style="margin: 0 0 10px; padding-left: 10px; border-left-color: #ffeb8e; font-size: 15px; color: #222222; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; background-color: #fff9e3;"><p style="margin-bottom: 1em;"><strong>scrollbar</strong></p><p>Indicates the element displays a classic scrollbar-type control when its content overflows. Unlike -ms-autohiding-scrollbar, scrollbars on elements with the -ms-overflow-style property set to scrollbar always appear on the screen and do not fade out when the element is inactive. Scrollbars do not overlay content, and therefore take up extra layout space along the edges of the element where they appear.</p></blockquote><p></p><p>Continuing with the out-of-box example, you can add this to the base Style Sheet associated to the Site's Default theme: body{-ms-overflow-style: scrollbar !important;}</p><p></p><p>This allows Internet Explorer to display the scrollbars correctly and the rendering is consistent with Chrome and Firefox:</p><p><img   alt="Consistent scroll bar render.jpg" class="image-2 jive-image" src="2ccb10cadb5c9704ed6af3231f9619fb.iix" style="height: 328px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><table border="1"><tbody><tr><td><p>PRB582664 is open to document the two scrollbars displaying with Fuji (this did not occur prior to this version).</p><p>PRB637950 is open to document the scrollbar rendering inconsistency in IE11.</p><p><a href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0552071" title="https://hi.service-now.com/kb_view.do?sysparm_article=KB0552071">ServiceNow KB: PRB648291: Vertical scroll bars are overlaid and overlapped on incident form in CMS on IE (KB0552071)</a></p><p>*For more IE solutions see <a title="" _jive_internal="true" href="/community/service-automation-platform/user-interface/blog/2015/08/07/think-like-a-servicenow-employee-internet-explorer-solutions">The top 5 Internet Explorer(IE) ServiceNow Solutions</a></p></td></tr></tbody></table>
