<p>I am half way through month 4 of my career at ServiceNow and continuously making progress on my understanding of the platform. Having spent most of career building custom websites in text editors and other offline tools, it was an initial adjustment writing code in the browser. In fact, for the level of customization that my team is doing, I desperately needed to find a way to go back to my roots and building using popular web dev tools.</p><p></p><p>My development needs:</p><ul><li>SASS - my team is transitioning to writing all of our CSS using SASS with all the joys of mixins, variables and nesting that come with it.<ul><li>This brings up issues with collaboration. How do we share the source SASS so the CSS stays up to date?</li></ul></li><li>HTML5 + Bootstrap 3 - we do a lot of bootstrap based responsive sites and I want the ease of autocomplete</li><li>AngularJS - With Geneva's move away from Jelly and embrace of AngularjS, I need quick access to all the tools that make AngularJS development faster</li></ul><p></p><p>My initial work environment:</p><ul><li>Sublime Text 3 - my current preferred code editor, with great plugins for autocomplete for most of the languages and frameworks I am using</li><li>CodeKit - A more visual approach to build task management a la gruntjs. Codekit does compilation of my SASS, minification of my js, and quick access to bower components</li><li><a title="john.andersen" __default_attr="2572" __jive_macro_name="user" class="jive_macro jive_macro_user" data-orig-content="john.andersen" data-renderedposition="305.375_38_108_16" href="/community?id=community_user_profile&user=42219665db981fc09c9ffb651f96199c">john.andersen</a> <a title="w.john-james-andersen.com/blog/service-now/updated-sublime-text-editor-for-servicenow.html" href="http://www.john-james-andersen.com/blog/service-now/updated-sublime-text-editor-for-servicenow.html">Sublime Text plugin </a>that allows me to post changes to any page, script, style sheet to the instance via ServiceNow's REST API.</li></ul><p></p><p>This solution worked very well for my first few projects in that I was able to just grab the existing record from the instance, paste into a new Sublime file and rock out. One of my favorite feature's of John's plugin work is the conflict detection. If someone changes a file you are working on, you are alerted to it as soon as to try and save changes. At this time there is no diff feature, but you can choose to override those changes or figure out the conflicts on your own.</p><p></p><p>My biggest challenge though came when my team started collaborating more on the CSS. Since the CSS stored on the instance is compiled from SASS, if someone just went and changed the CSS, we'd quickly get out of sync.</p><p></p><ol><li>Person A writes SASS and uploads compiled CSS to the instance</li><li>Person B logs into the instance and modifies the CSS new styles or changes existing properties</li><li>Person A makes more changes in their SASS, recompiles the CSS and uploads it to the instance</li></ol><p></p><p>In this scenario, all the hard work that Person B did is now gone!</p><p></p><p>So how do we solve this???? <a title="ithub.com/dynamicdan/filesync" href="https://github.com/dynamicdan/filesync">Filesync</a> to the rescue!</p><p></p><p>Thanks to our good friends at Fruition Partners and the hardwork by <a title="daniel.pettet" __default_attr="24775" __jive_macro_name="user" class="jive_macro jive_macro_user" data-orig-content="daniel.pettet" data-renderedposition="676.359375_436.625_98_16" href="/community?id=community_user_profile&user=66d11621dbd81fc09c9ffb651f96195f">daniel.pettet</a> from the ServiceNow TC team in Germany, this great little Mac/Windows tools syncs local files with their respective fields on your instance.</p><p></p><p>For example, a UI page has three fields; HTML, Client Script, and Processing Script.
