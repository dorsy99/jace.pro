<p style="font-size: 13.3333330154419px;"><span style="font-size: 13.3333330154419px; line-height: 1.5em;">Last week I wrote an <a title="" _jive_internal="true" href="/community/service-automation-platform/security/blog/2015/11/12/introducing-the-new-security-incident-response-application">overview of the new Security Incident Response application</a> that is included in the Geneva release. The buzz is still going strong over this useful new application that helps you track how security breaches are managed and resolved after they are detected. In this post, I want to focus on the powerful and visual options available in the application for monitoring security incidents.</span></p><p style="font-size: 13.3333330154419px;"></p><p style="font-size: 13.3333330154419px;"><span style="font-size: 13.3333330154419px; line-height: 1.5em;">The Security Incident Response application is part of the ServiceNow Security Operations package.</span></p><p style="font-size: 13px; font-family: arial, sans-serif; color: #666666;"></p><h2>Security dashboards</h2><p style="font-size: 13.3333330154419px;">The central hub for montioring security incidents is the security dashboards. Find the dashboards by opening the Application Navigator and navigating to <strong>Security Incident &gt; Security Dashboards</strong>.</p><p style="font-size: 13.3333330154419px;"></p><h3><strong>Monitor impacted business services</strong></h3><p>Using treemap data from ServiceNow Performance Analytics, there are six categories of incidents to view:</p><ul><li>Denial of Service</li><li>Incident Count</li><li>Incident Impact</li><li>Incident Priority</li><li>Incident Rish</li><li>Incident Severity</li></ul><p></p><p>Use the <strong>Category</strong> menu to select an option.</p><p></p><p style="text-align: center;"><img   alt="security_incident_dashboard7.gif" class="image-6 jive-image" height="376" src="5c728146dbdc9304b322f4621f961956.iix" style="height: 376px; width: 678.70036101083px;" width="679"/></p><p></p><p>With some category options, you can also select an <strong>Indicator</strong>.</p><p></p><p style="text-align: center;"><img   alt="security_incident_dashboard8.gif" class="image-7 jive-image" src="a8016842db10dfc03eb27a9e0f9619f2.iix" style="height: auto;"/></p><p style="text-align: center;"></p><p>Click inside any rectangle to view details about <span style="color: #636466; font-family: sans-serif; font-size: 13px;">security incidents for that business service.</span></p><p></p><h3><strong>Monitor trends in security incidents</strong></h3><p>A key piece of information about security incidents is trends. Is the total number going up or down? What configuration items or configuration item classes are most often affected by security breaches? Are our security incidents high priority or low priority? How many of our security incidents are access issues such as stolen passwords or unauthorized login attempts? All of these questions and more can be answered in the nine operations gauges. The gauges are particularly useful when communicating security information to management and other departments. A handy option enables you to save the individual gauges as a PNG or JPEG file that can, for example, be sent in an email, added to a document file, or included in a presentation.</p><p></p><p><img   alt="security_incident_dashboard2.png" class="image-2 jive-image" src="8ba361cedbdc1b04ed6af3231f96196a.iix" style="width: 620px; height: 484px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h3><strong>View analytic reports on security incidents</strong></h3><p>The analytic reports provide another look at security incidents. Using the tabs at the top, filter information by incident state (Open, New, or Closed) or when they were logged (today, up to 7 days ago, or up to 28 days ago). In the example, information such as total number and average age of Open incidents shown.</p><p style="text-align: center;"><img   alt="security_incident_dashboard3.png" class="image-3 jive-image" src="b6f18546db50130468c1fb651f96194e.iix" style="width: 620px; height: 461px;"/></p><p>Point to a specific location on a chart for a tooltip that can be clicked for detailed scorecard information:</p><p style="text-align: center;"><img   alt="security_incident_dashboard4.png" class="image-4 jive-image" src="954f500edb9c9fc03eb27a9e0f961954.iix" style="height: auto;"/></p><p>Just like the operations gauges explained earlier, you can save the analytics information as a PNG or JPEG file.</p><p></p><h3><strong>View all priority 1 and 2 security incidents in real time</strong></h3><p style="font-size: 13.3333330154419px;"><span style="color: #636466; font-family: sans-serif; font-size: 13px;">For a focused look at priority 1 and priority 2 incidents, view a treemap of P1 and P2 incidents logged in the past 24 hours.</span></p><p style="font-size: 13.3333330154419px;"></p><h2>BSM map</h2><p style="font-size: 13.3333330154419px; font-family: arial, sans-serif; color: #666666;"><span style="color: #636466; font-family: sans-serif; font-size: 13px;">The Business Service Management (BSM) map is a great way to see the dependencies of a configuration item involved in a security incident. Seeing these relationships can help you identify other configuration items that may be affected by the security incident. To view the BSM map directly from a security incident record form, check that the <strong>Affected CI</strong> field is filled in and then click the Show CI map icon to the right of that field. You can use the controls and filters in the upper right corner of the BSM map to view the map in different layouts.</span></p><p style="font-size: 13.3333330154419px; font-family: arial, sans-serif; color: #666666; text-align: center;"><span style="color: #636466; font-family: sans-serif; font-size: 13px;"><img   alt="security_incident_dashboard5.png" class="image-5 jive-image" src="c18469cadb90df048c8ef4621f9619e9.iix" style="width: 620px; height: 406px;"/><br/></span></p><p></p><p style="font-size: 13.3333330154419px; font-family: arial, sans-serif; color: #666666; text-align: center;"></p><h2>Additional information:</h2><ul><li>For an overview of the Security Incident Response application, see <a title="" _jive_internal="true" href="/community/service-automation-platform/security/blog/2015/11/12/introducing-the-new-security-incident-response-application">Introducing the new Security Incident Response application</a>.</li><li>The Security Incident Response application cross-references the <a title="w.nist.gov/" href="http://www.nist.gov/">National Institute of Standards and Technology</a> (NIST) database.</li><li>The Security Incident Response application integrates with several third-party software programs including Splunk.</li><li>You can launch a bridge call directly from a security incident.</li><li>To hear about the features in the Security Incident Response application from two senior ServiceNow developers that created the application, listen to <a title="" _jive_internal="true" href="/community/service-automation-platform/security/blog/2015/11/10/techbytes-episode-8-servicenow-security-incident-response">Episode 8 of Techbytes</a>, the ServiceNow podcast.</li></ul>
