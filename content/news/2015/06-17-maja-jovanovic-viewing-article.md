<p><a href="http://wiki.servicenow.com/index.php?title=Knowledge_Management" title="Knowledge Management">Knowledge management</a> has significantly changed in Fuji as we have gone from v2 to <a title="ki.servicenow.com/index.php?title=Migrating_to_Knowledge_Management_v3#gsc.tab=0" href="http://wiki.servicenow.com/index.php?title=Migrating_to_Knowledge_Management_v3#gsc.tab=0">Knowledge v3</a>. Some new features <a title="" _jive_internal="true" href="/community/service-automation-platform/knowledge-management/blog/2015/05/04/knowledge-v3-is-here">added in v3 </a>include multiple knowledge bases, the ability to pin articles, and feature popular and most helpful content right on the front page. With all of the new UI features of v3 there have also been some changes made in the security functionality.<span style="font-size: 10pt; line-height: 1.5em;"> If your <a title="ki.servicenow.com/index.php?title=Service_Catalog_Access_Controls" href="http://wiki.servicenow.com/index.php?title=Service_Catalog_Access_Controls">user criteria</a> is not configured properly in v3, you will find that non-admin users will not be able to see articles under specific topics and categories resulting in a "No items" found message.</span></p><p></p><p><a title="ki.servicenow.com/index.php?title=Migrating_to_Knowledge_Management_v3#Security" href="http://wiki.servicenow.com/index.php?title=Migrating_to_Knowledge_Management_v3#Security">Security in v3 allows access to articles managed with user criteria</a> rather than roles and ACLs. New ACLs are introduced and existing ACLs on the Knowledge [kb_knowledge] table are maintained as well as customized ones. If you do not have high security enabled, you may notice differences in default behavior.</p><p></p><p></p><p>The v3 OOB Fuji read ACL on kb_knowledge table:</p><p><img   alt="oob acl.jpg" class="image-6 jive-image" src="623f95c2db1c130468c1fb651f961945.iix" style="height: 225px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>The legacy OOB read ACL on kb_knowledge table:</p><p><img   alt="old acl.jpg" class="image-5 jive-image" src="ad8dfbb5dbd4d3049c9ffb651f961904.iix" style="height: 226px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h2>Enable non admin users to view articles:</h2><p>If your Fuji instance doesn't have the <a title="ki.servicenow.com/index.php?title=High_Security_Settings" href="http://wiki.servicenow.com/index.php?title=High_Security_Settings">High Security Plugin</a> activated, the legacy read ACL on kb_knowledge from above will be missing resulting in admin users being the only ones able to view articles under topics.</p><p></p><p>Here are one of ways to reproduce and to see if it affects you:</p><ol><li>Log into your instance as <strong>itil (non admin)</strong> user</li><li>Go to the old homepage (kb_home) your_instance_name.service-now.com/kb_home.do</li><li>Under <strong>Categories/Topics</strong> you will see "No Items" instead of actual articles listed</li></ol><p><img   alt="no items v3.jpg" class="jive-image image-4" src="bddd1ccadb5c9fc068c1fb651f961952.iix" style="height: 173px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p><strong>You will need either to import the old OOB read ACL for kb_knowledge (attached here in this article) or you need to add it as admin:</strong></p><p></p><p>Type: record</p><p>Operation: read</p><p>Name: kb_knowledge</p><p>Script: gs.hasRole("knowledge") || gs.hasRole(current.roles) || current.roles == "public" || current.roles == "";</p><p>Condition and Required roles should be empty.</p><p></p><p style="text-align: center;"><em>For more information on this issues see </em><a href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0549970" style="font-size: 10pt; line-height: 1.5em;" title="https://hi.service-now.com/kb_view.do?sysparm_article=KB0549970"><em>ServiceNow KB: Upgrading to Fuji without high security prevents access to knowledge articles (KB0549970)</em></a></p><p></p><h2>Add the glide.ui.escape_text system property:</h2><p>Another issue related to High Security Plugin being inactive on Fuji causing users not to see articles in the Knowledge Base, could be due to the glide.ui.escape_text being set to false. When the property "glide.ui.escape_text" is set to false, the kb_view displays message, "The content of elements must consist of well-formed character data or markup."</p><p><span style="font-size: 10pt; line-height: 1.5em;"> </span></p><p><span style="font-size: 10pt; line-height: 1.5em;">The s</span><span style="font-size: 10pt; line-height: 1.5em;">ystem property glide.ui.escape_text is missing due to plugin not being active. Adding this property in the system properties and setting it to true will resolve the issue. The 'glide.ui.escape_text' system property will escape XML values at the parser level for the user interface. This will prevent cross site scripting attacks. </span></p><p></p><p></p><p style="text-align: center;"><em>For more information see <a href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0549476" title="https://hi.service-now.com/kb_view.do?sysparm_article=KB0549476">ServiceNow KB: PRB629756: Knowledge v3 search and search results do not render correctly when glide.ui.escape_text is faâ€¦</a> (This issue is fixed in Geneva)</em></p><p></p><p>Before considering to activate <a href="http://wiki.servicenow.com/index.php?title=High_Security_Settings" title="High Security Settings">high security</a> you need to read over the basics to get a full understanding of what it enables and possible risks.</p>
