<p>Workflows in ServiceNow automate processes and contain a sequence of activities. Common workflows include routine changes, incident resolution, and notifying users of pending approvals. Here is an example of a routine change workflow:</p><p style="font-size: 13.3333330154419px;"><span style="font-size: 13.3333330154419px; line-height: 1.5em;"><img   alt="RoutineChange.png" class="image-0 jive-image" height="368" src="42f560cedb905f048c8ef4621f961938.iix" style="height: 368px; width: 683.11377245509px; display: block; margin-left: auto; margin-right: auto;" width="683"/></span></p><p>You can use a workflow field to display workflow progress. There are differences in how workflow fields are displayed in a list and in a form:</p><ul><li><strong>List:</strong> workflow field displays stage icons that represent the series of states, stages, phases, or tasks within a workflow.</li><li><strong>Form:</strong> workflow field displays the current stage as a choice list value.</li></ul><p></p><p>Workflow stage renderers determine how a workflow displays stages in a workflow field. There are multiple stage renderers available. Selecting the most appropriate renderer can help you avoid annoying issues with your workflows.</p><p></p><p><strong>Note:</strong> The most important information to know about workflow stage renderers is that in the vast majority of cases <em>a workflow should use the workflow-driven renderer</em>. This is the default option. Use a different stage renderer only after careful consideration and to satisfy specific requirements for how the stages appear.</p><p></p><h2>Available stage renderers</h2><p></p><p><strong>Workflow-driven</strong></p><p>Use the workflow-driven renderer as often as possible. This renderer displays icons for stages using the stage state controlled by the workflow. Icons are displayed in a way that is meaningful for many situations. This renderer can display stages from a main workflow and subflows. The order of the icons is determined by the expected path of the executing workflow. As the workflow progresses, stages on paths that the workflow did not take are removed from the display.</p><p></p><p><strong>Main flow</strong></p><p>The main flow renderer displays icons for stages defined in the main workflow only. Use this renderer when you do not want to expose the details of the subflows. For example, a single main workflow may run several subflows. If users do not need to see the subflow steps, using the main flow renderer leads to the best user experience.</p><p></p><p><strong>Linear</strong></p><p>Linear rendering displays all stage icons from the main workflow and all subflows within a single workflow field. It displays icons in a linear sequence regardless of the paths the workflow follows as it executes. This renderer uses stages defined in both the main workflow and any subflows that the main flow launches. The icons appear in the user-specified order. Skipped stages do not appear. <span style="font-size: 10pt; line-height: 1.5em;">Use this renderer when the workflow stages must display in a consistent order and the actual details of how the workflow runs are less important.</span></p><p></p><p><em>Linear main flow</em></p><p><img   alt="600pxLinearMainflow.png" class="image-1 jive-image" src="a841994edbd413043eb27a9e0f9619ae.iix" style="height: 211px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/></p><p><span style="font-size: 10pt; line-height: 1.5em;"><em>Linear sub-flow</em></span></p><p><img   alt="600pxLinearSubflow.png" class="jive-image image-2" src="e1f1388adb189304b322f4621f96196f.iix" style="height: 165px; width: 620px; display: block; margin-left: auto; margin-right: auto;"/><span style="font-size: 10pt; line-height: 1.5em;"><strong>Progress bar</strong></span></p><p>The progress bar renderer displays a single progress bar instead of a sequence of stage icons. This renderer is useful when you want to display the general progress of the workflow as a percent. Each stage in the workflow is displayed as an equal percentage of the progress bar. Stage names do not appear in the progress bar.<span style="font-size: 10pt; line-height: 1.5em;"> </span></p><p><img   alt="WfFieldProgressRenderer.png" class="image-0 jive-image" src="5fecadcadb9cd304b322f4621f9619e7.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p><strong>Legacy</strong></p><p>The legacy renderer displays stages in the same way as releases prior to Dublin did. This renderer sometimes produces unexpected behavior. When an instance is upgraded from a release prior to Dublin, all existing workflows are set to use the legacy renderer. Whenever possible, use a different stage renderer instead. If your workflow needs to maintain pre-Dublin behavior, then it is appropriate to use the legacy option. Â  <span style="font-size: 10pt; line-height: 1.5em;">With the legacy renderer, you can create and reference a custom workflow field icon set by setting the icons attribute to a new script include. </span></p><p></p><p></p><h2>Selecting a stage renderer</h2><ol><li>Navigate to <strong>Workflow &gt; Workflow Editor</strong>.</li><li>Open and check out a workflow.</li><li>In the title bar, click the menu icon <img   alt="hamburger_icon.png" class="image-1 jive-image" src="951ac1c6db90d304b322f4621f9619d0.iix" style="height: auto;"/>and select <strong>Properties</strong>.</li><li>In the <strong>Workflow Properties</strong> dialog box, click the <strong>Stages</strong> tab.</li><li>From the <strong>Stage rendering</strong> list, select a stage renderer.<br/><img   alt="workflow_stage_renderer_select.png" class="image-0 jive-image" src="fa59273ddbd41fc068c1fb651f961929.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></li><li>Click <strong>Update</strong>.</li></ol><p></p><h2>Tips:</h2><ul><li>If you are using two workflows to update two unique workflow fields on a single record, both workflows must use a non-legacy renderer. You can select a different stage renderer for each workflow, but do not select Legacy for either one. If you want to use the Linear renderer, make sure you select Linear in the properties for the parent workflow and all subflows.</li><li>In the Workflow-driven, Main flow, and Progress bar renderers, the <strong>Stage order</strong> field on the <strong>Stages</strong> tab has two options:<ul><li><strong>Computed</strong> uses the actual workflow path in order</li><li><strong>User-specified</strong> uses the order specified in the Order column on the Workflow Stages record<br/>&gt;</li></ul></li><li>If you are using the Progress bar stage renderer, you can simplify stage names to represent a percentage of the workflow. For example, if a process has four main steps, name the workflow stages as 25%, 50%, 75%, and 100%. Enter the numerical value of each stage, such as 25, in the <strong>Value</strong> and <strong>Order</strong> fields.</li><li>If you are using the Linear renderer, the parent flow and all subflows must contain the same stages. Use a workflow <a title="ki.servicenow.com/index.php?title=Using_Workflow_Stage_Sets" href="http://wiki.servicenow.com/index.php?title=Using_Workflow_Stage_Sets">stage set</a> to ensure the parent flow and all subflows have the same stages.</li></ul><h2></h2><h2>Additional information:</h2><ul><li><span style="font-size: 10pt;
