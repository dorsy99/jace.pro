<p>ServiceNow workflows automate multi-step processes consisting of multiple activities linked with transitions. For example, you can create workflows for emergency change requests, service catalog requests, and knowledge base article approvals. Depending on your instance configuration, there are more than 23 activities to choose from when designing a workflow.</p><p></p><p>Some of the most popular and widely used activities are the four <a title="ki.servicenow.com/index.php?title=Condition_Activities#gsc.tab=0" href="http://wiki.servicenow.com/index.php?title=Condition_Activities#gsc.tab=0">condition activities</a>:</p><ul><li>If</li><li>Switch</li><li>Wait for condition</li><li>Wait for WF event</li></ul><p>Here is a description and tips for each of the condition activities.</p><p></p><h2>"If" Condition Activity</h2><h2><span style="font-size: 10pt; line-height: 1.5em; font-weight: normal;">Add the If activity to check a condition or script and determine if a Yes or No transition should occur.<br/></span></h2><p></p><p><span style="font-size: 10pt; line-height: 1.5em; font-weight: normal;">For example, in the Employee Change HR workflow below, an If activity is added at the beginning to check if approval is required. If it is, the next activity sends an approval request to a group. If approval is not required, the next activity runs a script that updates the employee's profile. </span></p><p></p><p><span style="font-size: 10pt; line-height: 1.5em; font-weight: normal;"><img   alt="workflow_condition_activities_if.jpg" class="image-5 jive-image" src="de9b010edb5413043eb27a9e0f961947.iix" style="width: 620px; height: 349px; display: block; margin-left: auto; margin-right: auto;"/><br/></span></p><p><span style="font-size: 10pt; line-height: 1.5em; font-weight: normal;"><span style="color: #636466; font-family: sans-serif; font-size: 13px;">If the </span><span class="ph uicontrol" style="font-weight: bold; color: #636466; font-family: sans-serif; font-size: 13px;">Condition</span><span style="color: #636466; font-family: sans-serif; font-size: 13px;"> and the </span><span class="ph uicontrol" style="font-weight: bold; color: #636466; font-family: sans-serif; font-size: 13px;">Advanced</span><span style="color: #636466; font-family: sans-serif; font-size: 13px;"> script are both specified, then both must evaluate successfully for the </span><span class="ph uicontrol" style="font-weight: bold; color: #636466; font-family: sans-serif; font-size: 13px;">Yes</span><span style="color: #636466; font-family: sans-serif; font-size: 13px;"> transition to occur.</span></span></p><p></p><p><span style="color: #636466; font-weight: normal; font-size: 13px; font-family: sans-serif; line-height: 1.5em;">For some advanced tricks with the If activity, see Steve Bell's ( <a title="sabell2012" __default_attr="7849" __jive_macro_name="user" class="jive_macro jive_macro_user" data-orig-content="sabell2012" data-renderedposition="-146.8607940673828_87.06348419189453_83_15" href="/community?id=community_user_profile&user=838f86e9db181fc09c9ffb651f9619d9">sabell2012</a>) post titled <a title="" _jive_internal="true" href="/groups/servicenow-user-group-us-tx-north-texas/blog/2015/10/18/mini-lab-workflows--playing-with-the-if-activity">MIni-Lab: Workflows - Playing With the If Activity</a>. Steve generously shares his knowledge and walks you through the creation of a workflow containing an interesting If activity. </span></p><p></p><h2>"Switch" Condition Activity</h2><p>Add the Switch activity to check if the passed field or variable value is equal to one of several case values.</p><p></p><p>For example, in the Contract Approval workflow below, there are two switch activities. Each of the switch activities asks the question, "What are we processing?" Based on the answer, <strong>None</strong>, <strong>Renewal</strong>, or <strong>Expired</strong>, the workflow continues and a set values or run script activity changes the state.</p><p></p><p><img   alt="workflow_condition_activities_switch.jpg" class="image-6 jive-image" src="36d9c902dbd0d344e9737a9e0f9619b0.iix" style="width: 620px; height: 301px; display: block; margin-left: auto; margin-right: auto;"/></p><p><span style="color: #636466; font-family: sans-serif; font-size: 13px;">Customers sometimes have questions about whether to use an If activity or a Switch Activity. Use an If activity when the answer to the question is Yes or No and you are checking a single condition. Very simple. If there are different or more options than just Yes or No and you are checking multiple conditions at the same time, a Switch activity gives you more control. </span></p><p></p><h2>"Wait for" Condition Activity</h2><p>Add the Wait for condition activity to have the workflow pause indefinitely until the current record matches a specified condition. When the current record is updated, the workflow evaluates the Wait for condition activity.</p><p></p><p>For example, in the Employee Offboarding workflow below (only part of the workflow is shown), there are two wait for condition activities. One of the activities always causes the workflow to stop until the employee's end date is reached. Another wait for condition activity always checks if all sub-tasks are inactive before moving on to the next activity.</p><p></p><p><img   alt="workflow_condition_activities_waitforcondition.jpg" class="image-7 jive-image" src="7de06402db10dfc03eb27a9e0f961981.iix" style="width: 620px; height: 424px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>There is an important difference between the Wait for condition activity and the Timer activity. If you need the workflow to pause for a specific, timed duration, use the Timer activity instead of the Wait for condition activity. The Timer activity gives you greater control over how long the workflow pauses and what causes it to start again after pausing.</p><p></p><p>The Wait for condition activity can be handy if you have multiple tasks that you want completed before the workflow ends. You can configure the Wait for condition so the workflow pauses until ALL tasks are complete.</p><p></p><p></p><h2>"Wait for WF event" Condition Activity</h2><h2><span style="font-size: 10pt; font-weight: normal;">Add the Wait for WF Event activity to have the workflow pause until a specified event is fired.</span></h2><p></p><p><span style="font-size: 10pt; font-weight: normal;">For example, in the <span style="font-size: 13.3333330154419px;">Item Designer Fulfillment</span> workflow below, there is one Wait for WF Event activity following an If activity.   The workflow waits for the   sc_ic_req_task_complete event and then runs a script in the next activity.         </span></p><p></p><p><span style="font-size: 10pt; font-weight: normal;"><img   alt="workflow_condition_activities_waitforevent.jpg" class="image-8 jive-image" src="ac50cccadb185fc03eb27a9e0f9619a3.iix" style="width: 620px; height: 330px; display: block; margin-left: auto; margin-right: auto;"/><br/></span></p><p></p><p><span style="font-size: 10pt; font-weight: normal;">For an excellent explanation of the Wait for WF Event activity and several detailed examples, see Russ Sarbora's blog post <a title="Waiting for [the workflow event named] Godot" __default_attr="4405" __jive_macro_name="blogpost" class="jive_macro jive_macro_blogpost" data-orig-content="Waiting for [the workflow event named] Godot" data-renderedposition="2628.238525390625_7.997159004211426_266_35" href="/community?id=community_blog&sys_id=f22da6e5dbd0dbc01dcaf3231f9619ae">Waiting for [the workflow event named] Godot</a>.
