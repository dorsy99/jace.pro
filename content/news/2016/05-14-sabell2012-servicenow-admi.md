<p style="font-family: arial, sans-serif; color: #666666;"><span style="font-style: inherit; font-family: inherit;"><strong>NOTE</strong></span>: ON APRIL 1, 2016 ACCENTURE COMPLETED THE ACQUISITION PROCESS OF CLOUDSHERPAS.   AT THAT TIME THE CLOUDSHERPAS BLOG SITES WERE DOWNED FOREVER.</p><p></p><p style="font-family: arial, sans-serif; color: #666666;">THIS IS THE RE-PUBLICATION OF MY ARTICLE FROM <span style="font-style: inherit; font-family: inherit;"><strong>September 23, 2014</strong></span> ON THE CLOUDSHERPAS SERVICENOW ADMIN 101 BLOG.</p><p style="font-family: arial, sans-serif; color: #666666;">____________________________________________________________________________</p><p></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">At least once a week I am asked: Is there a way to create a single combined report of my Service Catalog Requested Item variables? And, for</span></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">that matter, where does this information reside inside of ServiceNow?</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Well the answer to the first question is yes, and the trick to how to do it is in the answer to the second question!</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">A lot of people want to see their Requested Item variables, including which variables are associated with which item by requestor, a list of</span></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">questions associated with a particular item or… well you get the idea!</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">It took a little bit to track down the solution when I first dug into it, but it really isn't anything bad. The best approach is a Database View</span></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">(/tag/database-views/) that ties all of the necessary tables together to form the report.</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">So, where does everything live? If you dig down a bit, you'll find that:</span></p><p></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">1) Service Catalog Request Items (RITM) are stored in the sc_req_item table</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">2) Variables are stored in the sc_item_option table</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">However, there does not appear to be any sort of direct relationship between the two.</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">In comes the trick:</span></p><p></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">3) RITM to sc_item_option associations are actually kept in a third table: sc_item_option_mtom</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">But how is this done? I nod sagely, and I say: "sys_id, my friend!"</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">The mtom table (by the way, does anyone have any idea what "mtom" stands for? Item—to-item maybe?) allows several Service Catalog</span></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Requested Items to be associated to the same variables. Cool, huh?!</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">When I figured this out, I smacked myself in the forehead for not getting it the first time. After that, the rest was simple.</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">The relationships:</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">sc_req_item relates to sc_item_option_mtom (via sc_req_item.sys_id)</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">sc_item_option_mtom (via sc_item_option.sys_id) relates to sc_item_option</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">For the confused, here is a diagram:</span></p><p></p><p><span style="color: #000000; font-size: 12pt; font-family: arial, helvetica, sans-serif; margin-bottom: 10pt;"><img class="jive-image" height="445" src="https://lh6.googleusercontent.com/xWmWK33U__4vbJP5iiWvUrrAnmgceiQ5Ytm7_LJEaq14KfBHuCk_YT3MvvoX4-dJSPnRpTcfE_hU4FjxOPdfmoseSko9-IARSluxOSfkXpM0Lj2eFxFB62wTBO-KkfZt7686qso" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="color: #000000; font-size: 12pt; font-family: arial, helvetica, sans-serif; margin-bottom: 10pt;">The next step is to weave these together in a Database View, which can then be accessed via a report.</span></p><p></p><p><span dir="ltr" style="font-size: 12pt; font-family: arial, helvetica, sans-serif;">1. Create a Database View:</span></p><p></p><p style="padding-left: 30px;"><span dir="ltr" style="font-size: 12pt; font-family: arial, helvetica, sans-serif;">a. From the ServiceNow navigation search type: database</span></p><p style="padding-left: 30px;"><span dir="ltr" style="font-size: 12pt; font-family: arial, helvetica, sans-serif;">b. Click on System Definition / Database Views</span></p><p><span style="font-size: 12pt; font-family: arial, helvetica, sans-serif;"><br/><span style="margin-left: 72pt; color: #000000; text-align: center;"><img alt="C:\Users\User 1\Desktop\1. Database Views.jpg" class="jive-image" height="344" src="https://lh6.googleusercontent.com/Jpi5Crk3XFzuA2Xd3jv3HuGpLfSzfgRlWZaey5IJSzOSoVUP54Io7sDVSRnbLG-pCMnXjW_e964vWGEBuDT45hYUaN0cdqKoQEFmVLwMzGjVb2agerA7gZlu_L4ftAkIYpF0Yk0-1SwgGimY" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="269"/></span><br/></span></p><p></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">c. From the Database Views list view, click on the new button to create a new Database View. Name it something meaningful so that you don't have to search far when pulling in the Database View for your report.</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">i.   <strong>Name</strong>:   SC RITM Variables (u_sc_ritm_variables)</span></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">ii. Click on the save button</span></p><p></p><p><span style="color: #000000; font-size: 12pt; font-family: arial, helvetica, sans-serif; margin-bottom: 10pt; text-align: justify;"><img alt="C:\Users\User 1\Desktop\2. Create New DB View.jpg" class="jive-image" height="336" src="https://lh5.googleusercontent.com/8V1kt0eCluVmQf2pOhzfGTjPOlEi_Xe0HkEEZFZcBjiUqqhThHMmsoPVIUfh_dM7-GD5l0NsYgGiJlnN8e4URrBxhgIXSz74PcYe_DWkubshIPrPwaBfDUOq3kwhpjAx26Fq_Tcn_CD6vqi9" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">iii. At the bottom of the form you will see the View Tables list view. Click the new button.</span></p><p style="padding-left: 60px;"></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">1. <strong>Table</strong>: Variable Ownership (sc_item_option_mtom)</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">2. <strong>Variable prefix</strong>: mtom</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">3, <strong>Order</strong>: 100</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">4. Submit</span></p><p></p><p><span style="color: #000000; font-size: 12pt; font-family: arial, helvetica, sans-serif; margin-bottom: 10pt;"><img alt="C:\Users\User 1\Desktop\3. mtom.jpg" class="jive-image" height="229" src="https://lh5.googleusercontent.com/oTUHzWiY0d6jmwBgGyXGvQij419Vj74D73dLaect7audiQdmXmPd2oUeyd-ti4J7Zk1RYmLl88UgRmRydqP9p8Ephy2ZqTee4O0KL1c_1EG_sR0J4-SVCsuvo9cwqBx9JgT8witiJkPqBHZy" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">iv. Add another table. Click the new button again.</span></p><p style="padding-left: 60px;"></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">1. <strong>Table</strong>: Options (sc_item_option)</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">2. <strong>Variable prefix</strong>: options</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">3. <strong>Order</strong>: 200</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">4. <strong>Where clause</strong>: mtom.sc_item_option = options.sys_id</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">5. Submit</span></p><p></p><p><span style="color: #000000; font-size: 12pt; font-family: arial, helvetica, sans-serif; margin-bottom: 10pt;"><img alt="C:\Users\User 1\Desktop\4. options.jpg" class="jive-image" height="227" src="https://lh6.googleusercontent.com/HJTYJ6h0v4fzXDgCPt__I3XFgG5OwbYxYy3L0hm_NfLggDozl-OCg59Cf37LHYzglbyY2dYjVXImWplmUoG92jnzdCqCuONPmI6xS7yIoAF0rtZ8oBkXmROddKWU44pcotnL_2k88EFgeCuT" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">v. Add yet another table (this is the final one, I promise). Click the new button again.</span></p><p style="padding-left: 60px;"></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">1. <strong>Table</strong>: Requested Item (sc_req_item)</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">2. <strong>Variable prefix</strong>: ritm</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">3. <strong>Order</strong>: 300</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">4. <strong>Where clause</strong>:   mtom.request_item = ritm.sys_id</span></p><p style="padding-left: 90px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">5. Submit</span></p><p></p><p><span style="font-size: 12pt; font-family: arial, helvetica, sans-serif;"><br/><span style="color: #000000;"><img alt="C:\Users\User 1\Desktop\5. ritm.jpg" class="jive-image" height="233" src="https://lh6.googleusercontent.com/mpaRXPrGob6qdPdMwZaV6MWNKboQyMUv6QaO-yFBDWi5cbMCacQE5eHiNzsC2OVNDm4bfSKBXJebWBrBsRQqXe-ZEfKbKwzmhTSxlMLKOlxhAVG4Ett4w02r9oUHrashJyBBYlb-h3GQ8P_m" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span><br/></span></p><p style="padding-left: 30px;"><span dir="ltr" style="margin-bottom: 10pt; font-size: 12pt; font-family: arial, helvetica, sans-serif;">d. Your Database View should now look like this:</span></p><p></p><p><span style="color: #000000; font-size: 12pt; font-family: arial, helvetica, sans-serif; margin-bottom: 10pt;"><img alt="C:\Users\User 1\Desktop\6. database view.jpg" class="jive-image" height="382" src="https://lh4.googleusercontent.com/53dWonhx6OEePGty9udcgTVj_GmNtYsjupribQJQfAgchvD1YmLwgPTPcJj4t8QNomdWsD4-oullU6ibW5CvkN5ACx4fh9Ebeef5LD8a_MXFQGvihzp_s3Lz7V1_0XNEDPgXEoeN0neRfFwS" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt; line-height: 1.5;">e. You are done with the Database View! To test it out click the "Try It" link under the Related Links section of the Database View form.</span></p><p></p><p><span dir="ltr" style="margin-bottom: 10pt; font-size: 12pt; font-family: arial, helvetica, sans-serif;">2. Now create a new report and use the "SC RITM Variables" (u_sc_ritm_variables) table.</span></p><p></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: arial, helvetica, sans-serif;
