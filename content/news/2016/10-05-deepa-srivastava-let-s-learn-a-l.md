<p><span style="color: #3334ca;"><span style="color: #303030;">Hi Friends ,</span> </span></p><p></p><p>It's my first blog and it may seem to be a small topic to write about but its surely an important concept in coding. I have seen quite a lot of posts (in community) and usage in Service Now .</p><p>I hope this blog helps you to understand about Regular expression <span __jive_emoticon_name="happy" __jive_macro_name="emoticon" class="jive_emote jive_macro" data-renderedposition="71_423.739990234375_16_16" src="/8.0.4.21bdc7e/images/emoticons/happy.png"></span></p><p></p><p>A <strong>regular expression</strong> is a special text/string for describing a <strong>search pattern</strong>. Use regular expressions to search through large numbers of text and binary files and then you can replace texts, rename files, copy files etc.</p><p></p><p><strong>Meaning of different character/symbols:</strong></p><p></p><ul><li>Twelve characters have special meanings in regular expressions: the backslash \, the caret ^, the dollar sign $, the period or dot ., the vertical bar or pipe symbol |, the question mark ?, the asterisk or star *, the plus sign +, the opening parenthesis (, the closing parenthesis ), the opening square bracket [, and the opening curly brace {. These special characters are often called "<strong>metacharacters</strong>". <em>Most of them will produce are errors when used alone.</em></li><li>If you want to use any of these characters as a literal in a regex, you need to escape them with a backslash. So, if you want to <strong>match 2+3=5, the correct regex is 2\+3=5</strong>.</li></ul><p></p><p>I would not mention all the concepts present in regular expression but will try to put all the important one's here.</p><p></p><ul><li>Anchors do not match any characters. They match a position. <tt class="code"><span class="regexspecial">^</span></tt> matches at the start of the string, and <tt class="code"><span class="regexspecial">$</span></tt> matches at the end of the string. <strong>Example:^l matches only first l in lol.</strong></li><li>A character class [ ] specify a range of characters, We can use more than one range inside brackets. A caret after the opening square bracket negates the character class.</li></ul><p><strong>           Example: [0-9a-z] matches only 0-9 digits and alphabets in (smaller case).</strong></p><ul><li>Use curly braces { } to specify a repetition .<strong>Example</strong>: <strong>[A-Z]{2} two characters, each of which can be between A-Z.</strong></li></ul><p></p><p><strong>Shorthand character classes</strong>:</p><p></p><ul><li>\d matches a single character that is a digit.</li><li>\w matches a "word character" (alphanumeric characters plus underscore).</li><li>\s matches a whitespace character.</li><li>The dot matches a single character. <strong>Example</strong><strong>: te.t matches text, test, tent etc.</strong></li><li><strong>|   </strong>is regular expression equivalent of "or".</li></ul><p></p><p><span style="color: #3334ca;">To test if a particular regex matches (part of) a string, you can call the string <strong>match() or test()</strong> method.</span></p><p><strong>Example</strong>:   <strong>if(regexp.test(someValue)){}</strong></p><p></p><p><strong>Note: Regex engines are case sensitive by default.</strong></p><p><br/>I found below link very useful for learning regex.
