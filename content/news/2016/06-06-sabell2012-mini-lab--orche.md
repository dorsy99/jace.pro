Personal instance with the Orchestration plug-in installed.   This will include the Active Directory activities which we will be using in another lab.</p><p>2. Install and configure a local MID Server (see the ServiceNow Wiki for how-tos)</p><p>3. Knowledge of the ServiceNow platform, Workflow Editor, Constructing Workflows, Orchestration</p><p>4. Familiarity with Microsoft Windows Server environment.   A knowledge on how to transfer files from one system to another is recommended.   The purpose of this article is not to teach Windows Server management.</p><p>5. Personal computer with enough space (RAM, Disk) to install and run VirtualBox with the Windows image loaded.</p><p>6. If you have problems with running PowerShell commands on your local PC then consider changing your execution policy: <a href="https://technet.microsoft.com/en-us/library/ee176961.aspx" title="https://technet.microsoft.com/en-us/library/ee176961.aspx">Using the Set-ExecutionPolicy Cmdlet</a> </p><p></p><p></p><p><span style="font-size: 12pt;"><strong>Lab 1.1: Installing and Configuring Oracle VirtualBox</strong></span></p><p></p><p>1. Download and install Oracle VirtualBox:</p><p style="padding-left: 30px;"><a title="w.oracle.com/us/technologies/virtualization/virtualbox/overview/index.html#ct02-Download" href="http://www.oracle.com/us/technologies/virtualization/virtualbox/overview/index.html#ct02-Download">Link to download</a></p><p style="padding-left: 30px;"><a title="w.oracle.com/technetwork/server-storage/virtualbox/documentation/index.html" href="http://www.oracle.com/technetwork/server-storage/virtualbox/documentation/index.html">Link for install instructions</a></p><p></p><p><span style="color: #1155cc; text-decoration: underline; font-size: 14.6667px; font-family: Arial;"><img   class="image-5 jive-image" height="467" src="e7a1ec0adb10dfc03eb27a9e0f961936.iix" style="width: 374px; height: 466.792px; display: block; margin-left: auto; margin-right: auto;" width="375"/></span></p><p></p><p>2. Download Windows 2008 Server 32-bit iso file.   You could opt to go with 64-bit if you wish, but 32-bit will be fine.</p><p style="padding-left: 30px;"><a title="ww.microsoft.com/en-us/download/details.aspx?id=5023" href="https://www.microsoft.com/en-us/download/details.aspx?id=5023">Link to download</a></p><p></p><p>         Side Note (these are interesting and worth investigating, but we will not be using them in the labs):</p><p>                   <a href="http://www.itworld.com/article/2981582/virtualization/microsoft-tells-developers-download-our-free-virtual-machines.html" title="http://www.itworld.com/article/2981582/virtualization/microsoft-tells-developers-download-our-free-virtual-machines.html">Microsoft tells developers: Download our free virtual machines | ITworld</a></p><p>                   <a href="https://developer.microsoft.com/en-us/microsoft-edge/tools/vms/windows/" title="https://developer.microsoft.com/en-us/microsoft-edge/tools/vms/windows/">Virtual Machine (VM), Windows Virtual PC &amp; BrowserStack</a></p><p></p><p>3. From you PC open a Command Prompt (cmd.exe) window, and do an IPConfig to find out what your current IP and gateway are.</p><p></p><p style="padding-left: 30px;"><strong>Note:</strong> I put mine in the parenthesis as an example.   Your results will likely be different.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">a. Write your local IP here:   ____________ (192.168.43.99)</p><p style="padding-left: 30px;">b. Write your Subnet Mask here: _____________ (255.255.255.0)</p><p style="padding-left: 30px;">c. Write your Default Gateway here: _____________ (192.168.43.1)</p><p></p><p><img   class="jive-image image-14" src="a275a006db14d704ed6af3231f961996.iix" style="width: auto; height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p></p><p>4. Open VirtualBox on your PC.</p><p></p><p><img   class="image-1 jive-image" src="9095e042db109304b322f4621f961904.iix" style="width: 620px; height: 493px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>5. Click the New button</p><p></p><p style="padding-left: 30px;">a. <strong>Name</strong>: Microsoft Server 2008</p><p style="padding-left: 30px;">b. <strong>Type</strong>: Microsoft Windows</p><p style="padding-left: 30px;">c. <strong>Version</strong>: Windows 2008 (32-bit)</p><p><img   class="image-2 jive-image" src="3b73cc8edb985fc03eb27a9e0f9619a1.iix" style="width: auto; height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p style="padding-left: 30px;">d. Click the Next button</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">e. <strong>Memory size</strong>: 512 Mb (default).   Click the Next button.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">f. <strong>Hard disk</strong>:   Create a virtual hard disk now.   Click the Next button.</p><p style="padding-left: 60px;">i. <strong>Hard disk file type</strong>: VDI (default). Click the Next button.</p><p style="padding-left: 60px;">ii.<strong> Storage on physical hard disk</strong>: Dynamically allocated.   Click on the Next button.</p><p style="padding-left: 60px;">iii.<strong> File location and size</strong>: 25.00 GB, take the defaults.   Click on the Create button.</p><p style="padding-left: 60px;">iv. You will be returned to the main VirtualBox console, and your new Virtual Machine should be visible.</p><p style="padding-left: 60px;"></p><p style="padding-left: 30px;">g. Under Storage, click on the [Optical Drive] empty link, and click on Choose Disk Image...</p><p style="padding-left: 60px;"></p><p><img   class="image-3 jive-image" src="a5f1e37ddb141704ed6af3231f96196d.iix" style="width: 620px; height: 495px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;">i. Navigate to where the Windows 2008 Server iso is stored</p><p style="padding-left: 60px;">ii. Choose the iso and click ok.   You should see the iso attached on SATA Port 1.</p><p></p><p></p><p><span style="font-size: 12pt;"><strong>Lab 1.2: Installing and Configuring Windows 2008 Standard Server</strong></span></p><p></p><p>6. Now click the big green Start button!   This will boot up your new server, and install from the iso image.</p><p></p><p><img   class="jive-image image-4" src="6ce4450adb141b04ed6af3231f9619a2.iix" style="width: 620px; height: 494px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p style="padding-left: 30px;">a. Install Windows: Take the defaults.   Click next.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-6 jive-image" src="5311c4c6db141344e9737a9e0f9619fb.iix" style="width: 620px; height: 520px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">b. Click the big "Install now" button.</p><p style="padding-left: 30px;">c. <strong>Type your product key for activation</strong>: Leave blank to evaluate the software (if you wish to purchase a license later).   Un-check Automatically activate Windows...</p><p style="padding-left: 30px;">d. Do you want to enter your product key now?   Click the no button if you are only evaluating.</p><p style="padding-left: 30px;">e. <strong>Windows Version</strong>: Choose Standard (Full Installation).   Then click on the "I have selected the edition of Windows that I purchased".   Again, if still an evaluation copy, then you will need to check this box to get the Next button to be clickable.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-7 jive-image" src="8510ef39db901fc03eb27a9e0f9619b9.iix" style="width: 620px; height: 523px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">f. <strong>License terms</strong>: Read the terms. If you agree, check the box and click the Next button.</p><p style="padding-left: 30px;">g. <strong>Which type of installation do you want?</strong>   Custom should be the only choice.   Click on Custom.</p><p style="padding-left: 30px;">h. <strong>Where do you want to install Windows?</strong>   There should only be the 25Gb drive.   Click Next.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="jive-image image-8" height="309" src="354c050adb5c97041dcaf3231f961930.iix" style="width: 412px; height: 309px; display: block; margin-left: auto; margin-right: auto;" width="412"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">i. At this point Windows will begin installing.   Go get some coffee as this could take a bit.   Allow the software to restart your device as needed.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px; text-align: center;"><img   class="image-9 jive-image" height="401" src="4b53277ddb141fc03eb27a9e0f96193e.iix" style="width: 477px; height: 400.834px;" width="477"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">j. When it finishes Windows will have you change the password when logging in for the first time.   Do so.   Windows will then be up and running!</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">k. Log into Windows using your Administrator credentials.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">l. In the lower-right corner click on the double monitor symbol and then click on the Network and Sharing Center link.   This will open the Network and Sharing Center form.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="jive-image image-10" src="8c484d4adb1413043eb27a9e0f961930.iix" style="width: 620px; height: 520px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">m. Click on the View Status link to bring up the Local Area Connection Status form.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-11 jive-image" src="f5752482db5457049c9ffb651f96191d.iix" style="width: 620px; height: 519px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">n. From the Local Area Status Form click on the Properties button.   This will display the Connection Properties form.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-12 jive-image" src="e2758c06db5cd3041dcaf3231f961960.iix" style="width: auto; height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">o. From the Connection Properties form click on the Internet Protocol Version 4 (TCP/IPv4) item.   Do NOT uncheck this.   Then click on the Properties button.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-13 jive-image" src="2627a335db50df04e9737a9e0f96195f.iix" style="width: 620px; height: 523px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">p. Now using the IP information you recorded back in step 3 configure your Server's IP information.   I made an IP up for my server that was below the .10, but not .1 or .0.   The Default gateway also served as my DNS server.   You have to have a fixed IP for any server being used as an Active Directory or for DNS work.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-15 jive-image" src="dcb12486db9057049c9ffb651f9619e6.iix" style="display: block; margin-left: auto; margin-right: auto; width: 620px; height: 520px;"/></p><p style="padding-left: 60px;">i. Fill in the information.</p><p style="padding-left: 60px;">ii. Click OK to save your work.   Then click Close on the Connection Properties form.   Then click the Close button on the Connection Status form.   It might be necessary to reboot your Virtual Machine to get the network settings to take.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">q. Now open the Windows Firewall link (in the lower left of the Network and Sharing Center form).</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-24 jive-image" src="85b1240adb5c5304b322f4621f96199b.iix" style="max-width: 1200px; max-height: 900px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">r. Click on the Change settings link.   This will open the Windows Firewall Settings panel.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="jive-image image-25" src="ac398c8edb9c5fc03eb27a9e0f9619b3.iix" style="max-width: 1200px; max-height: 900px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">s. Click on the Exceptions tab.   Make sure Remote Administration is checked (if this is off it will block your MID Server from performing PowerShell commands against your new server).   Then click the OK button to save your work.   Close out the Windows FIrewall form.</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-26 jive-image" src="e0636b3ddb541704ed6af3231f9619ed.iix" style="max-width: 1200px; max-height: 900px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;">t. At the top of the VirtualBox form click on the Machine menu item.   Then choose Settings...</p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><img   class="image-16 jive-image" src="8c21884edb58d3049c9ffb651f9619bb.iix" style="width: 620px; height: 522px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="padding-left: 30px;">u. Click on Network to display the network settings.</p><p style="padding-left: 30px;">v. Click on Attached to and choose Host-only Adapter.   This will set up the ability to communicate between your PC and the Virtual Machine Server.</p><p style="padding-left: 30px;">w. Click the OK button to save your work.   Your Server is now configured and ready to be accessed from your personal ServiceNow instance.</p><p style="padding-left: 30px;">x. You now will need to download and install the latest updates for your new server (requires quite a bit of time to babysit through several install cycles to bring it up to date).   The reasoning is to get to R2, and you will have the latest administrative tools including (important) Powershell 2.   <span style="text-decoration: underline;">Alternatively</span> you could download it from the following location and move it to your new Server.</p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;"><a href="https://www.microsoft.com/en-us/download/details.aspx?id=11829" title="https://www.microsoft.com/en-us/download/details.aspx?id=11829">Download Windows PowerShell 2.0 and WinRM 2.0 for Windows Server 2008 (KB968930) from Official Microsoft Download Center</a></p><p></p><p style="padding-left: 30px;">z.
