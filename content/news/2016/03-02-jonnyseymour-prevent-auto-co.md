<p>There are a few cases where the browser acts as <a title="n.wikipedia.org/wiki/Password_manager" href="https://en.wikipedia.org/wiki/Password_manager">password manager software</a>. It will store your passwords to safely save you time and efforts. Where it sounds like a good plan, there are times when you get an unexpected memo from the headache department. Sometimes Chrome, Firefox and IE will <a title="i.service-now.com/kb_view.do?sysparm_article=KB0563953" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0563953">set autocomplete for your password to "on"</a> as a result of passwords being saved for specific instances. We clearly set HTML on the password fields to say "do not auto-complete" but can be ignored.</p><p></p><p>In Chrome auto-complete for passwords is ON by default. To double check, just open your Chrome settings then go to advanced, and select "auto-complete passwords" ON. After that, to access your settings, go to: <strong>chrome://settings/passwords</strong> to manage your <a title="upport.google.com/chrome/?p=settings_password" href="https://support.google.com/chrome/?p=settings_password">passwords in Chrome.</a></p><p></p><p><img   alt="Chrome-security-extensions1.png" class="image-28 jive-image" height="106" src="22c964c2db94dfc03eb27a9e0f9619bf.iix" style="width: 141px; height: 106.271px; display: block; margin-left: auto; margin-right: auto;" width="141"/></p><p></p><p>If everything works fine, when you login into the instance the first time, Chrome will allow you to save the password.</p><p><img   alt="remember password chrome.jpg" class="image-18 jive-image" src="d90cc00adb509fc068c1fb651f9619fb.iix" style="width: 620px; height: 149px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>To see all of your passwords saved on Chrome, navigate to <a title="settings/passwords" href="chrome://settings/passwords">chrome://settings/passwords</a></p><p><img   alt="see all passwords.jpg" class="image-19 jive-image" src="1d982b71db509b048c8ef4621f9619f5.iix" style="width: 620px; height: 194px; display: block; margin-left: auto; margin-right: auto;"/></p><p>This works fine and the next time you try to login, it will auto-complete the password. You can recognize it because the background of the field usually changes to yellow to indicate that autocomplete is working.</p><p><img   alt="autocomplete success.jpg" class="image-20 jive-image" src="ebedc14edb14d344e9737a9e0f961945.iix" style="width: 620px; height: 255px; display: block; margin-left: auto; margin-right: auto;"/></p><p>When everything goes smoothly, you can efficiently login to your instance and save yourself the hassle of having to remember your password with auto-complete.</p><p></p><h2>When the browser password management changes go wrong</h2><p>I worship the ground Chrome walks. As painful for me as it is, there are some cases when the password fields are auto-completed even when it should not be. On Chrome, the browser matches the password with the "previous" field (not necessarily the login name). On ServiceNow, <a title="ki.servicenow.com/index.php?title=Introduction_to_Fields#gsc.tab=0" href="http://wiki.servicenow.com/index.php?title=Introduction_to_Fields#gsc.tab=0">password fields</a> could be anywhere in the form and even when the autocomplete="off" is set on them, some browser just ignore them.</p><p></p><table border="1"><tbody><tr><td><h4>Example #1: The User form (sys_user)</h4><p>If you open our sys_user form, change the password and click "update" (similar to form submit), the browser will try to save the password and associate it to the previous field.</p><p><img   alt="password associated.jpg" class="image-21 jive-image" src="7dc96486db909344e9737a9e0f9619cb.iix" style="width: 620px; height: 196px; display: block; margin-left: auto; margin-right: auto;"/></p><p>In this case, it will associate the password value with the 'Department' field.</p><p><img   alt="password department field.jpg" class="image-22 jive-image" src="483d8406db909fc068c1fb651f9619ee.iix" style="width: 620px; height: 141px; display: block; margin-left: auto; margin-right: auto;"/></p><p>If you open any other user from the same department, the autocomplete will incorrectly auto-complete the password. Saving the record will override the password. It will even ask you to save when leaving the form even if you have not changed anything.</p><p><img   alt="save password.jpg" class="image-23 jive-image" src="10176735db181704ed6af3231f9619ac.iix" style="width: 620px; height: 190px; display: block; margin-left: auto; margin-right: auto;"/></p><p>To avoid this problem of having the password associated with the wrong user, move the password field below an unique field like User ID or disable auto-complete.</p><p><img   alt="disable auto complete.jpg" class="image-24 jive-image" src="0626d106dbd4d304b322f4621f961950.iix" style="width: 620px; height: 148px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h4>Example #2: The X.590 form</h4><p>Another form with a similar problem is X.509 Certificate (sys_certificate). If auto-complete is enabled and have records, it will set the key store password that can cause problems on some integrations.</p><p><img   alt="x.590 form.jpg" class="image-25 jive-image" src="6c44f84adb589304b322f4621f961937.iix" style="width: 620px; height: 229px; display: block; margin-left: auto; margin-right: auto;"/></p><p>You can move the password field below an fairly unique field like Name or disable auto-complete to avoid the it from setting the key store password.</p><p><img   alt="key store password.jpg" class="image-26 jive-image" src="0298ef71dbd493049c9ffb651f961982.iix" style="width: 620px; height: 205px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h4>Example #3: SOAP message form</h4><p>On the SOAP message (sys_soap_message), the basic authentication fields get auto-completed. These fields can be overwritten when saving the form by the values auto-completed. The problem here is that both fields are initially empty.</p><p><img   alt="soap message.jpg" class="image-27 jive-image" src="98fd0546db1897049c9ffb651f961940.iix" style="width: 620px; height: 230px; display: block; margin-left: auto; margin-right: auto;"/></p></td></tr></tbody></table><p></p><p>Luckily, the auto-complete password problem will only affect forms with password fields. The complexity of the problem is that it could happen when users are UNAWARE of the auto-complete. Also, the password is saved to the whole domain (service-now.com sometimes) so it will cross into different instances or forms (e.g. passwords set on the sys_user form, then recalled by the browser on sys_soap_message (as showed on example #3). Note that most sophisticated<a title="n.wikipedia.org/wiki/List_of_password_managers" href="https://en.wikipedia.org/wiki/List_of_password_managers"> password managers like Lastpass, Keepass, 1password, etc</a> do not have the same problem as IE, Firefox and Chrome have. Also, Opera and Safari seems to work fine as well.</p><p></p><p>Here are the list of forms that can be affected by the browser password auto-complete:</p><p></p><table border="1" cellpadding="0" cellspacing="0" style="border: none; If you see YELLOW background, then you are in trouble.
