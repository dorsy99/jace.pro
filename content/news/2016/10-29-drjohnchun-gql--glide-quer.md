<p><a title="" _jive_internal="true" href="/community?id=community_blog&sys_id=8d2e266ddbd0dbc01dcaf3231f9619ef">Last time</a>, we built a simple app called GQL Pad that provided user interface for editing and running GQL statements; it also showed results in various formats that we could quickly inspect on the same web page. This time, let's walk through the steps for installing the app on your own instance from GitHub using the Helsinki Studio. If you're already familiar with importing files from GitHub, you may skip most steps below.</p><p></p><p><img __jive_macro_name="toc" class="jive_macro_toc jive_macro" data-renderedposition="92_8_61_72" src="/8.0.4.21bdc7e/images/tiny_mce3/plugins/jiveemoticons/images/spacer.gif"/></p><p></p><h3>FORKING A GITHUB REPOSITORY</h3><p></p><p>If you don't already, you'll need to have an account at <a title="ithub.com/join" href="https://github.com/join">GitHub.com</a>. You can sign up for free as long as your repositories are public. <a title="itlab.com/" href="https://gitlab.com/">GitLab</a> provides private repositories for free and you can use it with the Helsinki Studio. I'll be using GitHub throughout this post.</p><p></p><p>After logging into GitHub, point your browser to <a href="https://github.com/snowaid/GQL_v0.1" title="https://github.com/snowaid/GQL_v0.1">GitHub - snowaid/GQL_v0.1: Glide Query Language initial version from GQL blog</a> where I saved a copy of GQL v0.1 in its own repository. Towards the top right corner, click on "<strong>Fork</strong>" as shown in the screenshot below:</p><p></p><p><img   class="image-1 jive-image" src="65da1902db9c97049c9ffb651f961972.iix" style="border: 1px solid #ccc; max-width: 1200px; max-height: 900px;"/></p><p></p><p>This makes a copy of the repository in your own account, which should take only several seconds. After it's done, you'll be back in your account and see a screen similar to below. Towards top left, make sure you see <span style="color: #2873ee;">{your username}/GQL_v0.1</span> and "<span style="color: #666666; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 11px; background-color: #fafafa;">forked from </span><a title="ithub.com/snowaid/GQL_v0.1" href="https://github.com/snowaid/GQL_v0.1" style="background-color: #fafafa; color: #4078c0; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 11px;">snowaid/GQL_v0.1</a>" below it.</p><p></p><p><img   class="image-2 jive-image" src="e3073402dbdc5f048c8ef4621f961928.iix" style="border: 1px solid #ccc; max-width: 1200px; max-height: 900px;"/></p><p></p><p>Then click on the green "<strong>Clone or download</strong>" button and copy the URL as shown below (we'll paste it shortly below):</p><p></p><p><img   class="image-3 jive-image" src="42c59046db189fc068c1fb651f961963.iix" style="border: 1px solid #ccc; GQL Pad</strong> as shown below:</p><p></p><p><img   class="jive-image image-10" src="4d96b48adb989344e9737a9e0f961954.iix" style="border: 1px solid #ccc; max-width: 1200px; max-height: 900px;"/></p><p></p><h3>OPEN SOURCE COLLABORATION</h3><p></p><p>The GQL app is open source code licensed under a very permissive <a title="drlegal.com/l/mit" href="http://tldrlegal.com/l/mit">MIT license</a> and you're welcome to make contributions. While the app you just installed above will be frozen in GitHub in the <a title="ithub.com/snowaid/GQL_v0.1" href="https://github.com/snowaid/GQL_v0.1">GQL_v0.1 repository</a> (to ensure it stays in sync with what's been mentioned in this blog series), the working copy of the app is maintained in the separate <a title="ithub.com/snowaid/GQL" href="https://github.com/snowaid/GQL">GQL repository</a>. If you'd like to make contributions, please fork this repository.</p><p></p><h3>LIMITATIONS AND POTENTIAL ENHANCEMENTS</h3><p></p><p>This has been an attempt at demonstrating a proof of concept and we've successfully shown what we set out to accomplish: build a web service that takes SQL-like statements and returns result sets. Along the way, we created the GQL class as a Script Include, which can be called from any server-side scripts. We then built a processor to handle web requests and responses. To interactively demonstrate this, we built a UI Page app called GQL Pad. Everything was kept simple, with very little error handling. The app will work OK for positive cases, but may not perform gracefully when errors are encountered.</p><p></p><p>Here are some limitations and potential enhancements:</p><p></p><ol><li>GQL syntax checking: add to UI and GQL class.</li><li>GQL syntax highlighting: add to GQL Pad.</li><li>Error checking and handling: check for invalid syntax / column / table / encoded query.</li><li>Security: may need to use <span style="font-family: 'courier new', courier;"><a title="ki.servicenow.com/?title=GlideRecordSecure" href="http://wiki.servicenow.com/?title=GlideRecordSecure">GlideRecordSecure()</a></span> instead of <span style="font-family: 'courier new', courier;">GlideRecord()</span> to enforce ACL.</li><li>Add ability to run SELECT *.</li><li>Add calculated fields.</li><li>Add aggregate functions: COUNT(), MAX(), MEAN(), etc.</li><li>Add UNION: return result sets from multiple SELECT statements as one.</li><li>Add JOIN.</li><li>Add NEST: allow nesting of result sets from reference tables or related lists.</li><li>Add SAMPLE: return random rows based on statistical sampling.</li><li>Add XML output format using <span style="text-decoration: line-through;"><a title="ki.servicenow.com/?title=XMLHelper" href="http://wiki.servicenow.com/?title=XMLHelper">XMLHelper</a></span> (can't use in scoped app); alternative might be to use libraries like <a title="ithub.com/abdmob/x2js" href="https://github.com/abdmob/x2js">x2js</a>.</li></ol><p></p><p>This concludes this series for now. I'll continue to update the <a title="ithub.com/snowaid/GQL" href="https://github.com/snowaid/GQL">GQL repository</a> as I receive your open-source contributions and add enhancements. I may also post follow-up articles if any progress is worth sharing.
