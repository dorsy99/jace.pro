<p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><strong>NOTE</strong>: ON APRIL 1, 2016 ACCENTURE COMPLETED THE ACQUISITION PROCESS OF CLOUDSHERPAS.   AT THAT TIME THE CLOUDSHERPAS BLOG SITES WERE DOWNED FOREVER.   </span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">THIS IS THE REPUBLICATION OF MY ARTICLE FROM <span style="color: #666666; font-family: arial, helvetica, sans-serif; font-size: 16px;">August 21, 2014 ON THE CLOUDSHERPAS SERVICENOW DISCOVERY 101 BLOG.   </span></span></p><p></p><p><span style="color: #666666; font-size: 16px; font-family: arial, helvetica, sans-serif;">THE CONTENT CONTAINED HERE WORKS UP THROUGH THE FUJI VERSION OF SERVICENOW.   WITH GENEVA AND LATER YOU NEED TO BE USING THE install.bat (WIZARD) SOFTWARE PROVIDED WITH THE MID SERVER DOWNLOAD.</span></p><p><span style="color: #666666; font-size: 16px; font-family: arial, helvetica, sans-serif;">____________________________________________________________________________</span></p><p></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">I recently needed to do some Discovery testing, but wanted to monitor my MID Server closely in order to watch the local Windows system logs, </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">watch the temp directory and change and validate the MID Server service user. The issue was that I needed access to the MID Server all the </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">time, but the only people allowed to log into the server were network or system technicians. I was thinking there had to be a way to get access </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">when it occurred to me: Why couldn't I set up a MID Server service on my laptop?</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">So, here is the process I came up with for setting up my own laptop-run MID Server, which worked well for my needs around both Discovery </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">and Orchestration testing. This set-up can be installed on any desktop or laptop machine running either Windows or Linux (the example below </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">was done on Windows).</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><strong>Note</strong>: I would recommend installing the latest version of PowerShell on your proposed MID Server box (the minimum would be Powershell 2). You </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">may also find connectivity issues between your local machine and your ServiceNow instance if you are on your company's network. These will </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">need to be worked out with your local network administrator before the process described here can function.</span></p><p></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><strong>Setting Up a Local MID Server</strong></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">1. Log onto your favorite instance (preferably your <a title="" _jive_internal="true" href="/community?id=community_blog&sys_id=adfce2a5dbd0dbc01dcaf3231f961935">personal instance</a>).</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">2. Navigate to: User Administration -&gt; Users</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">a. Create a new User (Name: MIDUser; Password: your favorite password)</span></p><p style="padding-left: 30px;"></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">3. Now we need to create a group.   It is a best practice to create a group (even for one user), then add the roles to the group, and the users to the group.   You control a user's access through group membership.</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">a. Navigate to: User Administration -&gt; Groups</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">b. Create a new group (Name: MIDServer)</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">c. Add the role mid_server</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">d. Add the user MIDUser</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><img   class="image-12 jive-image" src="eed1d4cedb949704ed6af3231f961970.iix" style="width: 620px; height: 342px; display: block; margin-left: auto; margin-right: auto;"/></span></p><p style="padding-left: 30px;"></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">4. Navigate to: MID Server -&gt; Downloads</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\midserver download.jpg" class="jive-image" height="206" src="https://lh5.googleusercontent.com/YXQi_MeYr6gzULQlIM6U4zb4gog1uXLk9msKMzo2urArNCQdkNHam-vmOnxOrHauMIssgT_GQ6EvH4tK9_bIY7dAwqPwZyUXv813S6RJovLqBn0zBgoVDHKHkhai8ytMSlBG6l7A7Qa4SPmc" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">5. Pick your favorite version of MID Server install from either Linux or Windows. In this example, I use Windows 32-bit .</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">6. Download the file. It's <span style="color: #000000; font-size: 14.6667px; font-family: Arial;">85Mb</span>, just a short coffee break…</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">7. Unpack the file to your favorite location (it can be truly anywhere). In this example, I placed the file in c:\ServiceNow.</span></p><p></p><p style="text-align: center;"><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\WinRAR view.jpg" class="jive-image" height="232" src="https://lh4.googleusercontent.com/DPuO3OVeoPetDPv3jYGgBBptqsBOgBvifSFneDzQ-MMvwWhT6frshdn1Y2P3o8IRxxxTbblV9Ga4K_eyWrz84EOnUnBEH3vwC-BugjFXjwFCtnKDgkejTaqtyQkHV6FIKVZoDXp2sLKvEQjd" style="border-style: none;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">8. Now we are ready to configure the MID Server service. Navigate to the "agent" folder and find the file: config.xml.</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\Config file location.jpg" class="jive-image" height="249" src="https://lh3.googleusercontent.com/hieeqahlv8jtqUt2J3PZxtpzzH4cUEqXCzd_RoZKWbZZLzCvPyQGnuX1Pbr42rUnrUJRFGVxHk2ZIy7LAkDwSZS0Kgeu0E5gSWkD2HIUIEXHShZcanaUR_SFsyRr4ELtscCXUyoOO0XhW74u" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">9. Open it in your favorite text editor (I used Notepad++ for this example).</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">10. You will need to make a few modifications to the config before starting the service.</span></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">a. Modify the name="url" line and change value="…" to the url of your ServiceNow instance.</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><span>Example: &lt;parameter name="url" value="</span><a title="" _jive_internal="true" href="/yourinstancename.service-now.com/"" rel="nofollow" target="_blank">https://yourinstancename.service-now.com/"/</a><span>&gt;</span></span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">b. Modify the name="mid.instance.username" and change the value="…" to the MIDUser userid you set up in your instance.</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Example: &lt;parameter name="mid.instance.username" value="MIDUser"/&gt;</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">c. Modify the name="mid.instance.password" and change the value="…" to the password of the userid you set up in your instance. The </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">encrypt setting, if true (the default), specifies that the password will be encrypted in the config.xml file after a successful connection is </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">made. See the screenshot below.</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Example: &lt;parameter encrypt="true" name="mid.instance.password" value="password"/&gt;</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">d. Modify the name="name" and Change the value="…" to the name of your MidServer</span></p><p style="padding-left: 30px;"></p><p style="padding-left: 60px;"><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Example: &lt;parameter name="name" value="StevesEurekaServer"/&gt;</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\config.xml_snapshot.jpg" class="jive-image" height="281" src="https://lh4.googleusercontent.com/JvDjpT02XLUts7piWzjzqWEiFUG9iYtHIcMfGpVSAfam-EXfHO0mGbvhs-tlkFB74F7H8JeFg3kg3cl0cQjCh0A89l7CwMv9oTOtD2mFs9Dfil2RojAbuDU53lfeC3ikcZVcgVp88Q7E9bX5" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">11. Save</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">12. From the same directory, double-click start.bat (I prefer to do this from the command line in case there are errors thrown). Start.bat acts as </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">a service installer on the first time run. After that, it will be the method of starting and/or restarting your MID Server service. Stop.bat obviously </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">stops your service.</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Here is the start.bat from the Explorer window (view this as Administrator to avoid annoying messages from popping up!):</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\start.bat.jpg" class="jive-image" height="245" src="https://lh6.googleusercontent.com/5rkoM-qiGp3TXVp1fgarmZIhEaJxWwbgQfUdsbSgGZiMCFOtpuPEOPKtc5Tiyls0BtindRc3NWLSc-7SoNMHCHE2NnwRvrQbN_oCPz_abLryycuyJOw4Zk0KekuFb1878YXvi4cQEDaKHK65" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">13. The server will take a couple of minutes to come up, but there shouldn't be any errors.</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">14. In Windows, go to "Start" and type in "Task Manager." Choose "View Running Processes with Task Manager" (you can also get to this </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">window by right-clicking on the Start bar at the bottom of your screen and picking Task Manager). You should see something like the following </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">form appear.</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\verify that the service is running.jpg" class="jive-image" height="435" src="https://lh5.googleusercontent.com/1-qBK-jK0BUjU_EgUpeouiqhKA7X7GHMCxcfqOIon4CfImf20GGyhVEYeCslDLhrEo8kpYDe4xV4AcaX8lK-xq8kaq-g40O-FYgqtiWOUDNHdT_JHToMr_mUH4A7-K-khl3hMNow2fu_lA0u" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Go to the "Services" tab and scroll down until you find the snc_mid service. This is the default service name for the MID Server service. The </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">status should be "Running."</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">15. Once you see the MID Server service is up and running, you will find that your config.xml file has been modified. If you open it and scroll </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">down to the bottom of the config file, you will see that a mid_sys_id value has been retrieved from the ServiceNow instance and saved into the </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">config file. This helps the instance identify the MID Server and vice versa.</span></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\mid_sys_id.jpg" class="jive-image" height="68" src="https://lh5.googleusercontent.com/0fpXAYLfpMO9aByofg4wNrUSk0VXatsyPKoj6OJ8YpaSlS0a7gbaW6CeUA52zPexOiSwNgAuan5G-0Tr98MTPRzpdrP0ZlUE3RyYHcS-GpnP6IbI6mFWFAAhdrEw9J0wNd4rfUCdr-2OqC-f" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><strong>NOTE</strong>: You will also notice that I turned on the "use_powershell" attribute on the mid server.   This can be manually entered here, or better, added in as a capability from the ServiceNow interface.</span></p><p></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;"><strong>Connecting to Your Local MID Server</strong></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">Now back to the ServiceNow instance in your browser!</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">16. Navigate to Mid Server -&gt; Servers</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">17. You should see your new MID Server with the name you gave it spinning happily. The status should be green and say "Up," the "Name" field </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">should contain the same name as your "Name" parameter in the MID Server config file and the "Host name" is your local desktop/laptop </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">name.</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\MID Server list.jpg" class="jive-image" height="100" src="https://lh4.googleusercontent.com/3DIQn84lT-y5pUd6tlnEGlRYvvHSXS8AJjbT5Fk3Lr-gmSvS-Z_XL8XM3zbMvoQPEPG059HzAQq8-R-RFZfd6UTV6PLCLbFYpHe3TEGVuuviO-ZZ5biUxCAskIbgPl-5hAS_h8FwLXsBqpVF" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="623"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">18. Click into your MID Server link.</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">19. You should see all the current stats on your server, along with IP info, etc. Note that the configuration parameters you entered on your </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">local machine are now reflected in the Configuration Parameters tab. Slick, huh?! The mid_sys_id is the same you saw automatically written </span><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">into your local config file.</span></p><p></p><p><span style="color: #000000; font-size: 14.6667px; font-family: Arial;"><img alt="C:\Users\User 1\Desktop\MID Server stats.jpg" class="jive-image" height="376" src="https://lh5.googleusercontent.com/ZAE07pTqJV9PEdKRgcdeQrNA6YPCrJQqJ_s846qHITw7zEUqExjzfAR41G6Mneyn5zeIHmJS928Q2vtZVnYbPbNvd5nJuSEN2LldSfjBZvZ9kaBUpao-Lwa9unAnn4t0KaGchyTBP9k2gPPZ" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif; font-size: 12pt;">You are done: You set up your own local MID Server… Have fun!</span></p><p></p><p><span style="font-family: arial, helvetica, sans-serif;
