<p style="font-family: arial, sans-serif; color: #666666;"><span style="font-weight: inherit; font-family: inherit; font-style: inherit;"><strong>NOTE</strong></span>: ON APRIL 1, 2016 ACCENTURE COMPLETED THE ACQUISITION PROCESS OF CLOUDSHERPAS.   AT THAT TIME THE CLOUDSHERPAS BLOG SITES WERE DOWNED FOREVER.</p><p></p><p style="font-family: arial, sans-serif; color: #666666;">THIS IS THE RE-PUBLICATION OF MY ARTICLE FROM <span style="font-weight: inherit; font-family: inherit; font-style: inherit;"><strong>December 30, 2014</strong></span> ON THE CLOUDSHERPAS SERVICENOW ADMIN 101 BLOG.</p><p style="font-family: arial, sans-serif; color: #666666;"></p><p style="font-family: arial, sans-serif; color: #666666;">EVEN THOUGH THIS WAS ORIGINALLY CREATED FOR THE EUREKA RELEASE I THOUGHT THAT IT WAS STILL RELEVANT AS THE TECHNIQUES USED HERE WILL STILL WORK IN THE LATEST RELEASE.</p><p style="font-family: arial, sans-serif; color: #666666;">____________________________________________________________________________</p><p style="font-family: arial, sans-serif; color: #666666;"></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">The latest release of ServiceNow, Eureka, introduced a whole new interface: UI14. Among the many updates were a group of visual changes, including how the "background" color is displayed in the List View of all tables. Instead of filling in the background of a cell, there is now a colored dot next to the cell contents. Since that change can take some time to get used to, I've fielded a number of questions about how to change this back without losing all the other UI14 visual candy.</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">So is it possible? Yes! Here's how it's done:</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">We will need to do the following things to get us to where we want to go:</span></p><p></p><ol><li><span style="font-size: 12pt;">Upload simple background images into the images table. These will act as our new background colors.</span></li><li><span style="font-size: 12pt;">Create or modify styles to set our background colors.</span></li></ol><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">Got those? Then let's get started!</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">I used the following site in conjunction with my experimentation:   </span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;"><a title="" _jive_internal="true" href="/www.w3schools.com/cssref/default.asp" rel="nofollow" target="_blank">http://www.w3schools.com/cssref/default.asp</a></span></p><p></p><h3 dir="ltr" style="margin-top: 14pt;"><span style="color: #000000; font-size: 14pt; font-family: Calibri;"><strong>Import the Background Images</strong></span></h3><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">First I imported the (attached) list of images. To do this:</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">1. Navigate to System UI / Images</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;"><img alt="C:\Users\User 1\Desktop\1. import image zip.jpg" class="jive-image" height="390" src="https://lh4.googleusercontent.com/EeEjXx8I9jZDAwmLWjW5RBRryhAryreGjSa5A_XnNaLHG0pa8eAvQKDTHsJeJZQvsthRFIJZVgYzP1qBalIul_d9Ku7uowk_GPkuTQ9FI9OdYrJm5rszClTDyfsSBVVt0ePxfGHJjBs7V2Fm" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">2. Right-click on the header bar and pick "Import Image ZIP File"</span></p><p></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: Arial; color: #000000;"><img alt="C:\Users\User 1\Desktop\2. choose zip file to import.jpg" class="jive-image" height="163" src="https://lh5.googleusercontent.com/jFOABzzwjUYkORZhrC5jfmwYEQsNukJyBhw_wj_EIBYY8422gpD1hRvaKJVYSWwXUwHEdfY7w4mIFaoYRMocBBv6Q4H1gJPbjf-RiNTx370uiISVONcTSFEdps3GuLM4p_hC8k9nav-ZjuzV" style="border-style: none;" width="311"/></span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">3. Choose the zip file of images you want to upload. In this case, it's the images.zip file.</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">4. Click the upload button. ServiceNow automatically extracts the image files from the zip and puts them in the images table where they are ready to reference. Cool, huh?   </span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;"><img alt="C:\Users\User 1\Desktop\3. loaded images.jpg" class="jive-image" height="349" src="https://lh3.googleusercontent.com/cD3Ham2uofnfMQ0gAE_2-zXlMJK-frSE3aQXJ7lWdC-nFGo3CdHL5wReW3kJpHDqtrmbpthSgUpI5W7wxLvSnfr2yZBGF8tp9kj69VCY5ptEnzm6PFF7K54cgEhMk2nXjLwsIe0aPXC0VgaU" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p></p><h3 dir="ltr" style="margin-top: 14pt;"><span style="color: #000000; font-size: 14pt; font-family: Calibri;"><strong>Style Creation / Modification</strong></span></h3><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">Next, pick your favorite table. I chose the Incidents table. You can see here the "before" picture. I am going to pick on the Priority styles. Note the "dots."</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;"><img alt="C:\Users\User 1\Desktop\4. before with dots.jpg" class="jive-image" height="432" src="https://lh3.googleusercontent.com/GB7znusYDHONR2ZuPIA_yrP9GnbfMlncQoRYBAeAw4XnJBVHYWKZ92TcbHdJHVylR5aORqBRadmQLnyysLPYVsfUOQactXHV6nF2OFOSOyhfYbZdDzkufjEGfgltkrfEMRAp2_MxNL4Q7Z_X" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">You will then need to create or modify existing styles. Remember: if you change it, you own it. I modified the two existing out-of-the-box styles for the Incident/Task Priority field for the purposes of this article.</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;"><strong>A side note</strong>: One neat thing is that you can have multi-line styles -- it's not necessary to one-line the information. When I first figured this out, I reasoned: Since you can have "chained," semi-colon delimited-style information, you should be able to just put each on its own line. I tried it and it worked! You can also have comments just like in a normal CSS3 style. However, some CSS3 things flat out don't work (animations, for example). See more below about things I tried that worked -- you'll have to play around and see what you can do as well!</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">To continue, I modified the following styles:</span></p><p></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: arial, helvetica, sans-serif;"><span style="color: #000000;"><strong>Table</strong>: Task</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;"><strong>Field Name</strong>: Priority</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;"><strong>Value</strong>: 1</span></span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: 'courier new', courier;"><span style="color: #000000;">/* background-color:tomato - the original value*/</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">font-size: 18px; </span></span></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: 'courier new', courier;"><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">/* forces the image to the background - this MUST come before the background-image line */</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">border-style: solid; </span></span></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: 'courier new', courier;"><span style="color: #000000;">background-image: url("background_red.jpg"); </span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">border: 2px solid darkred; </span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">text-decoration: underline; </span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">text-align: right;</span></span><span style="font-size: 12pt; font-family: 'courier new', courier;"><br/><br/></span></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: arial, helvetica, sans-serif;"><span style="color: #000000;"><strong>Table</strong>: Task</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;"><strong>Field Name</strong>: Priority</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;"><strong>Value</strong>: 2</span></span></p><p style="padding-left: 30px;"></p><p style="padding-left: 30px;"><span style="font-size: 12pt; font-family: 'courier new', courier;"><span style="color: #000000;">/* background-color:orange - the original value*/</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">font-size: 18px; </span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">font-weight: bold; </span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">font-family: "Times New Roman", Georgia, Serif;</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">font-style: italic;</span><span style="color: #000000;"><br class="kix-line-break"/></span><span style="color: #000000;">color: orange; font-family: Arial; color: #000000;">Once I saved these, they immediately become active in the list view. Here is the "after" picture:</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;"><img alt="C:\Users\User 1\Desktop\5. after with background images.jpg" class="jive-image" height="378" src="https://lh5.googleusercontent.com/yvoXhSi_q_ATevJZfN0QRWAaC-Gfl-u1EUNbEM5gBgSvi6OvL3KTXaTUuS-OednyfWj6f8PVHBbg5jfyD1HAw52jlQXcrCxVx0efWDcISfE8CUu2FjoPkbXZdm83_MsFw4wYRlriANdBP8mx" style="border-style: none; display: block; margin-left: auto; margin-right: auto;" width="624"/></span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">So what's happening here? I set the following for the particular style:</span></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">Font color, size, weight, family and style. I also added in the underline and aligned the text to the right.</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">Then the magic happens: we replace the "dot" with one of our uploaded images. You MUST set the border-style before that line, as this causes the cell to "wake up" to the new image. If you skip this step, all you will get is a confused cell with a rounded square set to your default color (by the way, you can change that color using the background-color CSS tag). Our image is automatically set to a default of 100% x 100% so it fills the entire cell.</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">That's all there is to it! The only "downside" to this is that it requires that you go out and make this modification to every one of the styles you want to appear different than the "dot,"   as this method does not apply the change to all "dots."</span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">While these style modifications do work in Dublin and the old UI, I recommend testing them out because they may not act quite the same.</span></p><p></p><p><span style="color: #000000; font-size: 14pt; font-family: Calibri;"><strong>Additional Capabilities</strong></span></p><p></p><p><span style="font-size: 12pt; font-family: Arial; color: #000000;">Ready for more?
