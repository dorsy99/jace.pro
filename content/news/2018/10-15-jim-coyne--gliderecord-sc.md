<h5>Included in the <a href="community?id&#61;community_blog&amp;sys_id&#61;b3c843aadb892b40fece0b55ca961906" rel="nofollow">Developer Toolbox</a> Update Set available on Share (link in the &#34;Developer Toolbox&#34; post).</h5>
<p> </p>
<p>Here are a couple useful tools when you need to do some scripting.  They are list view Context Menus:</p>
<p><img src="26a0f903dbc12300fece0b55ca9619c0.iix" /></p>
<p>...and form UI Actions:</p>
<p><img src="32f73dcfdbc12300fece0b55ca96197e.iix" /> </p>
<p>They will either popup a window with a script template based on the filter of the list view, or the sys_id of the form record:</p>
<p><img src="281dedc7db812300fece0b55ca96190b.iix" /></p>
<p>...open the Background Script page in a new tab/window: and pre-populate the script text box a half second later, giving the page enough time to load:</p>
<p><img src="791fe5cfdb812300fece0b55ca961972.iix" /></p>
<p>...or open a new <a href="https://developer.servicenow.com/app.do#!/share/contents/9650888_xplore_developer_toolkit?v&#61;4.05&amp;t&#61;PRODUCT_DETAILS" rel="nofollow">Xplore: Developer Toolkit</a> window:</p>
<p><img src="7a587103db052300fece0b55ca961981.iix" /></p>
<p>I will often use these tools whenever I need to script something.  I&#39;ll start off with a list and filter it down to how I need it and then copy the template result and paste it wherever I need it and modify as required.</p>
<p>The &#34;Preview&#34; menus require a separate UI Page used to display the results.  The &#34;Xplore&#34; one requires the &#34;Xplore: Developer Toolkit&#34; to be installed separately, available <a href="https://developer.servicenow.com/app.do#!/share/contents/9650888_xplore_developer_toolkit?v&#61;4.05&amp;t&#61;PRODUCT_DETAILS" rel="nofollow">here</a> on the Share site.</p>
<p>The XML files have been attached for you to import into your instance.  As always, try it out in your company&#39;s development instance first, or better yet, your own personal development instance.</p>
