<p>One question I have been asked a lot over the past few months is, "How do I hide the 'New' button on the table lists in Service Portal using 'Data Table' or 'Data Table from Instance Definition Widget'"? This button allows users to create new records directly from the list page on Service Portal, just like the 'New' UI Action works from the list view on the platform. You may not want users to create records through the list and would rather them utilize the Service Catalog instead where they can follow a more step-by-step process. Omitting this button will force users to do this.</p><h2><img   alt="hide New button.jpg" class="image-12 jive-image" src="12084c42db9c5fc03eb27a9e0f961985.iix" style="width: 620px; height: 161px; display: block; margin-left: auto; margin-right: auto;"/></h2><h2>"Omit new button" in Service Portal</h2><p>This button works much like any UI Action on the back-end platform. In order for you to be able to utilize this as a user, you will need to have the appropriate ACL(s) for the specified table of the list view.</p><p></p><p>One difference you will see between Service Portal and the back-end platform, is the List Control option found on the list view. On the back-end platform, this allows you to easily set the "Omit new button" field to false, removing the ability to create a new record directly through the UI Action from the list view. Unfortunately, this "Omit new button" field does not cross over to the Service Portal. However, getting that button to disappear is pretty easy. Maybe not as easy checking a box, but nevertheless, easy.</p><p><img   alt="list_control.jpg" class="image-13 jive-image" src="7592eb3ddb9093041dcaf3231f96190b.iix" style="width: 620px; height: 320px; display: block; margin-left: auto; margin-right: auto;"/></p><h2></h2><h2>How to remove the "New" button on a Service Portal list page</h2><p>Out of box, the Service Portal list view is defined through the Data Table Widget (widget-data-table), and an out-of-box example can be seen on the 'List' page. In fact, if you ever want to pull a table list in Service Portal, you can go directly to <strong>/sp?id=list&amp;table=&lt;table_name&gt;.</strong></p><p></p><p>To get to the List page, we first need to go the page editor. You can get there by going through <strong>/sp_config</strong> and selecting the <strong>Page Editor,</strong> or directly by going to /sp_config?id=page_edit. From the reference drop down, type and select 'List'. From here you will see the list branch. Let's first click the Data Table widget to review the source of the 'New' button.</p><p></p><p><img   alt="list page editor.png" class="image-8 jive-image" src="14b844cadb109704ed6af3231f961987.iix" style="width: 620px; height: 68px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>Here you will see the "New" button defined through the button tag on line 13 in the body of the HTML template for the widget. You will see 'ng-if="options.show_new &amp;&amp; data.canCreate"'. This is checking to see if the Boolean value for New button option is set to true or false and if the user has access to create against this table.</p><p></p><p>Now, I know what you're thinking, 'Sweet! I'll just remove this line of code"! Although this is true, don't get too excited (yet), because if you haven't already noticed, this widget is in read-only state. This is by default and cannot be modified. One way to remove, or comment-out, this line of code is to simply clone the widget.</p><p></p><p><strong>You can clone the widget by:</strong></p><ol><li>Clicking the the hamburger icon in the top-right corner of the widget's page</li><li>Select 'Clone "Data Table"</li></ol><p><img   alt="clone widget.jpg" class="image-17 jive-image" src="2475c8c6db581304b322f4621f9619c8.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p>From here you will need to give it a widget name and ID.</p><p><img   alt="Screen Shot 2017-01-04 at 2.20.53 PM.png" class="image-15 jive-image" height="427" src="0743c586db5897041dcaf3231f9619ec.iix" style="width: 450px; height: 427.099px; display: block; margin-left: auto; margin-right: auto;" width="450"/></p><p>Once created, you will be able to modify the code and from here you can delete or comment out the line (line 13) and save. You can then go back to the List page, and set the widget to your new cloned Data Table widget without the new button.</p><p><img   alt="data table widget2.png" class="image-16 jive-image" src="9d72640edb10dfc068c1fb651f961970.iix" style="width: 620px; height: 213px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><pre __default_attr="info" __jive_macro_name="alert" alert="info" class="jive_text_macro jive_macro_alert" data-renderedposition="2408.6875_8_1192_43"><p>I recommend setting the options directly through the page instead.</p></pre><p></p><p>If you are looking to hide the New button across any page utilizing the Data Table widget, then perhaps the best way (and the way I recommend), is to simply hide the new button through the options schema. On the Data Table widget, you will see options listed in the Client Controller (they are commented out — Line 3 to 16). Take notice of the 'show_new' Boolean value.</p><p><img   alt="data table widget.jpg" class="image-9 jive-image" src="d9009182db5897049c9ffb651f9619e1.iix" style="width: 620px; height: 184px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p></p><p>This is the option we are going to set to false on the List page. At the top, on the list branch, select List to open the page record.</p><p><img   alt="list branch.jpg" class="image-10 jive-image" src="719649cedbd013043eb27a9e0f961941.iix" style="width: 620px; height: 63px; display: block; margin-left: auto; margin-right: auto;"/></p><p>At the bottom of the form you will see the 'Additional options, JSON format' field. Here is where you are going to set the option for 'show_new' to false through the following:</p><pre __default_attr="plain" __jive_macro_name="code" class="jive_macro_code _jivemacro_uid_14835713896573444 jive_text_macro" data-renderedposition="2907.6875_8_1192_96" jivemacro_uid="_14835713896573444"><p>{</p><p>   "show_new" : {</p><p>   "displayValue" : false,</p><p>   "value" : false</p><p>   }</p><p>}</p></pre><p></p><p>From here, you can click the <strong>Save</strong> button in the bottom right. Now test to see whether or not the button is showing through any table, like the incident table: /sp?id=list&amp;table=incident</p><p><img   alt="Service portal new button.jpg" class="image-11 jive-image" src="2f07ec06db14dfc03eb27a9e0f96190f.iix" style="width: 620px; height: 272px; display: block; margin-left: auto; margin-right: auto;"/></p><p>Now the new button is gone and users will not be able to create new records through the list. If you ever need to revert back to showing the New button, you can always remove this from the field. For more information on configuring the options-schema, check out the <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/build/service-portal/concept/c_WidgetInstanceOptions.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/build/service-portal/concept/c_WidgetInstanceOptions.html">Options Schema Docs page</a>.
