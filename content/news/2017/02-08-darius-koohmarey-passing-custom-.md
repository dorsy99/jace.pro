<p><span style="line-height: 1.5;">When customers utilize Okta as their SSO/Integration solution between their current AD/LDAP environment and ServiceNow, they may have noted that there are a set number of fields they can map to:</span></p><p></p><p>(Okta field: ServiceNow field format)</p><p>User ID: user_name</p><p>Email: email</p><p>First Name: first_name</p><p>Last Name: last_name</p><p>SysID: sys_id</p><p>Password: user_password</p><p>Active Status: active</p><p>Manager: manager</p><p>Location: location</p><p>Department: department</p><p>Business Phone: phone</p><p>Mobile Phone: mobile_phone</p><p>Cost Center: cost_center</p><p>Title: title</p><p></p><p>However, by using clever Profile Mappings from your directory connection to Okta, and Okta to ServiceNow, you can automatically take in any attribute from your directory environment to your user records in ServiceNow.</p><p></p><pre __default_attr="warning" __jive_macro_name="alert" alert="warning" class="jive_text_macro jive_macro_alert" data-renderedposition="461.97918701171875_7.986111640930176_1191_62"><p>Keep in mind that mapping a custom field will come at a cost of one of the base system text fields above, so we suggest using an item that is not often utilized, or that is less business critical. Using a text field is needed as a reference field will not properly map the data.</p></pre><p></p><h1>How to map custom fields from your directory using Okta and ServiceNow</h1><p><span style="line-height: 1.5;">In this example, we will use the Business Phone instead.</span></p><p></p><h3><span style="line-height: 1.5;">Part I. Okta configuration</span></h3><ol><li><span style="line-height: 1.5;">Begin by logging into your Okta instance as an administrator, and navigate to the <strong>Directory</strong> tab in the upper left.<br/><img   alt="okta+configuration+directory.png" class="image-17 jive-image" height="144" src="720a0c02db5017041dcaf3231f9619df.iix" style="width: 250px; height: 144.295px; display: block; margin-left: auto; margin-right: auto;" width="250"/></span></li><li><span style="line-height: 1.5;">In the <strong>Profile Mappings</strong> section, select <strong>Edit Mappings</strong> for the ServiceNow application.<br/><img   alt="okta+configuration+profile+mappings.png" class="image-18 jive-image" src="63146ff5db1c5b048c8ef4621f961980.iix" style="width: 620px; height: 99px; display: block; margin-left: auto; margin-right: auto;"/></span></li><li>Click the section titled <strong>Okta to ServiceNow - Eureka and later releases</strong>.<br/><img   alt="okta+configuration+to+servicenow.png" class="image-19 jive-image" src="41770146db1413043eb27a9e0f9619c2.iix" style="width: 620px; height: 102px; display: block; margin-left: auto; margin-right: auto;"/></li><li>Find the field you want to map over in the right. (Keep in mind that you will not be able to use this field then.) In this example, we will write your <strong>CUSTOMATTRIBUTE</strong> over the <strong>businessPhone</strong> field.<br/><img   alt="okta+configuration+custom+attribute.png" class="image-20 jive-image" src="2d26284edbdc5344e9737a9e0f961977.iix" style="width: 620px; height: 102px; display: block; margin-left: auto; margin-right: auto;"/></li><li>Navigate to the ServiceNow application in the <strong>Applications</strong> section. Under the <strong>Provisioning</strong> tab, click <strong>Edit</strong>.<br/><img   alt="okta+configuration+provisioning.png" class="image-21 jive-image" src="a126808edb9c5704ed6af3231f96198c.iix" style="width: 620px; height: 251px; display: block; margin-left: auto; margin-right: auto;"/></li><li>Find the field you mapped over in step 4 and define the field name in ServiceNow to be mapped over. In this example, we map to a <strong>u_CUSTOMATTRIBUTE </strong>field.<br/><img   alt="okta+configuration+text+field.png" class="image-22 jive-image" src="85532739db9c9f04e9737a9e0f961949.iix" style="width: 620px; height: 148px; display: block; margin-left: auto; margin-right: auto;"/></li></ol><h3>Part II: ServiceNow configuration</h3><p></p><ol><li>Start by searching for the user records from the application navigator, and open any user record.<br/><img   alt="okta+configuration+user+record.png" class="image-23 jive-image" src="7b480dc6dbd897041dcaf3231f961974.iix" style="width: 620px; height: 150px; display: block; margin-left: auto; margin-right: auto;"/></li><li>Right-click the user record header and select the <strong>Configure &gt; Form Design</strong> option.<br/><img   alt="okta+configuration+user+record+configure.png" class="image-24 jive-image" src="6944e402db90dfc068c1fb651f961952.iix" style="width: 620px; height: 134px; display: block; margin-left: auto; margin-right: auto;"/></li><li>From the <strong>Field Types</strong> section, click a <strong>String</strong> field and drag it anywhere on the form.<br/><img   alt="okta+configuration+field+types+string.png" class="image-25 jive-image" src="b565c842db98db048c8ef4621f961996.iix" style="width: 620px; height: 325px; display: block; margin-left: auto; margin-right: auto;"/></li><li>Define this field to have any label you prefer with the field name <strong>CUSTOMATTRIBUTE</strong>. Keep in mind that ServiceNow automatically appends <strong>u_ </strong>to custom field names, as well as setting them all to lowercase. So since you entered <strong>CUSTOMATTRIBUTE</strong> as the name, we are mapping (from step 6 in the Okta configuration) to the <strong>u_customattribute</strong> field.<br/><img   alt="okta+configuration+custom+attribute+label.png" class="image-26 jive-image" src="59a2ef75db141fc03eb27a9e0f961934.iix" style="width: 620px; height: 181px; display: block; margin-left: auto; margin-right: auto;"/></li></ol><p>Congratulations - you have now mapped custom fields from your directory using Okta and ServiceNow! Stay tuned for more updates as we enhance the functionality to include custom mappings by default.</p>
