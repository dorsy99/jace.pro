<p>Most fields in ServiceNow store a single inputted text value. To view previous values one needs to go into the record's audit history. Journal fields are different, in that all previous values of this field are shown as distinct posts within the Activity Stream timeline. This allows them to be used as a method of communication with users and coworkers about the current record, and also as a log of the history of the record.</p><p></p><p>With<a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/task-table/reference/r_JournalFields.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/task-table/reference/r_JournalFields.html"> journal fields</a>, the content of notifications are commonly based on the newly updated value of the journal fields, and that they only accept input as plain-text. Out of box, there are two journal fields on the task table:</p><p></p><ul><li>Additional comments <span style="color: #909090;">[comments]</span></li><li>Work notes<span style="color: #909090;"> [work_notes]</span></li></ul><p></p><h1><span style="color: #505050;">Adding HTML code in Journal fields</span></h1><p>As with most fields, if the new value contains HTML code it is escaped. For example, if you were to type the below <a title="eveloper.mozilla.org/en-US/docs/Web/HTML/Element/blockquote" href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/blockquote">valid HTML code</a> in a journal field:</p><p><img   alt="html code journal fields.jpg" class="image-27 jive-image" src="3244a802db90dfc068c1fb651f961980.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p>It would appear in the Activity Stream as follows:</p><p><img   alt="INC0010056___ServiceNow.png" class="image-40 jive-image" src="4f650406db5cd3041dcaf3231f96196f.iix" style="width: auto; height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p>The way this is done is by replacing <span style="color: #505050;">any characters in a journal field that could be interpreted by a browser as HTML with it's <a title="ev.w3.org/html5/html-author/charref" href="https://dev.w3.org/html5/html-author/charref">HTML entity</a><span style="color: #505050;"> code</span>.</span></p><p></p><p><span style="color: #505050;"><span style="color: #505050;">For example, HTML tags make use of <strong><em>&lt;</em></strong> and <strong><em>&gt;</em></strong> to declare the start and end of a tag. ServiceNow replaces these with their HTML entity</span><span style="color: #505050;"> code </span><span style="color: #505050;"><strong>&amp;lt;</strong></span><span style="color: #505050;"> (less than) and </span><span style="color: #505050;"><strong>&amp;gt;</strong></span><span style="color: #505050;"> (greater than), meaning that when the browser reads the code it doesn't get processed as a literal <em><strong>&lt;</strong></em> and <em><strong>&gt;</strong></em>, and therefore does not mistakenly become a HTML element.</span></span></p><p></p><h1><span style="color: #505050;"><strong>Using the [code] tag in Journal Fields</strong></span></h1><p><span style="color: #505050;">Text wrapped in<a href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/field-administration/reference/r_CODETagsRenderHTMLInAJrnalFld.html"> <em><strong>[code]</strong></em> tags</a> bypasses the escaping process described above, and will instead be rendered as HTML. For example, the same input used above, but wrapped in <em><strong>[code] </strong></em>tags:</span></p><p><span style="color: #505050;"><img   alt="INC0010056___ServiceNow.png" class="image-41 jive-image" src="79a2eb35dbd01fc068c1fb651f9619df.iix" style="display: block; margin-left: auto; margin-right: auto; width: auto; height: auto;"/></span></p><p>Will appear in the Activity Stream as follows:</p><p><img   alt="INC0010056___ServiceNow.png" class="image-42 jive-image" src="5e858d02db549f048c8ef4621f961962.iix" style="width: auto; height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p><span style="color: #505050;">Note that the <em><strong>&lt;blockquote&gt;</strong></em> tags are now being interpreted as HTML elements, and are not being escaped. </span><span style="color: #505050;"><span>Also note that you can use multiple sets of </span><em><strong>[code]</strong></em><span> tags within a single journal entry.</span></span></p><p></p><h1><span style="color: #505050;"><strong>Formatting styles in Journal entries</strong></span></h1><p><span style="color: #505050;">There are a number of HTML elements that you can use to add styles to your journal updates.
