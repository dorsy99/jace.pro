<p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"><span style="font-weight: inherit; font-size: 10.5pt; font-family: inherit; font-style: inherit; background-position: initial;"><strong>NOTE</strong></span><span style="font-weight: inherit; font-style: inherit; font-size: 10.5pt; font-family: Arial, sans-serif; background-position: initial;">: MY POSTINGS REFLECT MY OWN VIEWS AND DO NOT NECESSARILY REPRESENT THE VIEWS OF MY EMPLOYER, ACCENTURE.</span></p><p></p><p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"><span style="font-weight: inherit; font-style: inherit; font-size: 11pt; font-family: Arial, sans-serif; background-position: initial;">DIFFICULTY LEVEL:   </span><span style="color: #7f6000; font-weight: inherit; font-size: 14.6667px; font-family: inherit; font-style: inherit; background-position: initial;"><strong>INTERMEDIATE</strong></span></p><p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"><span style="font-weight: inherit; font-style: inherit; font-size: 11pt; font-family: Arial, sans-serif; background-position: initial;">Assumes basic knowledge and/or familiarity of the Studio IDE in ServiceNow.</span></p><p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"><span style="font-weight: inherit; font-style: inherit; font-size: 10.5pt; font-family: Arial, sans-serif; background-position: initial;">____________________________________________________________________________</span></p><p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"></p><p><span style="font-weight: inherit; font-style: inherit; font-size: 12pt; font-family: Arial, sans-serif; background-position: initial; color: #303030;">Working inside of the Studio Integrated-Development-Environment (IDE) has some definite benefits!   However, there are some interesting omissions as well.   Some of these center around the IDE, while others around Scoped Applications in general.</span></p><p></p><p></p><p><span style="color: #303030; font-size: 14pt; font-family: Arial, sans-serif; font-style: inherit; background-position: initial;"><strong>Significant Advantages of the Studio IDE</strong></span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">1. Navigation and Tabbed environment</span></p><p></p><p><span style="font-size: 12pt;">Let me gush about this for a bit!   The navigation bar gives us only what is in our application!   This makes is so much easier to find, open, and move between application files.   I am not sure how you are handling this right now, when you are developing in Global, but for me I would normally open several browser tabs to accomplish the tabbed bit.   The browser tab method can be painful, and if I have too many open, it becomes hard to find things.   So, this is a definite improvement.</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="image-1 jive-image" height="318" src="7246d802db989704ed6af3231f9619c3.iix" style="max-width: 1200px; max-height: 900px; width: 531px; height: 317.57px;" width="531"/></span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">2. Code Search</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">Here is the ONE thing I wish was available for Globally scoped applications as well!   You can search just your project, or you can search all of the code-base on your instance!   The function returns the results as a tab, and you can open any of the code found in the search to look it over.   NICE feature!</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="image-2 jive-image" height="316" src="ee1a7f31db545fc068c1fb651f96196d.iix" style="max-width: 1200px; max-height: 900px; width: 530px; height: 316.418px;" width="530"/></span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">3. Quick Creation of Most Scripts and Files</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">The Create Application File button opens up a search form that allows you to pick from what can be created in the scoped world.   Simply navigate to the object you want to create, or begin typing it in the search text box.   Once you choose the object it is automatically generated in the Application Namespace, and opened as a tab in the IDE.   Once saved you will then see it appear in the left navigation bar.</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="image-3 jive-image" height="318" src="66eb5942db5c1b04ed6af3231f9619d4.iix" style="max-width: 1200px; max-height: 900px; width: 531px; height: 318.26px;" width="531"/></span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">4. Push to Repository - External </span><span style="color: #303030; font-size: 12pt; font-family: Arial;">Source Control</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">Another really great feature of the IDE.   You can link your project to an external repository like GitLab, GitHub, or BitBucket (to name a few).   Then either manually push your project updates, or they will be automatically pushed when you publish to the local Application Repository.   You can do branching of your project if you have to have a separate set of code (for example: maintenance mods).   In moving from branch-to-branch it will take a bit as the old application is removed and the new one is installed (slow), but still a nice feature.</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="jive-image image-4" height="318" src="a3b3f08adb1cdfc03eb27a9e0f9619ac.iix" style="max-width: 1200px; max-height: 900px; width: 529px; height: 318.2px;" width="529"/></span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">5. Push to Internal Application Repository — Deployment</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">In a recent article (<span style="font-size: 16px; font-family: arial, sans-serif; color: #666666;"><strong><a title="" _jive_internal="true" data-containerid="1007" data-containertype="37" data-objectid="6589" data-objecttype="38" href="/community?id=community_blog&sys_id=d8fd622ddbd0dbc01dcaf3231f9619c9" style="font-weight: inherit; font-style: inherit; font-family: inherit; color: #266fc8;">Community Code Snippets: Scoped Application Distribution</a></strong></span>) I talked about the various ways to deploy applications.   Well this is how and where it can be done in the IDE (File -&gt; Publish).   The Company, Vendor Prefix, and App Name are all filled in for you (you cannot change the Vendor Prefix - this is a ServiceNow only editable field).   You will need to make sure that the Version number is one that has not yet been used, or when you click on the submit button it will detect the fact and kick it back to you.   It is considered a Best Practice to fill in the Dev Notes field.   Something describing what is in the release is usually preferable.<br/></span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="image-5 jive-image" height="346" src="66d25c8adbd49704ed6af3231f961950.iix" style="height: 346px; width: 355.678px;" width="356"/></span></p><p></p><p></p><p><span style="color: #303030; font-size: 14pt; font-family: Arial, sans-serif; font-style: inherit; background-position: initial;"><strong>Outside of or Missing From the Studio IDE</strong></span></p><p></p><p><span style="font-size: 12pt;">So, what do I look at as needing improvement, or needing to be added to the IDE?   What follows are my</span><span style="font-size: 12pt;"> </span><span style="font-size: 16px;">observations.</span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">1. For form Construction only the Form Designer is available (no Form Layout)</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">The Form Designer has a problem that keeps me from using it very often:   It does not automatically name new fields from the Label!   It can be great when moving fields around or creating/modifying sections.   When creating new fields I have to remember to go and rename a field or it will get saved with that automatically generated name that has nothing to do with the real function of the column.   Then I have to go drop the field and re-add it.   Ugh.</span><span style="color: #303030; font-size: 12pt; font-family: Arial;"><br/></span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="image-6 jive-image" height="257" src="459b99cedb1c1b04ed6af3231f9619aa.iix" style="max-width: 1200px; max-height: 900px; width: 537px; height: 256.787px;" width="537"/></span></p><p></p><p><img   class="image-13 jive-image" height="341" src="13d29c8adbd49704ed6af3231f9619f8.iix" style="width: 536px; height: 340.619px;" width="536"/></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">However, I am a big fan of the Form Layout tool because it automatically names a new field from the label. This is a nice time, and annoyance saver!   </span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">Inside the Studio IDE I will not create fields with the Form Designer, but if I have to tweak where things go it is useful.   I <span style="text-decoration: underline;">will</span> will create new fields in the Table form (where it DOES automatically name a new field from a label).   Anyway, you get the idea.   </span></p><p></p><p><img   class="jive-image image-14" height="458" src="2f61f7bddbdc1704ed6af3231f9619eb.iix" style="max-width: 1200px; max-height: 900px; width: 553px; height: 458.331px;" width="553"/></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">2. Previous number check not done when you open the Publish form in the IDE</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">So, not sure why this was not included as a feature inside the IDE.   It works fine from outside via the System Application -&gt; Applications properties form (the "Make App available..." link).   This is basically an inconsistency in the interface.   If you get it wrong and try to re-use a number you will only find out about it after you attempt to publish; the IDE will throw up an error, and ask you to change the version number before continuing.</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">From inside the IDE:</span></p><p></p><p><img   class="image-12 jive-image" src="58c3348adb1cdfc03eb27a9e0f96191c.iix" style="max-width: 1200px; max-height: 900px;"/></p><p></p><p><span style="font-size: 12pt;">From the Properties Page:</span></p><p></p><p><span style="font-size: 12pt;"><img   class="jive-image image-10" src="5feb59c6db58d304b322f4621f9619d0.iix" style="max-width: 1200px; max-height: 900px;"/></span></p><p></p><p></p><p><span style="font-size: 12pt;">Which does this; which is quite a bit friendlier:</span></p><p></p><p><img   class="image-15 jive-image" height="396" src="cbd26502db50d7049c9ffb651f9619e3.iix" style="width: 543px; height: 395.865px;" width="543"/></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">3. Can only push to an Update Set via the Application Properties Form</span></p><p></p><p><span style="font-size: 12pt;"><span style="color: #303030; font-family: Arial; font-size: 16px;">So while I can't do this from inside the IDE, I can do it outside through my Scoped Application's properties page.   </span>You can only push the project to an update set if you navigate to System Applications -&gt; Applications and go into your Application's property page.   </span><span style="color: #303030; font-size: 12pt; font-family: Arial;">I would like to have this in the Studio.   I do a lot of deployment to not-my-company instances, and it would be handier from inside the IDE.</span><span style="color: #303030; font-size: 12pt; font-family: Arial;"> </span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="jive-image image-8" height="321" src="eef8f006dbd89304b322f4621f961989.iix" style="max-width: 1200px; max-height: 900px; width: 536px; height: 320.555px;" width="536"/></span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">4. Testing of any part of the application</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">In order to test your application you will need to jump out of the Studio IDE.   All testing has to be done on the instance itself.   It would be nice to be able to trigger a test of a form or table modification from inside the IDE!</span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">5. Cannot protect script Intellectual Property</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><span style="color: #303030; font-family: Arial; font-size: 16px;">In my article (</span><span style="font-size: 16px; font-family: arial, sans-serif; color: #666666;"><strong><a title="" _jive_internal="true" data-containerid="1007" data-containertype="37" data-objectid="6589" data-objecttype="38" href="/community?id=community_blog&sys_id=d8fd622ddbd0dbc01dcaf3231f9619c9" style="font-weight: inherit; font-style: inherit; font-family: inherit; color: #266fc8;">Community Code Snippets: Scoped Application Distribution</a></strong></span><span style="color: #303030; font-family: Arial; font-size: 16px;">) I describe this particular issue.   You can only protect your code from prying eyes if you push it out to your local Application Repository or the ServiceNow Store.   Otherwise, you cannot protect your hard-earned, hard-to-write code.   It is only possible to deploy to other not-your-company instances via the update set mechanism, and this does not contain any IP protection whatsoever.</span></span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">6. Cannot add Globally scoped files to your project</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">So basically anything Globally scoped is anathema in the Studio IDE.   You can't add Globally scoped files to your application, but you can add them as data (see #11 below).   This is a real problem especially if you have to create Globally scoped scripts to handle things that scoping will not (such as Package calls - see #8 below).</span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">7. Cannot have a Globally scoped Studio application</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">I would really, REALLY like to have the ability to organize my Globally Scoped applications in the same manner is my Scoped applications!   I consider the Studio to be a great tool, but it would be even better if it allowed me this functionality!   Perhaps do this, but don't allow me to deploy to the ServiceNow store?   I would like to toss the Update Set process altogether, and this would allow me to do that!   So now, as a developer, I live between two development worlds: The old Global (which is being called legacy out on the development community), and the new: scoped.   I am seeing the vast majority of new development being done in Global still, so I think it is a bit premature to call this legacy.</span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt;"><span style="font-family: Arial;">8. Scoped applications do not accept any of the Java libraries</span><span style="font-family: Arial;">!</span></span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">I would like to have a list of what is not available inside of Scoped applications.   Instead I am compiling one as I go along.   Nothing about it in the online docs (other than gs.log and Package calls not being allowed).   Anyway, nothing with "Package" on it is available.   There goes Packages.Java.Lang, one of the most useful libraries we have on the platform!   There are several things that do work, but they are mostly from the Glide side of things.</span></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt;"><span style="font-family: Arial;">9. Scoped applications do not accept many of the Global </span><span style="font-family: Arial;">ServiceNow</span><span style="font-family: Arial;"> Libraries (</span><span style="font-family: Arial;">JSUtil</span><span style="font-family: Arial;">, J2JS)!</span></span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">Well, mostly because of #8, we lose several useful JavaScript built-in Global libraries.   I have ended up dissecting JSUtil and extracting code into a Scoped Application Library of my own to get back this much used functionality.   I was able to mostly (sans Packages.Java.Lang.String) get back the JSUtil.nil() and .notNil() functions which are heavily used by the ServiceNow Global base code, and which I find are much more reliable than good ol' gs.nil().   Just saying.</span></p><p></p><p></p><p><span style="color: #303030; font-family: Arial; font-size: 12pt;">10. Cannot Merge Source Control Branches</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">While I can create branches of my application in my external repository (i.e. GitLab); I cannot merge them.   BTW, it doesn't work to go merge them in the repository itself then download the "new" merged copy back to your instance via the Studio.   ServiceNow places the various components of your project out in the repository as a series of xml files, and merging these does not merge the actual differences.   The merge really would have to happen back in the Studio.   This bit of functionality would round out most of what needs doing with Source Control.</span></p><p></p><p></p><p><span style="font-size: 12pt;">BTW, for really great reading on this topic, Microsoft has several patterns on branching and merging that do a good job explaining why both are necessary to development:</span></p><p></p><p style="padding-left: 30px;"><span style="font-size: 12pt;"><a href="https://msdn.microsoft.com/en-us/library/bb668955.aspx" title="https://msdn.microsoft.com/en-us/library/bb668955.aspx">Chapter 5 — Defining Your Branching and Merging Strategy</a> </span></p><p style="padding-left: 30px;"><span style="font-size: 12pt;"><a href="https://msdn.microsoft.com/en-us/library/aa730834(vs.80).aspx" title="https://msdn.microsoft.com/en-us/library/aa730834(vs.80).aspx">Branching and Merging Primer</a> </span></p><p></p><p><img   class="image-16 jive-image" height="350" src="8a6a744adb101b04ed6af3231f961948.iix" style="width: 460px; height: 350.194px;" width="460"/></p><p></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">11. Adding data to the project</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;">This is a cool, but little known feature of the External portion of the Studio.   It is not available from the IDE as you have have the data listed (which you can't do in the tool).   You have the ability to go to a list view and add records as data to your project.   Simply right-click on any List View header (in List v2), or in the tri-lipse (hamburger) button in List v3, and choose the Create Application Files option.     This will then ask you how you want to add the listed data to your Scoped Project.   BTW, this is the trick for adding Global Scripts to your deployment package.</span></p><p></p><p><span style="color: #303030; font-size: 12pt; font-family: Arial;"><img   class="image-9 jive-image" height="340" src="9746d80edb9817049c9ffb651f96193f.iix" style="max-width: 1200px; max-height: 900px; width: 595px; height: 340.086px;" width="595"/></span></p><p></p><p><img   class="image-17 jive-image" height="349" src="53d810c2db945344e9737a9e0f96190c.iix" style="height: 349px; width: 584.811px;" width="585"/></p><p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"></p><p style="margin-bottom: 0.0001pt; font-family: arial, sans-serif; color: #666666;"><span style="font-size: 12pt;">And that is all I have on this topic for the moment!   The Studio IDE is a great tool, and I would sure like to see more integration!   :-)</span></p><p style="margin-bottom: 0.0001pt;
