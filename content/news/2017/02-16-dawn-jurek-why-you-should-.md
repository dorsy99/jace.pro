<p>You're fleshing out an HTML page for your ServiceNow instance, and you'd like to embed some useful content for your users. Can't you just edit the HTML to include an external iFrame in the code? No, you shouldn't, but there's one exception to this rule. So what should you do instead? In this seventh installment of our best practices series, we look at ways to correctly embed content for your ServiceNow instance, and the one time that you can use external iFrames.</p><p></p><p>The &lt;iframe&gt; tag creates an inline frame in an HTML document for the purpose of embedding another document within that HTML document. While this functionality was never documented or supported by ServiceNow, it did work in older versions of the platform. But now, due to security constraints, that functionality is no longer available. So if you try to embed ServiceNow content in an external iFrame, it will not work, or it will break when you upgrade.</p><p></p><p>Note that the operative word here is <em><strong>external</strong></em>. You can use iFrames to your heart's content, as long as they are <em><strong>internal</strong></em>.</p><p></p><h2>How to correctly use iFrames in your ServiceNow instance</h2><p>You can use <em>internal</em> iFrames if you configure them through the <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/content-management/concept/c_ContentManagementSystem.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/content-management/concept/c_ContentManagementSystem.html">Content Management System (CMS)</a>. To add a new iFrame in CMS, navigate to <strong>Content Management &gt; Specialty Content &gt; iFrames</strong> and click <strong>New</strong>. The Product Documentation provides more information about <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/content-management/task/t_IFrame.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/content-management/task/t_IFrame.html">how to configure iFrames</a>.</p><p></p><p><img   alt="iFrame.JPG" class="image-2 jive-image" src="7b49d8c2db989fc03eb27a9e0f9619ad.iix" style="width: 620px; height: 288px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p></p><h2>Another way to deliver ServiceNow content from a web page</h2><p><a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/build/service-portal/concept/c_ServicePortal.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/build/service-portal/concept/c_ServicePortal.html">Service Portal</a> is an alternative to CMS and a simple way to create portals for end users of ServiceNow. It interacts with the underlying ServiceNow platform, so you can access specific platform components within Service Portal and build content quickly.</p><p></p><p>Both expert developers and total beginners can use Service Portal. Less technical users can make basic configuration changes to the UI using Branding Editor and other components of Service Portal, and more advanced users can edit and extend portals, pages, and widgets. Expert users can use the Widget Editor tool to write AngularJS, SCSS, and JavaScript to power a portal, and even create rich web applications on the ServiceNow platform.</p><p></p><p>Check out <a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/build/service-portal/concept/c_QuickStart.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/build/service-portal/concept/c_QuickStart.html">Get Started with Service Portal </a>to explore this option.</p><p></p><p><img   alt="ServicePortalHome.png" class="image-3 jive-image" src="d93e3fb9db141304b322f4621f961936.iix" style="width: 620px; height: 416px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h2>The one exception to the rule - the login portlet</h2><p>Now back to our original quandary. The only time you should ever use an external iFrame is when you want to create an unbranded user and password prompt on an HTML page. By adding the <a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/integrate/inbound-other-web-services/task/t_IntegratingTheLoginPortlet.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/integrate/inbound-other-web-services/task/t_IntegratingTheLoginPortlet.html">login portlet</a> to an iFrame HTML element, the browser transfers the user to a logged-in ServiceNow session once the login is validated.</p><p></p><p><img   alt="PortalLogin.png" class="image-4 jive-image" src="04556186db18d7041dcaf3231f9619fb.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h2>iFrame best practice</h2><p>Do not embed any ServiceNow content—except the <a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/integrate/inbound-other-web-services/task/t_IntegratingTheLoginPortlet.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/integrate/inbound-other-web-services/task/t_IntegratingTheLoginPortlet.html">login portlet</a>, shown above—in an external iFrame. Instead, link directly to the content in your ServiceNow instance.</p><p></p><p style="font-weight: inherit; font-style: inherit; font-family: inherit; color: #666666;">--</p><p></p><p style="font-weight: inherit; font-style: inherit; font-family: inherit;">Behind the scenes here at ServiceNow, the Knowledge Management and Multimedia teams work closely with subject matter experts to disseminate critical information to our customers.
