<p>Want to populate a table on your instance quickly with data, without having to mess around with XML or Transform Maps? If so, <a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/concept/c_EasyImport.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/concept/c_EasyImport.html">Easy Import</a> is the feature you've been looking for. Easy Import is a simplified import process that provides a template with only the columns you want to import. It allows you to import data to tables within the current scope, and tables that grant write access to other applications.</p><p></p><p>In our example, we will be using a free Windows discovery tool, Lansweeper, to take data for a one-time import into our CMDB. In this mock scenario, the user does not have ServiceNow Discovery automatically populating their CMDB with CI data and BSM relationships. In addition, they do not want an integration - just a one-time data load.</p><p></p><p style="text-align: center;"><img   alt="lansweeper+discovery+easy+import.png" class="image-11 jive-image" src="c145694edb101744e9737a9e0f96198f.iix" style="width: 620px; height: 337px;"/></p><p style="text-align: center;"></p><h2 style="text-align: left;">How to use Easy Import for a one-time import of your current asset data into your CMDB</h2><p></p><h3>1. Select the data you want to transfer.</h3><p>After scanning our network using Lansweeper (or any other program), the first task is to select all the data we want to transfer over and save that data as an xls file.</p><p><img   alt="easy+import+data+load.png" class="image-12 jive-image" src="c990c58edb909f048c8ef4621f961906.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><pre __default_attr="info" __jive_macro_name="alert" alert="info" class="jive_text_macro jive_macro_alert" data-renderedposition="988.420166015625_7.986111640930176_1191_62"><p><span style="color: #1287b5;">In this example, Lansweeper allows us to do an "Export to Excel." If the program you're using does not have this functionality, you can either copy and paste your data into Excel or export the data as a CSV file and re-open it in Excel to save as xls.</span></p></pre><h3 style="text-align: left;">2. Check the format (specifically the column titles and field values).</h3><p style="text-align: left;">Once the data from your discovery tool is saved in an xls, ensure the format is set so that row 1 contains all column titles with the relevant data beneath.</p><p style="text-align: left;"><img   alt="easy+import+xls.png" class="image-14 jive-image" src="e8be98c2db9c9fc03eb27a9e0f9619de.iix" style="width: 620px; height: 423px; display: block; margin-left: auto; margin-right: auto;"/></p><ul><li>It is critical to ensure that any column titles in the xls map directly to columns in ServiceNow (this is case sensitive - ServiceNow CMDB field Operating Systems cannot be written in excel as operating systems)! Set the xls columns to the titles of fields in ServiceNow that you are attempting to map your data to. The order of these columns does not matter.</li><li>It is equally important to note that the values of the fields need to map to legal values in ServiceNow. This means that if your column has data that imports into a choice field of choices <strong>A </strong>and <strong>B</strong>, all values must be <strong>A </strong>or <strong>B </strong>in that corresponding column, otherwise data will not import.</li></ul><p style="text-align: left; padding-left: 30px;"></p><p style="text-align: left;">In our example, we need to add additional "Operating System" choices such as Linux to map properly to the data in our excel file. We can do this by navigating to <strong>Personalize &gt; Form Design</strong>.</p><p style="text-align: center;"><img   alt="easy+import+form+design.png" class="image-15 jive-image" src="31ef3b31db58d3049c9ffb651f961954.iix" style="height: auto;"/></p><p style="text-align: left;">From there, we select the choice field Operating Systems that we need to add choices to.</p><p style="text-align: center;"><img   alt="easy+import+operating+system.png" class="image-16 jive-image" src="b1562146db5cd304b322f4621f9619a2.iix" style="width: 620px; height: 231px;"/></p><p style="text-align: left;">Using the field editor on the left, we can add all our discovered Operating System choices that are present in our xls but not in our ServiceNow form.</p><p style="text-align: center;"><img   alt="easy+import+field+editor.png" class="image-17 jive-image" src="6003c5cadbdc9344e9737a9e0f9619fa.iix" style="height: auto;"/></p><p style="text-align: left;">If there is a column in your discovery application that does not exist in ServiceNow, simply create that field using the Form Designer seen above. As long as the column title matches the field name, and the data type matches the field type, the import will be successful.</p><pre __default_attr="info" __jive_macro_name="alert" alert="info" class="jive_text_macro jive_macro_alert" data-renderedposition="2265.642578125_7.986111640930176_1191_42"><p><strong>Tip:</strong> if you are unsure what to select, a String field can handle almost every data type! Please note that reference fields require their destination value to exist prior to import).</p></pre><p style="text-align: center;"></p><h3 style="text-align: left;">3. Start the import.</h3><p style="text-align: left;">Once all additional fields have been successfully added to the ServiceNow table that you are importing into, the next step is to conduct the actual import.</p><p style="padding-left: 30px;"><strong>a.</strong> Navigate to the table you wish to import your CMDB data into. In our example, we have an xls list of discovered computers, so we will navigate to Computers and right-click any column header to select <strong>Import</strong>.</p><p style="text-align: center;"><img   alt="easy+import+cmdb.png" class="image-19 jive-image" src="ec50ad02db10d7049c9ffb651f96191c.iix" style="height: auto;"/></p><p style="text-align: left; padding-left: 30px;"><strong>b. </strong>From the import view, either:</p><ul><ul><li>Select <strong>Insert</strong> if this is the first time you are uploading these assets.</li><li>Select <strong>Update</strong> and then <strong>Create Excel Template</strong> if you are trying to update assets already in your CMDB, to be able to have all the required sys_id's for the update.</li></ul></ul><p style="padding-left: 30px;">Then select the .xls file you created in in steps 1 and 2.</p><p style="text-align: center;"><img   alt="easy+import+upload.png" class="image-20 jive-image" src="11feb882db5013043eb27a9e0f961969.iix" style="height: auto;"/></p><p style="padding-left: 60px; text-align: left;"><strong>c.</strong> After submitting the import, you will be given a preview of all the data that is being added into ServiceNow. Any errors that have occurred will be listed here as well. In our example, we ca see that we have no errors as all our imported fields are mapping cleanly to fields in ServiceNow!</p><p style="padding-left: 60px; text-align: left;"></p><pre __default_attr="info" __jive_macro_name="alert" alert="info" class="jive_text_macro jive_macro_alert" data-renderedposition="3205.347412109375_7.986111640930176_1191_62"><p style="padding-left: 30px;">Note: If your fields are truncating to 40 characters, open a record in the view below, navigate to Form Designer, and edit that variable's max length to be greater than 40 characters.</p></pre><p style="padding-left: 30px; text-align: center;"><img   alt="easy+import+complete.png" class="image-21 jive-image" height="319" src="793d33b1dbd4d3041dcaf3231f9619ac.iix" style="height: 319px; width: 828.098px;" width="828"/></p><p style="text-align: left;"></p><p style="text-align: left;">You have successfully used Easy Import to do a quick, one-time data load in ServiceNow!</p><p style="text-align: left;"></p><p style="text-align: left;">When using Easy Import, there are some properties you need to pay attention to, such as max import row limits (glide.import_template.row_limit) and data type restrictions. For example, by default, Easy Import will not import read-only fields.</p><p style="text-align: left;"></p><p style="text-align: left;">For more information on Easy Import, here are some other resources in our Product Documentation and Community:</p><ul><li><span style="font-family: arial, helvetica, sans-serif; font-size: 10pt;"><a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/reference/r_UseEasyImport.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/reference/r_UseEasyImport.html">Use Easy Import</a></span></li><li><span style="font-family: arial, helvetica, sans-serif; font-size: 10pt;"><a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/reference/r_EasyImportRowLimit.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/reference/r_EasyImportRowLimit.html">Controlling the Easy Import template row limit</a></span></li><li><span style="font-family: arial, helvetica, sans-serif; font-size: 10pt;"><a title="ocs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/reference/r_TemplateValidation.html" href="https://docs.servicenow.com/bundle/helsinki-servicenow-platform/page/administer/import-sets/reference/r_TemplateValidation.html">Easy Import template validation</a></span></li><li><span style="font-family: arial, helvetica, sans-serif; font-size: 10pt;"><a title="" _jive_internal="true" href="/community?id=community_blog&sys_id=ccbc2a25dbd0dbc01dcaf3231f961927">Why Easy Import Is Ignoring You</a></span></li></ul>
