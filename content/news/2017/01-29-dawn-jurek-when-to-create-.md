<p>Starting from scratch. Building from the ground up. Sometimes, there's no way around it. We believe in sparing you from expending unnecessary effort whenever possible. For example, when you want to create a new table for a new application you are building; you may want to look at extending an existing table instead. In this fifth installment of our best practices series, we take a closer look at how leveraging an existing table may be more efficient than creating a new table.</p><p></p><h1><strong>Why extend an existing table instead of creating a new one</strong></h1><p>The ServiceNow platform comes with a robust and thoroughly tested collection of tables, fields, and related scripts to serve a wide variety of needs. For example, the <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/task-table/concept/c_TaskTable.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/task-table/concept/c_TaskTable.html">Task [task] table</a> provides functionality to manage the state of a request for work, including fields that store the state and reference fields that store who is working on it. The Configuration Item [cmdb_ci] table should be used as the base for any new CIs, while the User [sys_user] table contains details of people.</p><p></p><p>As shown below, the <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/task-table/concept/c_TaskTable.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/task-table/concept/c_TaskTable.html">Task table</a> is a parent table to Incident, Problem and various other tables in the system.</p><p></p><p><img   alt="parent_child_tables.png" class="image-3 jive-image" src="982c1d82dbdc9f048c8ef4621f9619ae.iix" style="width: 620px; height: 341px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>Extending a base table incorporates all the functionality of the original table—including fields and scripts—into the new table. This inheritance is used to create major subcategories of data. Extending an existing table can save a lot of development and debugging time.</p><p></p><h2>Determine if a new table or an extended table is more efficient</h2><p>To decide whether to create a new table or extend an existing one, you may want to review the list of tables in the system. Navigate to <strong>System Definition &gt; Tables</strong>, which displays not only whether the table is an extended table (<strong>Extends table</strong> column), but also whether it can be extended as well (<strong>Extensible</strong> column). In the example below, we've outlined the <strong>Alias</strong> table, which extends the <strong>Configuration Item</strong> table, and is extensible itself (Extensible = true). You can extend tables that are marked as extensible if they are in the same scope or if they allow configuration from other scopes.</p><p><img   alt="system_definition_tables1.jpg" class="image-5 jive-image" src="5ac67b71db905fc068c1fb651f961954.iix" style="width: 620px; height: 298px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>To create an extended table, click <strong>New</strong> on the <strong>Tables</strong> list page, and select the table to extend from the <strong>Extends table</strong> search field. Note that this option is only available when <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/table-administration/task/t_CreateATable.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/table-administration/task/t_CreateATable.html">creating a table</a>, as it incorporates all of the fields of the original table and creates system fields for the new table.</p><p></p><p style="text-align: center;"><span style="color: #7ed529;"><img   alt="extends table .jpg" class="image-6 jive-image" src="2cba110edbd813043eb27a9e0f961954.iix" style="width: 620px; height: 367px;"/><br/></span></p><p></p><h3>Table development best practice</h3><p>Instead of creating a new table, consider extending an existing one. If an existing table contains fields, attributes, and scripts similar to those you need, extend that table rather than creating a new one. Then modify the components of your extended table, as needed, to tailor its functionality to the new need.</p><p></p><pre __default_attr="warning" __jive_macro_name="alert" alert="warning" class="jive_text_macro jive_macro_alert" data-renderedposition="1740_8_1192_43"><p>If you are extending the <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/task-table/concept/c_TaskTable.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/task-table/concept/c_TaskTable.html">Task table</a>, you may need a Platform Runtime license to cover the new application you are building.
