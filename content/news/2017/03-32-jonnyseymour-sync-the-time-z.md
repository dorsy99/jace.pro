<p>In ServiceNow all <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/time/reference/r_TimeZones.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/time/reference/r_TimeZones.html">date and times</a> are stored in Coordinated Universal Time (UTC). They are displayed globally based on the system time zone (TZ), but are shown to users in their local time zone, according to his or her user preferences. It can be confusing to set the date and times as you see them, when you seem them. This causes mismatched times on applications and on your PC. Even more complicated, is if you check the saved data, it shows a different time.</p><p></p><p><img   alt="timezones.jpeg" class="image-3 jive-image" height="201" src="12356375db90dfc0b322f4621f96197f.iix" style="width: 268px; height: 201px; display: block; margin-left: auto; margin-right: auto;" width="268"/></p><p></p><p>When you open your application, the date and time will be displayed based on the <a title="ocs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/navigation-and-ui/reference/r_UserPreferenceSettings.html" href="https://docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/navigation-and-ui/reference/r_UserPreferenceSettings.html">User Preferences</a> if it is not set on the <a title="tps//docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/time/task/t_SetASystemTimeZone.html" href="http://https//docs.servicenow.com/bundle/istanbul-servicenow-platform/page/administer/time/task/t_SetASystemTimeZone.html">system time zone</a>. Even when the clients will display time based on the Operating System (OS) timezone, the application will still show the date and time based on the user profile.</p><p></p><h2>Example of a time zone based on the user's preferences</h2><p>The reason it shows 7:38:01 is because the computer is in the <strong>America/Los_Angeles TZ</strong>. Because my PC is in London Timezone, it shows 14:38.</p><p><img   alt="timezone user system.jpg" class="image-8 jive-image" src="607e4d0edb9413043eb27a9e0f9619c2.iix" style="width: 620px; height: 264px; display: block; margin-left: auto; margin-right: auto;"/></p><p>The difference in PC time and application time is caused by a mismatch on the time zones controlled by the "Display" value.</p><p><img   alt="user time zone.jpg" class="image-9 jive-image" src="24a92c06db909344e9737a9e0f961955.iix" style="width: 620px; height: 215px; display: block; margin-left: auto; margin-right: auto;"/></p><p>If I want both time zones to match, I can either make my User profile to match my PC timezone, so my app shows the same time that my PC. Or, I can change the PC time to match the timezone of my user profile.</p><p></p><h2>Changing the time of my User Profile to match the time zone of my PC</h2><p><br/>For example, instead of just changing the time manually*, you should just adjust the User Time Zone field to your location so that this date and times are displayed correctly. This small but signification action proactively prevents this mix-up from happening down the line. It might also be a good practice to have everyone in your organization utilize the user profile Time Zone to match their location so that it updates on its own.</p><p><img   alt="update user timezone.jpg" class="image-10 jive-image" src="ef07000adbdc5704ed6af3231f9619fd.iix" style="width: 620px; height: 246px; display: block; margin-left: auto; margin-right: auto;"/></p><p>Then, if I set the current "closed" time to, it shows the same time as displayed on my PC:</p><p><img   alt="servicenow timezone.jpg" class="image-11 jive-image" src="0aa4e7f9db541fc03eb27a9e0f961931.iix" style="width: 620px; height: 184px; display: block; margin-left: auto; margin-right: auto;"/></p><pre __default_attr="warning" __jive_macro_name="alert" alert="warning" class="jive_text_macro jive_macro_alert" data-renderedposition="1679_8_1177_64"><p>* Please note that changing the time manually (e.g. Manually typing 22:02 ) on the app, when the User profile timezone is incorrect, it will store the time on the _wrong_ UTC (database) time because the system will translate the display value   incorrectly. If you need to manually type a date and time, please ensure your   User profile timezone is correct.</p></pre><p></p><p>The next time you see the date and times on your application are mismatched, remember the time is displayed based on your User profile, and not your client time so they can be different.</p><p></p><p>I have tested using Istanbul and Chrome browser.</p><p></p><p>More
