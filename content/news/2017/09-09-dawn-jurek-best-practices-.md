</iframe></p><p></p><h2 style="text-align: left;">Basics of configuring lists:</h2><ul><li>Know whether your instance uses List v2 or List v3.</li><li>Understand the difference between list personalization and configuration.</li><li>Five best practices for user-friendly lists.</li></ul><p></p><h2>Know whether your instance uses List v2 or List v3</h2><p>List v2 is the default version of lists and is supported in all versions of the user interface (UI). Users with the admin role can <a title="ocs.servicenow.com/bundle/jakarta-servicenow-platform/page/use/using-lists-v3/reference/t_ActivateListV3.html" href="https://docs.servicenow.com/bundle/jakarta-servicenow-platform/page/use/using-lists-v3/reference/t_ActivateListV3.html">activate List v3</a>, which is supported in UI16 only. When configuring lists, it's important to know which list version applies to your instance, because List v2 and List v3 look, feel, and perform differently. See <a title="ocs.servicenow.com/bundle/jakarta-servicenow-platform/page/use/using-lists-v3/reference/r_ComparisonOfListV2V3.html#r_ComparisonOfListV2V3" href="https://docs.servicenow.com/bundle/jakarta-servicenow-platform/page/use/using-lists-v3/reference/r_ComparisonOfListV2V3.html#r_ComparisonOfListV2V3">Comparison of List v2 and List v3</a> for more information.</p><p></p><pre __default_attr="info" __jive_macro_name="alert" alert="info" class="jive_text_macro jive_macro_alert" data-renderedposition="745.9090576171875_7.997159004211426_1192_43"><p>Images in this post are from List v2.</p></pre><p></p><h2>Understand the difference between list personalization and configuration</h2><p>There are two ways to change which columns are displayed in a list: by <em>personalizing</em> the list layout and by <em>configuring</em> the list layout. Each user can personalize the list layout by clicking the Personalize List gear icon in List v2, or from the list title menu in List v3.</p><p></p><p><img   alt="List blog--personalize list gear.png" class="image-10 jive-image" src="08a9a735db981704ed6af3231f961901.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/></p><p>Changes made in this way apply <strong>only</strong> to the logged in user:<span style="color: #7ed529;"> </span></p><p></p><p><img   alt="List blog--Personalize list.png" class="image-2 jive-image" src="2c168846db5c5fc068c1fb651f9619c3.iix" style="width: 620px; height: 358px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><p>By selecting <strong>Configure &gt; List Layout</strong> from any column context menu, users with the appropriate role can configure the list with changes that apply to all users. See <a title="ocs.servicenow.com/bundle/jakarta-servicenow-platform/page/administer/list-administration/task/t_ConfigureTheListLayout.html#t_ConfigureTheListLayout" href="https://docs.servicenow.com/bundle/jakarta-servicenow-platform/page/administer/list-administration/task/t_ConfigureTheListLayout.html#t_ConfigureTheListLayout" style="color: #954f72;">Configure the list layout</a> for more information.</p><p></p><p><img   alt="Lists blog--Configure List Layout.png" class="image-3 jive-image" src="8d80ebf5db1c9f04e9737a9e0f961947.iix" style="width: 620px; height: 427px; display: block; margin-left: auto; margin-right: auto;"/></p><p style="text-align: center;"></p><p><img   alt="List blog--Configure Incident list.png" class="image-5 jive-image" src="3cbf894adb54d344e9737a9e0f9619f6.iix" style="width: 620px; height: 379px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h1>Five best practices for user-friendly lists</h1><ol><li>Use the record number or other unique info as the first column</li><li>Limit the number of columns to minimize horizontal scrolling</li><li>Avoid displaying fields with complex material</li><li>Be aware of which users have access to related operations</li><li>Limit the number of records per page</li></ol><p></p><p></p><h3>Use the record number or other unique information as the first column</h3><p>Clicking an item in the first column in the list should open a record on the list's table. That's why clicking an item in the first column of the Incident list opens an incident record, and clicking an item in the first column of the User list opens a user record. Avoid using a reference field as the first column in a list. If the Incident list showed <strong>Caller</strong> as the first column, a user might click the caller name, expecting to open the incident record. However, clicking an item in the <strong>Caller</strong> column would open the caller's user record, not the incident. A previous blog post, <a title="" _jive_internal="true" href="/community?id=community_blog&sys_id=1b3d2ee5dbd0dbc01dcaf3231f961933" style="color: #954f72;">Where to avoid linking to a reference field when configuring a list</a>, addresses this in more detail.</p><p></p><h3>Limit the number of columns to minimize horizontal scrolling</h3><p>Many users find horizontal scrolling annoying.</p><p style="font-size: 12pt; font-family: Calibri, sans-serif; color: #000000; text-align: center;"><img   alt="Lists blog--horizontal scrolling.png" class="image-13 jive-image" src="48268082db181344e9737a9e0f9619a0.iix" style="width: 620px; height: 195px;"/></p><p></p><h3>Avoid displaying fields that contain complex material</h3><p><span style="font-size: 11pt;">List views are intended to provide a meaningful overview of multiple records simultaneously. Fields that contain long strings of text—such as journal entries or HTML, images, or other complex material—take up a lot of vertical space in list views, so users have to scroll further to see more records. So it's best to avoid displaying these fields. Depending on the instance configuration, users can still see them by personalizing their list view or opening the record.</span></p><p style="text-align: center;"></p><p style="font-size: 12pt; font-family: Calibri, sans-serif; color: #000000;"><img   alt="Lists blog--long fields.png" class="image-7 jive-image" src="147ba8cadbd857041dcaf3231f961912.iix" style="width: 620px; height: 217px; display: block; margin-left: auto; margin-right: auto;"/></p><p></p><h3>Be aware of which users have access to the related operations</h3><p><span style="font-size: 11pt;">Review the list controls and list UI actions to understand which users have access to the related operations. For example, if you expect users to create new incidents from a record producer instead of the Incident list, configure the Incident list controls to omit the <strong>New</strong> button. Here's how to check list controls and UI actions:</span></p><p style="text-align: center;"></p><p style="font-size: 12pt; font-family: Calibri, sans-serif; color: #000000; text-align: center;"><img   alt="List blog--List controls and UI actions.jpg" class="image-12 jive-image" src="9ac7680adb509304b322f4621f96193f.iix" style="width: 620px; height: 322px;"/></p><p style="text-align: center;"></p><h3>Limit the number of records per page</h3><p><span style="font-size: 11pt;">For performance reasons, consider limiting the number of records that are displayed per page in a list. The available values in the base system are 10, 15, 20, 50, and 100, with 20 selected by default. The more rows per page selected, the more slowly each page loads. Users can list more or fewer records per page from the list context menu:</span></p><p></p><p style="font-size: 12pt; font-family: Calibri, sans-serif; color: #000000;"><img   alt="List blog--number of records.png" class="image-11 jive-image" src="ec88494edb50d304b322f4621f96192e.iix" style="height: auto; display: block; margin-left: auto; margin-right: auto;"/> <span style="font-size: 11pt;">Users with the admin role can change the available rows per page options by navigating to </span><span style="font-size: 11pt;"><strong>System Properties &gt; System</strong></span><span style="font-size: 11pt;"> and changing the </span><span style="font-size: 11pt;"><strong>'Items per page' drop-down options (comma separated, no spaces)</strong></span><span style="font-size: 11pt;">:</span></p><p style="text-align: center;"></p><p style="font-size: 12pt; font-family: Calibri, sans-serif; color: #000000; text-align: center;"><img   alt="List blog--system properties.png"
