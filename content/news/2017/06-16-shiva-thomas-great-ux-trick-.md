<p><img   alt="Header.png" class="image-1 jive-image" src="3e9299cedb54d304b322f4621f9619e5.iix" style="width: 620px; height: 131px;"/></p><p></p><p>As a software platform, ServiceNow has a great and modern GUI that works with any web browser. Yet there is one glaring missing <a title="n.wikipedia.org/wiki/User_interface" href="https://en.wikipedia.org/wiki/User_interface">UI</a> feature from modern web technology: HTML Placeholders. Let me present you this <em>ninj'admin</em> trick that you can use on your implementations.</p><p></p><h2>What are HTML Placeholders?</h2><p>Placeholders are the ghost texts present in many web sites' input fields. They are an <a title="ruitionpartners.eu/blog/2012/07/02/feature-coverage-embedded-help-text/" href="https://fruitionpartners.eu/blog/2012/07/02/feature-coverage-embedded-help-text/">old trick</a> introduced with the <a title="n.wikipedia.org/wiki/HTML5" href="https://en.wikipedia.org/wiki/HTML5">HTML5 norm</a>, back in 2011.</p><p>The most common usage is with Search input fields.</p><p><img   alt="Placeholders - Search.png" class="image-2 jive-image" src="64fe684edb98d704ed6af3231f9619af.iix" style="height: auto;"/></p><p>As soon as you type something in the box, the placeholder text will disappear, to be replaced by your own input.</p><p>Used in the Service Catalog, or in your Forms, they can add useful information without being obnoxious for the users.</p><p><img   alt="Placeholders - SC with.png" class="image-3 jive-image" src="da5e8442dbd09fc068c1fb651f96190a.iix" style="width: 620px; height: 177px;"/></p><p></p><p>ServiceNow current help system for the Service Catalog is way more obnoxious to my liking. Placeholders are way better design-wise.</p><p><img   alt="Placeholders - SC without.png" class="image-4 jive-image" src="854ee3b1dbd81fc068c1fb651f961930.iix" style="width: 620px; height: 334px;"/></p><h2></h2><h2>Native support in Jakarta Service Catalog</h2><p><strong>In Jakarta, HTML Placeholders can only be configured for for Service Catalog variables</strong> <span __jive_emoticon_name="cry" __jive_macro_name="emoticon" class="jive_macro_emoticon jive_emote jive_macro" data-renderedposition="1104_590.703125_16_16" src="/8.0.4.21bdc7e/images/emoticons/cry.png"></span> via the field "Example Text" that you'll have to add to the Variable Form.</p><p>You can only apply them for: Single Line Text, Wide Single Line Text, Multi Line Text, IP Address, Email, URL, Date, Date/Time.</p><p>In Jakarta, there is no native function to add HTML Placeholder to your forms outside Service Catalog Variable.</p><p></p><h2>Not using Jakarta? Want them outside the Service Catalog? Script your placeholders</h2><p>If you are using an older version, you can use this simple client script to add placeholders to your forms.</p><p></p><h5>Platform interface</h5><p>I recommend adding the feature globally using a UI Script include.</p><p><img   alt="Placeholders - UI Script.png" class="image-5 jive-image" src="3bd35942db18130468c1fb651f961928.iix" style="width: 620px; height: 462px;"/></p><p>You'll now be able to add lean onLoad Client Script to the forms of relevant tables.</p><p><img   alt="Placeholders - OnLoad Client Script.png" class="image-6 jive-image" src="981d604adb58d704ed6af3231f96194e.iix" style="width: 620px; height: 451px;"/></p><p>Another great thread on the subject: <a title="Support for HTML 5 placeholder attribute" __default_attr="152916" __jive_macro_name="thread" class="jive_macro_thread jive_macro" data-orig-content="Support for HTML 5 placeholder attribute" data-renderedposition="2287_235.25_273_16" href="/community?id=community_question&sys_id=06dc4729db9cdbc01dcaf3231f961951">Support for HTML 5 placeholder attribute</a></p><p></p><h5>Catalog Items &amp; Record Producers</h5><p>The same design works, with some minor adaptations. The scripts should be added to the 'Catalog Client Script' table instead. Also, catalog items and record producers will not load the UI Scripts, so you'll have to copy the whole <em>addHtlmPlaceholder</em> function directly in each 'Catalog Client Script'.</p><p><img   alt="Placeholders - Catalog Client Script.png" class="image-7 jive-image" src="078c80c6dbdcdb048c8ef4621f961922.iix" style="width: 620px; height: 571px;"/></p><h2>Styling Placeholders</h2><p>Placeholders appearance can be <a title="ss-tricks.com/almanac/selectors/p/placeholder/" href="https://css-tricks.com/almanac/selectors/p/placeholder/">easily modified via CSS</a>, using the<em>:</em> <em>:placeholder</em> pseudo element.</p><p></p><h2>Limitations</h2><ul><li>Placeholder texts must not contain <strong>carriage returns</strong> or line-feeds.</li><li>They won't be applied to <strong>reference fields</strong>, nor on journaled fields grouped under UI16's Activity.</li><li>They will not work with the <strong>Service Portal or mobile interfaces</strong>. The <a title="eveloper.servicenow.com/app.do#!/api_doc?v=geneva&id=c_MobileGlideForm_API" href="https://developer.servicenow.com/app.do#!/api_doc?v=geneva&amp;id=c_MobileGlideForm_API">Mobile GlideForm</a> (g_form) is currently too limited, and DOM Manipulation using stuff like $ and jQueryare prohibited by the platform.</li><li>In <strong>scoped applications</strong>, ServiceNow blocks updating DOM objects.
