<p>When you move to <a title="ocs.servicenow.com/bundle/jakarta-servicenow-platform/page/build/service-portal/concept/c_ServicePortal.html" href="https://docs.servicenow.com/bundle/jakarta-servicenow-platform/page/build/service-portal/concept/c_ServicePortal.html">Service Portal</a>, there is a possibility that the <a title="i.service-now.com/kb_view.do?sysparm_article=KB0623310" href="https://hi.service-now.com/kb_view.do?sysparm_article=KB0623310">client scripts</a> defined on catalog items and record producers do not work on Service Portal. The reason is that Service Portal is a mobile framework and when you created the scripts on platform few months or years ago you didn't consider if the script should apply to <a title="ocs.servicenow.com/bundle/jakarta-servicenow-platform/page/administer/tablet-mobile-ui/concept/c_MobileWebUI.html" href="https://docs.servicenow.com/bundle/jakarta-servicenow-platform/page/administer/tablet-mobile-ui/concept/c_MobileWebUI.html">mobile platform</a> as well. If you were not using the ServiceNow native mobile app, then it is more likely to happen. The good news is that you don't have to write more code in your client scripts to make it compatible for Service Portal, it is just a simple configuration change.</p><p><img   class="image-1 jive-image" height="272" src="d12f284adb949304b322f4621f9619aa.iix" style="display: block; margin-left: auto; margin-right: auto; width: 437px; height: 272.068px;" width="437"/></p><p></p><h1>What to do when Catalog Client Scripts do not work on Service Portal</h1><p>You need to go back to the definition of client script and make it available for <span style="text-decoration: underline;">Mobile or both Desktop and Mobile</span>. Yes, it's that simple. Follow the steps below, and keep in mind that in some older versions the selection options will be different.</p><p></p><ol><li>Navigate to <strong>Maintain Items </strong>in the Filter Navigator and select the catalog item where you defined the client script.<p style="text-align: center;"><img   alt="Service catalog portal script.jpg" class="image-9 jive-image" src="47f89502db1c1b04ed6af3231f961953.iix" style="width: 620px; height: 357px;"/></p></li><li>Click on the options button (hamburger icon) and select<strong> Configure &gt; Client scripts</strong>.<p style="text-align: center;"><img   alt="Service catalog portal script 1.jpg" class="image-10 jive-image" src="5d93b4c2db5097041dcaf3231f961961.iix" style="height: auto;"/></p></li><li>On the definition of catalog client script, you need to change the value of field: "UI Type" to either "All" (if using both platform and Service Portal) or "Mobile/Service Portal" (if using only Service Portal)<p style="text-align: center;"><img   alt="Service catalog portal script 2.jpg" class="image-11 jive-image" src="8a51940adb149fc03eb27a9e0f961962.iix" style="width: 620px; height: 686px;"/></p></li></ol><p>In some old versions, the value options may look like this. Either select "Both" (if using both platform and Service Portal) or "Mobile" (if using only Service Portal)</p><p><img   alt="Service catalog portal script 3.jpg" class="image-12 jive-image" src="474fef7ddbd81fc068c1fb651f961991.iix" style="width: 620px; height: 606px; display: block; margin-left: auto; margin-right: auto;"/></p><p>While you should make sure that the script is applicable on Mobile UI type, you must also make sure the script is compatible with the <a title="ocs.servicenow.com/bundle/jakarta-servicenow-platform/page/build/service-portal/reference/supported-client-scripts.html" href="https://docs.servicenow.com/bundle/jakarta-servicenow-platform/page/build/service-portal/reference/supported-client-scripts.html">mobile scripting or Portal supported APIs. </a></p><p></p><p>You can use onLoad, onChange, and onSubmit client scripts.</p><ul><li>addDecoration(fieldName, icon, title)</li><li>addErrorMessage(message)</li><li>addInfoMessage(message)</li><li>addOption(fieldName, value, label, index)</li><li>clearMessages()</li><li>clearOptions(fieldName)</li><li>clearValue(fieldName)</li><li>getActionName()</li><li>getBooleanValue(fieldName)</li><li>getDecimalValue(fieldName)</li><li>getDisplayValue(fieldName)</li><li>getEncodedRecord()</li><li>getFieldNames()</li></ul><p></p><p><img   class="image-5 jive-image" height="97" src="ef9e88cedb101f048c8ef4621f9619d2.iix" style="width: 97px; height: 97px;" width="97"/></p><h2>Using Client Scripts on Variable Editors and Order Guides</h2><p>Since we are on the Catalog client scripts, let me clarify one last thing. If you have selected the checkbox for "Applies on requested Item" or/and "Applied on catalog tasks" to make it work for Variable Editors, then you may run into an issue with the variable editor.</p><p style="text-align: center;"><img   alt="catalog client script.jpg" class="image-6 jive-image" src="8df8abf5dbd493049c9ffb651f961933.iix" style="width: 620px; height: 195px;"/></p><p></p><p></p><p></p><pre __default_attr="info" __jive_macro_name="alert" alert="info" class="jive_text_macro jive_macro_alert" data-renderedposition="3256.40625_8_1177_64">Unfortunately, it won't work on Service Portal because vEditor or default Variable Editor is not yet supported on Service Portal. We are working on it, appreciate your patience while we fix it Â  :) </pre><p></p><p>If you use Order Guides and do <a title="ocs.servicenow.com/bundle/jakarta-it-service-management/page/product/service-catalog-management/task/t_CreateAnItemVarAssignment.html" href="https://docs.servicenow.com/bundle/jakarta-it-service-management/page/product/service-catalog-management/task/t_CreateAnItemVarAssignment.html">item variable assignments</a>, that won't work on Service Portal. There will be huge changes in Service Catalog's default behavior on Service Portal in the next upcoming release, so stay tuned. I will definitely brag about it.</p>
